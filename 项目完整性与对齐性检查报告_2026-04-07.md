### 2.2 发现问题

#### 问题 1: Admin 路由未注册到主 API 路由

**严重程度**: 🔴 高  
**位置**: `backend/app/api/__init__.py`  
**描述**: Admin 模块的所有路由已实现，但未在主 API 路由中注册，导致 API 端点无法访问。

**当前代码**:
```python
# backend/app/api/__init__.py
api_router.include_router(auth_router, prefix="/auth", tags=["认证鉴权"])
api_router.include_router(users_router, prefix="/users", tags=["用户模块"])
# ... 其他路由
# 缺少 admin 路由注册
```

**建议修复**:
```python
from app.api.v1.admin import (
    audit_router,
    config_router,
    content_router,
    dashboard_router,
    feedback_router,
    llm_provider_router,
    packages_router,
    users_router as admin_users_router,
)

api_router.include_router(admin_users_router, prefix="/admin/users", tags=["管理后台-用户管理"])
api_router.include_router(content_router, prefix="/admin", tags=["管理后台-内容审核"])
api_router.include_router(audit_router, prefix="/admin", tags=["管理后台-审计日志"])
api_router.include_router(config_router, prefix="/admin", tags=["管理后台-系统配置"])
api_router.include_router(feedback_router, prefix="/admin", tags=["管理后台-反馈处理"])
api_router.include_router(packages_router, prefix="/admin", tags=["管理后台-扩容包管理"])
api_router.include_router(dashboard_router, prefix="/admin", tags=["管理后台-仪表盘"])
api_router.include_router(llm_provider_router, prefix="/admin", tags=["管理后台-LLM配置"])
```

#### 问题 2: Payment 路由未注册到主 API 路由

**严重程度**: 🟡 中  
**位置**: `backend/app/api/__init__.py`  
**描述**: Payment 模块的路由文件存在，但未在主 API 路由中注册。

**建议修复**: 在 `api/__init__.py` 中添加 payment 路由注册。
