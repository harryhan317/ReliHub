## 1. 迁移概述

### 1.1 迁移背景

根据项目技术栈要求，所有测试数据库必须严格采用 PostgreSQL 架构，以确保测试结果与生产环境行为完全一致。本次迁移将所有使用 SQLite 的测试文件迁移至 PostgreSQL。

### 1.2 迁移目标

| 目标项 | 说明 |
|--------|------|
| **数据库统一** | 所有测试使用 PostgreSQL 数据库 |
| **外键约束一致性** | 确保测试阶段能发现数据完整性问题 |
| **数据类型兼容性** | 消除 SQLite 与 PostgreSQL 数据类型差异 |
| **生产环境对齐** | 测试结果与生产环境行为完全一致 |

## 4. 迁移实施方案

### 4.1 迁移策略

1. **共享 Fixtures 设计**: 在 `conftest.py` 中创建 PostgreSQL 测试数据库连接 fixtures
2. **事务隔离**: 每个测试用例使用独立事务，测试结束后自动回滚
3. **逐步迁移**: 按文件逐个迁移，确保每个文件迁移后测试通过

### 4.2 迁移步骤

```
步骤 1: 识别所有 SQLite 引擎创建代码
步骤 2: 移除 SQLite 引擎创建和会话管理代码
步骤 3: 添加 db_session fixture 依赖
步骤 4: 验证测试通过
步骤 5: 更新技术文档
```

### 4.3 代码变更示例

**迁移前 (SQLite)**:
```python
@pytest.fixture
def db_session(self):
    engine = create_engine("sqlite:///:memory:", echo=False)
    Base.metadata.create_all(engine)
    SessionLocal = sessionmaker(bind=engine)
    session = SessionLocal()
    yield session
    session.close()
```

**迁移后 (PostgreSQL)**:
```python
class TestFileUpload:
    @pytest.fixture
    def file_service(self, db_session):
        return FileService(db_session)
```

## 8. 迁移过程中遇到的问题与解决方案

### 8.1 问题清单

| 序号 | 问题描述 | 影响范围 | 解决方案 |
|------|----------|----------|----------|
| 1 | SQLite 不强制外键约束 | 数据完整性 | 迁移到 PostgreSQL 后自动检测并修复 |
| 2 | 测试用例使用不存在的外键 | test_ai_integration.py | 先创建父记录再创建子记录 |
| 3 | 多个测试类重复定义 db_session | 代码冗余 | 移除类级别 fixture，使用共享 fixture |

### 8.2 解决方案详情

#### 问题 1: 外键约束不一致

**现象**: 在 SQLite 中可以插入无效外键，但在 PostgreSQL 中会抛出 IntegrityError

**解决方案**: 
- 修复测试用例，确保先创建父记录
- 这实际上帮助发现了潜在的数据完整性问题

#### 问题 2: 测试类重复定义

**现象**: 多个测试类各自定义 db_session fixture

**解决方案**:
- 移除类级别的 db_session fixture
- 统一使用 conftest.py 提供的共享 fixture

```python
@pytest.fixture(scope="function")
def db_session(test_engine):
    """Create a database session for each test."""
    connection = test_engine.connect()
    transaction = connection.begin()
    session = Session(bind=connection)
    yield session
    session.close()
    transaction.rollback()
    connection.close()
```
