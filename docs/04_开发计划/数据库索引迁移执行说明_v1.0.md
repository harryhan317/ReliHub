# 数据库索引迁移执行说明

**创建日期**: 2026-04-10  
**迁移版本**: sprint_e_search_indexes  
**适用环境**: PostgreSQL 14+

## 迁移前准备

### 1. 确认数据库类型

本迁移脚本针对 PostgreSQL 数据库优化，需要在 PostgreSQL 环境下执行。

### 2. 安装 pg_trgm 扩展

PostgreSQL 需要安装 pg_trgm 扩展以支持 GIN 三字符索引：

```sql
-- 连接到数据库
psql -U postgres -d relihub

-- 创建 pg_trgm 扩展
CREATE EXTENSION IF NOT EXISTS pg_trgm;

-- 验证扩展已安装
\dx

-- 应该看到：
--  Schema |  Name   | Version | Description 
-- --------+---------+---------+-------------------------------------
--  public | pg_trgm | 1.6     | text similarity measurement
```

### 3. 备份数据库

```bash
# 备份数据库
pg_dump -U postgres relihub > backup_$(date +%Y%m%d).sql
```

## 执行迁移

### 方法一：使用 Alembic（推荐）

```bash
cd backend

# 查看当前版本
python3 -m alembic current

# 执行迁移
python3 -m alembic upgrade sprint_e_search_indexes

# 验证迁移结果
python3 -m alembic current
```

### 方法二：手动执行 SQL

如果 Alembic 迁移失败，可以手动执行 SQL：

```sql
-- 创建扩展
CREATE EXTENSION IF NOT EXISTS pg_trgm;

-- Resource 表索引
CREATE INDEX idx_resources_title ON resources (title);
CREATE INDEX idx_resources_description ON resources USING gin (description gin_trgm_ops);
CREATE INDEX idx_resources_category_status_created ON resources (category_id, status, created_at);
CREATE INDEX idx_resources_tags ON resources USING gin (tags gin_trgm_ops);
CREATE INDEX idx_resources_fts ON resources USING gin(to_tsvector('simple', coalesce(title, '') || ' ' || coalesce(description, '')));

-- Topic 表索引
CREATE INDEX idx_topics_title ON topics (title);
CREATE INDEX idx_topics_content ON topics USING gin (content gin_trgm_ops);
CREATE INDEX idx_topics_category_heat ON topics (category_id, heat_score DESC);
CREATE INDEX idx_topics_fts ON topics USING gin(to_tsvector('simple', coalesce(title, '') || ' ' || coalesce(content, '')));

-- AI Session 表索引
CREATE INDEX idx_ai_sessions_title ON ai_sessions USING gin (title gin_trgm_ops);
CREATE INDEX idx_ai_sessions_user_created ON ai_sessions (user_id, created_at);

-- User 表索引
CREATE INDEX idx_users_nickname ON users USING gin (nickname gin_trgm_ops);
```

## 故障排查

### 问题 1: pg_trgm 扩展不存在

**错误信息**:
```
operator class "gin_trgm_ops" does not exist for access method "gin"
```

**解决方案**:
```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

### 问题 2: 权限不足

**错误信息**:
```
permission denied to create extension
```

**解决方案**:
```sql
-- 使用超级用户执行
psql -U postgres -d relihub
CREATE EXTENSION pg_trgm;
```

### 问题 3: 迁移版本冲突

**错误信息**:
```
Multiple head revisions are present
```

**解决方案**:
```bash
# 查看头部分支
python3 -m alembic heads

# 指定具体分支
python3 -m alembic upgrade sprint_e_search_indexes

# 或者合并分支（需要谨慎）
python3 -m alembic merge heads
```

## 生产环境执行注意事项

1. **选择低峰时段**: 索引创建可能锁表，建议在业务低峰期执行
2. **监控资源使用**: 创建 GIN 索引可能消耗较多 CPU 和内存
3. **逐步执行**: 可以先创建普通索引，再创建 GIN 索引
4. **监控进度**: 使用 `pg_stat_progress_create_index` 监控索引创建进度
5. **备份验证**: 确保备份可用，以便紧急情况下恢复
