## 🎯 设计原则

### 1. 用户中心设计
```
✓ 以用户需求为出发点
✓ 简化用户操作流程
✓ 提供清晰的反馈
✓ 降低认知负荷
```

### 2. 一致性原则
```
✓ 统一的视觉风格
✓ 一致的交互模式
✓ 统一的信息架构
✓ 统一的微交互
```

### 3. 可用性原则
```
✓ 易于学习
✓ 易于使用
✓ 易于记忆
✓ 错误恢复快
```

### 4. Apple 设计哲学
```
✓ 极简主义
✓ 深思熟虑的间距
✓ 优美的排版
✓ 精心的色彩搭配
✓ 流畅的动画
```

```
✓ Color Consistency (一致性)
  └─ 同类型组件使用相同色彩
  └─ 同状态组件使用相同色彩

✓ Color Psychology (色彩心理)
  └─ 绿色 = 成功/正向
  └─ 红色 = 危险/警告
  └─ 蓝色 = 信息/品牌
  └─ 橙色 = 注意/次要
```

## 📝 排版系统

### 字体

```
Primary Font: SF Pro Display (系统默认)
Fallback: -apple-system, BlinkMacSystemFont, "Segoe UI"

Font Weights:
├─ Regular: 400
├─ Medium: 500
├─ Semibold: 600
└─ Bold: 700
```

### 排版等级

```
H1 - Display Large
├─ Font Size: 32pt
├─ Font Weight: 700 (Bold)
├─ Line Height: 38pt (1.19)
├─ Letter Spacing: 0
└─ 用途: 页面主标题

H2 - Display Medium
├─ Font Size: 28pt
├─ Font Weight: 700 (Bold)
├─ Line Height: 34pt (1.21)
├─ Letter Spacing: 0
└─ 用途: 模块标题

H3 - Display Small
├─ Font Size: 22pt
├─ Font Weight: 600 (Semibold)
├─ Line Height: 28pt (1.27)
├─ Letter Spacing: 0
└─ 用途: 子模块标题

H4 - Headline
├─ Font Size: 17pt
├─ Font Weight: 600 (Semibold)
├─ Line Height: 22pt (1.29)
├─ Letter Spacing: -0.4pt
└─ 用途: 卡片标题

Body - Body Large
├─ Font Size: 16pt
├─ Font Weight: 400 (Regular)
├─ Line Height: 22pt (1.38)
├─ Letter Spacing: -0.2pt
└─ 用途: 正文内容

Caption1 - Caption Large
├─ Font Size: 12pt
├─ Font Weight: 400 (Regular)
├─ Line Height: 16pt (1.33)
├─ Letter Spacing: 0
└─ 用途: 辅助文字

Caption2 - Caption Small
├─ Font Size: 10pt
├─ Font Weight: 400 (Regular)
├─ Line Height: 14pt (1.4)
├─ Letter Spacing: 0
└─ 用途: 标签/备注
```

### 排版应用规则

```
✓ 标题优先级: H1 > H2 > H3 > H4
✓ 正文可读性: 最小 16pt
✓ 行高舒适度: 正文行高 ≥ 字号 × 1.2
✓ 对比强调: 标题与正文字号比例 ≥ 1.3
✓ 对齐方式: 左对齐优先
✓ 行宽舒适度: 正文 40-70 字/行
```

## 🎒 间距系统

### 基础栅格

```
基础单位: 8pt (1 unit)

倍数系统:
├─ 4pt (0.5x) - 极小间距
├─ 8pt (1x) - 小间距
├─ 12pt (1.5x) - 中间距
├─ 16pt (2x) - 标准间距
├─ 24pt (3x) - 大间距
├─ 32pt (4x) - 超大间距
└─ 48pt (6x) - 极大间距
```

### 组件内间距 (Padding)

```
Button
├─ Horizontal: 16pt
└─ Vertical: 12pt

Card
├─ Horizontal: 16pt
└─ Vertical: 16pt

Cell
├─ Horizontal: 16pt
└─ Vertical: 12pt

Form Input
├─ Horizontal: 12pt
└─ Vertical: 10pt
```

### 组件间间距 (Margin)

```
页面顶部间距: 16pt
页面底部间距: 16pt
页面左右间距: 16pt
组件间竖向间距: 8pt / 16pt
列表项间距: 0pt (分割线代替)
卡片组间距: 16pt
```

### 应用规则

```
✓ 始终使用 8pt 栅格
✓ 间距保持一致
✓ 避免随意设置间距
✓ 使用倍数关系
✓ 页面与组件间距一致
✓ 考虑内容密度
```

## 📐 响应式设计

### 屏幕尺寸断点

```
Mobile Small: 320pt (iPhone SE)
├─ Padding: 12-16pt
├─ Font Size: 14-16pt
└─ Grid: 2 列

Mobile Standard: 375pt (iPhone 12)
├─ Padding: 16pt
├─ Font Size: 16-17pt
└─ Grid: 2 列

Mobile Large: 414pt (iPhone 12 Pro Max)
├─ Padding: 16-20pt
├─ Font Size: 16-17pt
└─ Grid: 2 列

Tablet: 768pt (iPad)
├─ Padding: 20-24pt
├─ Font Size: 16-18pt
└─ Grid: 3-4 列

Desktop: 1024pt+
├─ Padding: 24-32pt
├─ Font Size: 16-18pt
└─ Grid: 4-6 列
```

### 响应式规则

```
✓ 流体布局: 使用百分比而非固定宽度
✓ 灵活间距: 相对间距而非绝对值
✓ 自适应字体: 使用相对字号
✓ 优先级: 移动优先，渐进增强
✓ 图片优化: 根据设备尺寸加载
✓ 性能: 减少移动设备上的动画
```
