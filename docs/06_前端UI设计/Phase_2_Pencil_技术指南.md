## 🎯 项目设置

### 创建新 Pencil 文件

**步骤**:
```
1. 打开 Pencil
2. 文件 > 新建
3. 命名: ReliHub-Prototypes.pen
4. 设置画布尺寸: 1440 x 900 (Desktop)
5. 保存到: /docs/06_前端UI设计/prototypes/
```

### 初始化设置

**第一步: 创建色板**
- 创建一个页面: `00_Color_Styles`
- 添加所有 12+ 颜色
- 为每个颜色创建样式

**第二步: 创建排版样式**
- H1, H2, H3, H4: 标题
- Body: 正文
- Label: 标签
- Caption: 说明文字

**第三步: 创建基础组件**
- 按钮 (主要、次要、禁用)
- 输入框 (默认、焦点、错误)
- 卡片
- 加载器

## 🔧 交互指南

### 交互类型参考

#### 1. 页面转换 (Page Transition)
```
操作: 对象 → 鼠标点击
目标: 转到另一页面
方向: 水平/竖直
速度: 中等 (0.3-0.5秒)
```

#### 2. 状态变化 (State Change)
```
禁用按钮: 灰色 + 不可点击
焦点输入框: 蓝色边框 + 光标
加载状态: 旋转动画
错误状态: 红色边框 + 错误文本
```

#### 3. 模态框 (Modal Dialog)
```
操作: 点击按钮
显示: 半透明遮罩 + 对话框
交互: 可关闭 (X按钮/取消)
```

#### 4. 动画 (Animation)
```
加载动画: 旋转 360° (无限循环)
滑动动画: 从底部滑入 (0.3秒)
淡入淡出: 透明度变化 (0.2秒)
```

### Pencil 交互设置步骤

```
1. 选择要添加交互的对象
2. 右侧面板 > Interactions
3. 选择交互类型 (Click, Hover, Load)
4. 设置目标页面或动作
5. 配置动画效果 (可选)
6. 测试交互 (预览模式)
```

## ✨ 最佳实践

### 1. 命名规范

```
页面名称:
- 模块名称_页面描述
- 例: Onboarding_Login, AI_Chat_Detail

对象命名:
- 功能_状态 或 功能_类型
- 例: SubmitButton, InputField_Error

颜色和样式:
- 用途_色调 或 用途_状态
- 例: PrimaryButton, Text_Secondary
```

### 2. 代码复用

```
创建主组件:
1. 设计一个完整的登录表单
2. 作为主组件保存
3. 在其他页面复用 (拖拽)
4. 修改主组件自动更新副本

好处:
- 保证一致性
- 节省时间
- 易于维护
```

### 3. 层级结构

```
使用分组 (Group) 管理复杂布局:
- 页头组
- 内容组
- 页脚组

使用锁定 (Lock) 保护基础元素:
- 背景
- 参考线
- 固定菜单
```

### 4. 注释和文档

```
为每个页面添加注释:
- 页面目的
- 关键交互
- 特殊说明

示例:
"登录页 - 用户输入邮箱和密码"
"关键交互: 邮箱验证, 密码显示"
```
