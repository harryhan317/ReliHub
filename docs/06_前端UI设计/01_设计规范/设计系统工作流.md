## 🎨 Pencil 交互原型

### 什么是 Pencil?

Pencil 是一个开源的 UI 原型设计工具，现集成为 VS Code 插件。

**核心特性**:
- ✅ 快速画出原型
- ✅ 明确交互流程
- ✅ 支持导出 PDF/图片
- ✅ 轻量级，易于学习
- ✅ 集成 VS Code

### Pencil 快速开始

#### 1. 安装 Pencil 插件

在 VS Code 中:
```
Ctrl/Cmd + Shift + X → 搜索 "Pencil" → 安装
```

或者在插件市场搜索: `Pencil`

#### 2. 创建新项目

```
VS Code 左侧 Pencil 图标 → New Document → 创建
```

输入文件名: `模块名_prototype.pen`

#### 3. 基本操作

**画布导航**:
- 缩放: 鼠标滚轮
- 平移: 按住空格 + 拖拽
- 重置: Ctrl/Cmd + 0

**常用快捷键**:
| 快捷键 | 功能 |
|--------|------|
| R | 矩形 |
| C | 圆形 |
| T | 文字 |
| L | 线条 |
| I | 图片 |
| Z | 框选缩放 |
| G | 分组 |
| Ctrl/Cmd + Z | 撤销 |
| Ctrl/Cmd + S | 保存 |

#### 4. Pencil 中的原型设计流程

**示例: 爱问AI助手模块的原型设计**

##### Step 1: 创建屏幕 (Screen)

```
画布 → 右键 → Add Screen → 输入屏幕名 "AI 对话首屏"
```

或使用工具栏的 "Screen" 按钮

**推荐的屏幕尺寸** (移动设计):
- iPhone 14: 390 × 844px
- iPad: 768 × 1024px
- Desktop: 1440 × 900px

##### Step 2: 绘制 UI 元素

在屏幕上绘制以下元素:

```
AI 对话首屏布局:
┌─────────────────┐
│  导航栏          │  (Header, 50px)
├─────────────────┤
│                 │
│  消息区域        │  (Message List, 剩余高度)
│                 │
├─────────────────┤
│ [输入框] [发送] │  (Input Footer, 60px)
└─────────────────┘
```

**绘制步骤**:

1. **绘制导航栏** (矩形)
   - 工具栏选择 "Rectangle"
   - 在画布顶部绘制 390×50 的矩形
   - 右侧属性面板设置颜色、圆角等

2. **绘制消息区域** (列表)
   - 使用 "组件库" → "List" 添加消息列表
   - 或手动绘制若干消息气泡
   
   消息气泡设计:
   ```
   用户消息 (右对齐):
   ┌──────────────────┐
   │ 这是用户的消息   │  <- 浅蓝色背景
   └──────────────────┘
   
   AI消息 (左对齐):
   ┌──────────────────┐
   │ 这是AI的回复     │  <- 浅灰色背景
   └──────────────────┘
   ```

3. **绘制输入框和按钮**
   - 使用 "TextField" 组件添加输入框
   - 使用 "Button" 组件添加发送按钮

4. **标注重要信息**
   - 选择文字工具 (T) 添加注释
   - 标注尺寸、间距、颜色等

##### Step 3: 添加交互链接

**交互链接** 让屏幕间可以导航:

1. 选择一个元素 (如"发送"按钮)
2. 右侧属性面板 → "Link" 标签
3. 选择 "Go to Screen" → 选择目标屏幕
4. 设置交互类型: "On Click"

**常见交互类型**:
- On Click - 点击时
- On Hover - 悬浮时
- On Double Click - 双击时

**创建交互示例**:

```
屏幕1 (AI对话首屏)
  ↓ [输入文本+点击发送按钮]
屏幕2 (等待回复状态)
  ↓ [等待1秒]
屏幕3 (AI回复显示)
  ↓ [用户再次输入+发送]
屏幕1 (回到对话首屏)
```

##### Step 4: 标注关键信息

在原型中添加文字注释:

```
【爱问AI助手 - 对话界面】

关键交互:
1. 输入框: placeholder="问我任何问题..."
2. 发送按钮: 蓝色, 右下角固定
3. 消息区域: 可滚动
4. 配额显示: 顶部右侧 "剩余次数: 5/50"

动画:
- 消息进入: 从下往上 0.3s
- 发送按钮: 点击时快闪
```

##### Step 5: 创建多个屏幕

重复以上步骤为 AI 助手模块创建多个屏幕:

```
爱问AI助手模块的原型屏幕:
1. 会话列表 → 新建对话
2. 对话首屏 (输入状态)
3. 对话首屏 (等待状态) - 显示加载动画
4. 对话首屏 (接收回复) - 显示 AI 回复
5. 附件上传 - 文件选择界面
6. 对话详情 - 展开单个对话
7. 会话管理 - 删除/编辑对话
```

### Pencil 输出与验收

- 包含注释: ✅

#### 验收清单


## 🎬 Figma 高保真设计

### 什么是 Figma?

Figma 是云端协作设计工具，用于制作高保真设计稿和自动切图。

**核心特性**:
- ✅ 云端协作
- ✅ 实时多人编辑
- ✅ 自动切图和 CSS 生成
- ✅ 组件库管理
- ✅ 原型交互设计

### Figma 快速开始

#### 1. 创建账号

访问 https://www.figma.com，注册账号

#### 2. 创建项目

```
Dashboard → New file → 输入文件名
```

文件名规范: `ReliHub_MVP_UI_Design`

#### 3. 建立设计系统

##### Step 1: 创建颜色库

```
Assets 面板 → Color (+ 按钮) → 添加颜色

推荐颜色:
- 主色: #007AFF (Apple Blue)
- 成功: #34C759 (Apple Green)
- 警告: #FF9500 (Apple Orange)
- 错误: #FF3B30 (Apple Red)
- 中性色: #F2F2F7 (Light), #000000 (Dark)
```

##### Step 2: 创建排版样式

```
Assets 面板 → Typography → 创建文本样式

推荐样式:
- Display (34px, Bold)
- Headline (28px, Semibold)
- Body (17px, Regular)
- Callout (16px, Regular)
- Caption (13px, Regular)
```

##### Step 3: 创建组件库

```
为以下组件创建 Master Component:
- Button (3 states: default, hover, active)
- Input Field (3 states)
- Card (default, shadow, selected)
- Message Bubble (user, AI)
- Avatar (4 sizes)
- ...
```

#### 4. 设计流程 (以 AI 助手模块为例)

##### Frame 1: AI 对话首屏

```
1. 创建 Frame
   - 尺寸: 390 × 844 (iPhone 14)
   - 名称: "AI Chat - Conversation Screen"

2. 设计布局
   
   ┌─ Frame: Header (390 × 60)
   │  ├─ Text: "爱问" (Center)
   │  └─ Button: Close (Right)
   │
   ├─ Frame: Message List (390 × 724)
   │  ├─ Component: User Message Bubble
   │  ├─ Component: AI Message Bubble (with loading)
   │  ├─ Component: User Message Bubble
   │  └─ (scrollable area)
   │
   └─ Frame: Input Footer (390 × 60)
      ├─ Component: Input Field
      └─ Component: Button (Send)

3. 应用颜色和排版
   - Header 背景: 白色 / 深灰
   - 消息区域: 白色 / 深灰
   - 文字: 使用预定义的排版样式
   - 按钮: 使用主色 (#007AFF)

4. 添加细节
   - 圆角: 按钮 8px, 输入框 6px
   - 阴影: 按钮 (0, 2, 8, rgba(0,0,0,0.1))
   - 间距: 使用 8px 网格系统
```

##### Frame 2: 消息发送 - 等待状态

```
1. 复制 Frame 1 (右键 → Duplicate)
2. 名称: "AI Chat - Waiting State"
3. 修改:
   - 消息区域: 显示新消息 + 加载动画
   - 发送按钮: 改为禁用状态
```

##### Frame 3: AI 回复接收

```
1. 复制 Frame 2
2. 名称: "AI Chat - AI Response Received"
3. 修改:
   - 移除加载动画
   - 显示完整的 AI 回复
   - 恢复发送按钮为可用状态
```

## 🔄 从 Pencil 到 Figma

### 转换流程

```
Pencil (.pen 文件)
  ↓ Export PDF
  ↓
PDF 截图 / 参考
  ↓
在 Figma 中重新设计 (参考 Pencil 流程)
  ↓
Figma 高保真设计稿
```

### 最佳实践

1. **在 Pencil 中明确交互**
   - 标注清晰的屏幕流程
   - 明确用户操作
   - 标注关键信息

2. **在 Figma 中应用设计系统**
   - 使用预定义的颜色库
   - 使用排版样式
   - 使用组件库

3. **保持一致性**
   - 所有模块遵循同一设计语言
   - 间距、颜色、排版统一
   - 交互模式一致

#### Step 4: 编写交互 JavaScript

```javascript
// 获取 DOM 元素
const inputField = document.querySelector('.input-field');
const sendButton = document.querySelector('.send-button');
const messageList = document.querySelector('.message-list');

// 发送消息
sendButton.addEventListener('click', () => {
  const message = inputField.value.trim();
  
  if (!message) return;
  
  // 添加用户消息
  const userMsg = document.createElement('div');
  userMsg.className = 'message user-message';
  userMsg.textContent = message;
  messageList.appendChild(userMsg);
  
  // 清空输入框
  inputField.value = '';
  
  // 滚动到底部
  messageList.scrollTop = messageList.scrollHeight;
  
  // 模拟 AI 回复 (延迟 1 秒)
  setTimeout(() => {
    const aiMsg = document.createElement('div');
    aiMsg.className = 'message ai-message';
    aiMsg.textContent = 'AI 的自动回复（这是模拟）';
    messageList.appendChild(aiMsg);
    messageList.scrollTop = messageList.scrollHeight;
  }, 1000);
});

// 按 Enter 发送
inputField.addEventListener('keypress', (e) => {
  if (e.key === 'Enter') {
    sendButton.click();
  }
});
```

## ❓ 常见问题

### Pencil 相关

**Q1: Pencil 如何导出为可在浏览器中打开的文件?**

A: Pencil 可以导出为 PDF 或 PNG，但不是完全可交互的。如果需要可交互的原型，建议导出为 PDF 后截图参考，或直接在 Figma 中重新设计。

**Q2: 如何在 Pencil 中快速创建多个相似的屏幕?**

A: 
1. 右键 Frame → Duplicate
2. 修改内容
3. 添加交互链接

**Q3: Pencil 的团队协作怎么做?**

A: Pencil 可以导出为 .pen 文件，通过 Git/GitHub 进行版本控制和协作。

### Figma 相关

**Q4: Figma 如何管理多个团队成员的协作?**

A:
1. 创建 Team Workspace
2. 邀请成员加入
3. 实时协作编辑
4. 版本历史自动保存

**Q5: 如何在 Figma 中创建组件库并供其他文件使用?**

A:
1. 创建组件 (右键 → Create component)
2. 右上角 Assets → 发布组件库
3. 其他文件中导入使用

**Q6: Figma 导出的 CSS 代码可以直接用吗?**

A: 可以参考，但通常需要根据项目架构进行调整 (如使用 Tailwind CSS、CSS Modules 等)。

### 工作流相关

**Q7: 如果在 Figma 中发现设计有问题，需要回到 Pencil 修改吗?**

A: 不需要。Figma 是独立的设计工具，可以直接在 Figma 中修改。Pencil 仅作为快速原型参考。

**Q8: 如何确保 HTML 原型与 Figma 设计稿完全一致?**

A: 
1. 使用 Figma Inspect 工具获取准确的尺寸、颜色、排版
2. 使用量尺工具验证
3. 与 Figma 设计稿进行像素对比
4. 使用自动化工具 (如 Percy) 进行 UI 对比

**Q9: 如何处理 Figma 设计到 React 代码的转换?**

A:
1. HTML 原型是中间桥梁
2. 将 HTML 代码转换为 React 组件
3. 保持样式和交互逻辑一致
4. 优化性能和可维护性
