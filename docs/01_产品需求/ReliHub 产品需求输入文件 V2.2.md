# 电子产品可靠性AI社区（ReliHub）产品需求输入文件

文档版本：V2.2  
适用范围：ReliHub Web端（模拟微信小程序）全模块设计、开发、测试、部署  

技术栈约束：前端 React+Vite+TypeScript；后端 Python FastAPI；主数据库 PostgreSQL  

产品形态：Phase1 为 Web 应用（适配 iPhone12Pro 尺寸，模拟微信小程序交互），Phase2 迁移为原生微信小程序

---

## 一、产品概述

### 1.1 产品定位

ReliHub 是面向电子产品研发、测试、质量、工艺、失效分析等领域技术人员的可靠性专业 AI 社区微信小程序（Phase1 Web 模拟），集 AI 智能问答、专业资源共享、技术社区交流、专家专项咨询于一体，打造“我为人人、人人为我”的专业技术生态，解决行业人员可靠性相关问题检索、交流、咨询的高效性需求。

“ReliBot”作为 AI 助手模块的内部代号或 AI 角色名。

### 1.2 核心目标

1. 汇集电子产品可靠性领域全链路技术人员与专家，搭建专业交流与资源共享平台；
2. 以 AI 助手为核心能力，为用户提供可靠性问题的建设性解答与资料解读服务；
3. 实现可靠性相关标准、手册、工具、案例等资源的规范化检索与共享；
4. 支持领域内技术话题讨论与专家一对一咨询，推动技术落地与合作。

### 1.3 核心用户

- 核心用户：电路设计工程师、质量工程师、测试工程师、生产工艺工程师、失效分析工程师；
- 专家用户：电子产品可靠性领域资深专家、行业大咖；
- 运营用户：超级管理员（负责后台配置、权限管理、系统管理）和管理员（负责日常运营、内容审核）。

## 二、产品设计总则

### 2.1 外观风格要求

1. 尺寸基准：以 iPhone12Pro 屏幕尺寸为设计基准，页面所显示内容的区块、图框、卡片、图标、按键、字符高宽比适配协调；
2. 视觉调性：科技感、高级感，配色体现可靠性与专业性，全部页面风格统一；
3. 细节要求：页面、弹窗、按键、图标、卡片、字体字号、配色美观精致、简洁统一；
4. 交互适配：符合微信小程序的样式、界面风格及手机端操作习惯；
5. 开发形态：Phase 1 Web 方式开发，用户端页面带手机边框，模拟微信小程序显示效果；

> 管理后台为独立 PC 端 Web 管理系统。

### 2.2 核心设计原则

1. 逻辑清晰，操作顺畅：功能模块划分明确，交互路径不超过 5 级子页面，无冗余操作；
2. 权限分级，体验分层：区分游客用户、注册用户、不同等级用户、专家、管理员权限，匹配对应功能与体验；
3. 模块联动（phase2 实现），生态闭环：实现 AI、资源、社区三大核心模块的深度联动，避免孤立；
4. 价值驱动，用户增长：通过可可豆、等级体系激励用户贡献，通过注册福利、广告可可豆等降低用户参与门槛；
5. 安全合规，稳定可靠：保障数据安全、内容安全，防止爬虫、刷可可豆等行为，符合微信内容安全规范。
6. AI 回答超时或资源访问超时以及其它异常情况，系统应用提示错误原因或提醒用户重试（提升用户体验）

### 2.3 核心名词定义（术语表）

| **术语**         | **定义**                                                                                                   |
| ---------------- | ---------------------------------------------------------------------------------------------------------- |
| 可可豆           | 平台虚拟币（可可豆为代币单位），用于资源下载、AI问答额度扩充、悬赏、超赞、专家一对一咨询等。可通过充值(Phase 2)、注册、签到、广告点击、邀请好友、贡献资源、贡献的资源被下载、悬赏话题回答被采纳等途径获得；Phase 2阶段可区分「金可可」(资产豆,可取现)和「福利豆」(赠送豆,不可取现)，超级管理员可配置取现功能开关 |
| 信誉分           | 评估用户平台行为的核心指标，与用户等级强关联，优质行为（贡献资源、热门话题、优质回答、社区活跃度、点赞、点评）加分，长期不登录或不当行为（违规言论、重复上传）减分；**信誉分仅用户本人可见**，其他用户查看资源或话题时仅可看到发布者的等级徽章 |

| RAG专业知识库    | 平台构建的可靠性领域专属知识库，将文档按章节切片入库，为AI助手提供专业回答支撑                                 |
| 种子用户         | 平台启动初期，管理员通过后台配置初始资料与功能模块后，首批入驻使用的用户                                       |

## 三、核心功能与模块需求

### 3.1 底部导航核心模块

小程序底部固定 4 个导航按键：爱问（AI 助手）、资源（资源库）、社区（技术社区）、我的（个人中心），所有页面底部导航样式、功能完全统一，为用户核心操作入口。

### 3.2 启动与游客模式（M0 核心，MVP 阶段必做）

#### 3.2.1 欢迎页

1. 触发规则：仅首次安装/长期未使用时展示，双击/上划可跳过；首次使用展示 5 秒完整动画，7 天内复登直接跳过，超 7 天未使用展示 3 秒品牌 logo 动画；
2. 内容设计：包含品牌 Logo、动态动画，随机轮换可靠性领域名言（如“可靠性是设计出来的，不是测试出来的”）；
3. 跳转逻辑：欢迎页结束后直接进入游客模式，首屏为“爱问”主页初始状态。

#### 3.2.2 游客模式

1. 身份分配：系统自动为游客分配虚拟 ID，保留 AI 问答内容至退出，退出前系统提示游客注册以保留交互信息。如游客坚持退出则不保留 AI 问答内容。
2. 功能权限：
   - AI 问答：每日可发起 3 个新会话（单个会话轮次上限根据用户等级权益额度进行配置，每日问答总轮次上限根据用户等级权益额度进行配置，详见「PRD_可可豆与信誉分体系.md」）（管理后台可配置（管理员需超级管理员授权）），单对话支持多轮问答，仅支持文字输入；
   - 资源浏览：仅可划动查看 10 条热门推荐资源卡片及资源详情，无[下载、收藏、点赞、点踩、点评]等权限（图标灰色）；
   - 社区浏览：仅可划动查看 10 条热门话题卡片，无[回复、点赞、点踩、点评、悬赏、关注、举报]等权限（图标灰色）；
   - 通用限制：仅可浏览公开页面，无法进行任何触发数据库变更的操作（上传、下载、编辑、删除、收藏、关注、点赞、评论等功能图标为灰色不可用），无法进入“我的”页面；
3. 注册引导：当游客触发超限制操作（AI 问答超量、点击下载/评论/"我的"等）时，弹出价值驱动型注册引导弹窗，内容包含：✅注册即送 30 可可豆、✅基础资源/权益额度内免费下载（管理员可设置下载资源数量）、✅每日 15+轮次 AI 专业问答、✅无限浏览、✅参与社区讨论与专家交流；
   弹窗提供【微信一键注册】（主选项）、【手机号注册】（备选）、【随便看看】（返回当前页面）三个操作按钮。
4. 早鸟用户奖励机制：在产品冷启动初期（缺少用户时），游客用户浏览期间进行触发注册引导弹窗，醒目增加提示内容“前 200 名注册用户额外加送 20 可可豆”。

### 3.3 登录/注册模块（M0 核心，MVP 阶段必做）

#### 3.3.1 页面设计

登录/注册为同一页面，页面底部需展示用户协议、隐私政策勾选框（必选），未勾选无法进行登录/注册操作。

#### 3.3.2 登录/注册方式

1. 微信一键登录（主方式）：通过微信小程序授权获取用户基础信息，新用户自动完成注册，老用户直接登录；
2. 手机号登录/注册（备选）：支持密码登录、验证码登录两种方式，系统自动判断新老用户：
   - 新用户：登录流程默认为注册，验证码登录后可选择是否设置密码，密码登录需先设置密码；
   - 老用户：直接验证密码/验证码登录；
3. 密码规则：长度 8-20 位，至少包含大小写字母、数字、特殊符号（!@#$%^&*等）中的两种；
4. 忘记密码：支持通过手机号验证码重置登录密码；
5. 安全机制：连续 5 次输入错误密码，触发账号锁定 15 分钟（管理后台可配置锁定次数和锁定时长（管理员需超级管理员授权））。

#### 3.3.3 注册后流程

新用户完成注册后，直接进入**个人信息填写页面**，信息填写按**用户等级**区分**必填项 / 选填项**（管理后台可配置（管理员需超级管理员授权）），填写完成后进入平台首屏（爱问主页）。

### 3.4 爱问（AI 助手）模块（M1 核心，MVP 阶段必做）

#### 3.4.1 爱问主页

1. **页面布局**：
   1. 上部（1/4 显示区域）：系统自我介绍 / 管理员发布内容 / 广告栏（后期可切换）；
   2. 中部：历史对话卡片列表，按**时间倒序**排列（最新靠前），卡片包含对话标题、发起日期时间；无历史对话时展示 AI 助手问候语 + 随机预设的可靠性相关问题（可点击直接提问）；
   3. 下部：AI 对话输入框 + 上传附件功能图标；phase3 加入语音输入功能。
2. **核心操作**：支持历史对话卡片的**检索**（按日期 / 关键字）、**单条删除**、**批量删除**、**重命名标题**、**点击继续对话**。

#### 3.4.2 AI 对话功能

1. **输入方式**：MVP 阶段仅支持**文字输入**，后期迭代**语音输入**；登录用户可**上传附件**（游客仅能文字输入），附件类型包含拍摄图片、相册图片、本地文件、微信文件，1 次上传不超过 3 个（管理后台可配置数量 / 文件大小（管理员需超级管理员授权）），支持的格式由管理后台统一设置（管理员需超级管理员授权）；
2. **AI 能力边界**：
   - 可回答**可靠性相关专业问题**，对无关问题 / 敏感问题礼貌回绝；
   - 可解读用户上传的附件，给出分析意见与建设性建议；
   - 可发现用户输入的不当言论 / 敏感词，给予提醒并婉拒回答；
3. **对话规则**（管理后台可按用户等级配置（管理员需超级管理员授权））：
   - 单个对话轮次限制：根据用户等级权益额度进行配置，详见「PRD_可可豆与信誉分体系.md」；
   - 历史对话保存数量：根据用户等级权益额度进行配置，详见「PRD_可可豆与信誉分体系.md」，**游客的历史对话基于虚拟ID存储，退出后自动清空**；接近超量时提示用户手动删除，用户未处理则自动覆盖最早的对话；
   - Token / 存储限制：限制单个对话、总历史对话的输入输出 Token 及存储文档大小，超量可通过可可豆兑换扩充额度与有效期；
   - 对话超限预警：当对话接近限额，系统根据超限具体情况，给予用户相应的提示和建议。
4. **对话标题**：系统根据**单轮新对话的首次问题主旨**自动生成，并实时保存为历史对话。

#### 3.4.3 对话详情页

1. **核心操作**：支持对对话内容**按轮次**进行点赞、点踩、分享、删除、全文复制 / 选取文字复制；
2. **快捷功能**：页面提供**新对话**图标，点击后先自动保存当前对话后直接返回爱问主页；
3. **模块联动（Phase 2 实现）**：AI 回答中自动推荐**相关资源链接**、**相关社区话题链接**，支持一键跳转。AI 推荐资源/话题后，用户进行任何操作后点“爱问”，返回 AI 当前对话详情页继续对话。管理员可以设置启动 AI 助手参与社区话题回复。

### 3.5 资源模块（M2 核心，MVP 阶段必做）

#### 3.5.1 资源主页

1. **页面布局**：资源搜索框 + 资源分类卡片 + 热门推荐资源列表 + **分享资源**按键图标；
2. **资源分类**：默认资源主分类包含标准规范、器件手册、应用笔记、工具模版、案例分享、培训资料、推荐书单、综合杂谈（管理后台可自定义增删分类、隐藏部分分类、修改分类名称、调整在用户端页面上的分类卡片显示顺序（管理员需超级管理员授权）），点击分类卡片跳转至对应分类详情页；空分类卡片不隐藏，进入后显示空状态 + 上传引导（“这个分类还没有资源，您将成为第一个贡献者，获得 30 可可豆 奖励！”）
3. **分类卡片排序**：用户可以通过拖拽调整资源分类卡片的排序（phase3 实现）
4. **核心操作**：支持资源**关键字检索**、**分类筛选**、**热门推荐浏览**。

#### 3.5.2 资源上传（贡献）

1. **上传流程**：用户点击"分享资源"→上传文件→填写资源信息（标题、分类、简介、最多 5 个关键词、下载所需的可可豆（定价区间默认下限5，上限100000，由管理员配置））→提交→系统 AI 预审（**查重相似度 + 关键词匹配 + 自动摘要预览 + 分类建议**）并给出预审意见报告→提交管理员**人工审核**（48 小时内）→审核结果反馈至用户；
2. **审核异议**：用户对被拒绝的资源的审核结果反馈有异议，可发起**申诉沟通留言**（仅管理员和本人可见），管理员回复申诉意见，再行审核。
3. **待审核操作**：用户可在“我的资源”中对**待审核**资源进行**修改**、**撤回**、**重新提交**；
4. **文件规则**：支持的上传资源的文档格式、大小由管理后台统一设置（管理员需超级管理员授权）（Phase 1 支持 txt、doc、pdf、excel、PPT、PNG、JPG、BMP 等）。
5. **AI 协助**（phase3 实现）：用户上传资源后，AI 预读后自动填写资源信息标题（30 个字）、简介（150 个字）、关键词，用户修改后或默认即可提交。

#### 3.5.3 资源消费

1. **核心操作**：用户可对资源进行**查看**、**预览**（预览量由管理员在审核资源时配置，每个资源可单独设置预览页面数）。登录用户可以对资源进行**收藏**、**下载**、**点评、点赞、点踩**；
2. **下载规则**：基础资源在用户权益额度内下载不消耗可可豆，超出权益额度或下载非基础资源需按定价消耗可可豆（贡献者获得70%，平台销毁30%采用通缩模型）；首次下载后1年内可免费重下载2次，超过1年后再次下载视为新下载，按资源下载规则处理。
3. **下载扩充**：用户超出权益额度后，可选择购买下载扩充包享受折扣优惠：
   - 下载特惠包：80可可豆，永久有效，可下载总大小不超过100M的资源，每次下载打85折，剩余容量有零头可多下载一次
   - 下载畅享包：350可可豆，永久有效，可下载总大小不超过500M的资源，每次下载打7折，剩余容量有零头可多下载一次
   - 折扣后的可可豆按贡献者70%、平台30%分配
4. **预览安全**：资源预览页添加**用户水印**（昵称+用户ID+时间戳），防止截图 / 录屏截取。

#### 3.5.4 资源状态管理

用户上传的资源在“我的资源”中列表展示，通过**状态标签**区分：待审核、已通过、被拒绝、申诉中；**已通过**资源仅支持修改下载所需可可豆数量（用户可调整、上线超过6个月后管理员也可在管理员定价区间内调整），不可撤回。待审核 和 申诉中的 资源可以被提交的用户修改、撤回、重新提交，资源的状态相应发生变化。

已通过审核的资源将进入分类“资源列表”页中（再按最新（7天）、精华、基础资源、闲置 进行二级分类，一个资源可能有多个分类标签），供所有用户（包括贡献者本人）进行检索（资源列表页有搜索图标）、浏览、查看详情，下载、收藏。

#### 3.5.5 模块联动（Phase 2 实现）

1. 资源详情页设置**“用 AI 解读此文档”**按钮，点击跳转至 AI 对话页，AI 自动解读该资源；
2. 社区话题讨论中，支持**引用资源库中的文档**，一键插入话题回复 / 评论。

## 四、补充核心模块需求（MVP 阶段必做基础，细节迭代完善）

### 4.1 AI 助手领域定义与行为规范（核心能力，M1 配套）

#### 4.1.1 身份定义

爱问(ReliBot)是ReliHub平台的**电子产品可靠性领域AI助手**，定位为"可靠性领域专业技术顾问"，具备该领域的标准、手册、案例、工艺、失效分析等专业知识，辅助用户解决各类可靠性相关问题。

#### 4.1.2 知识边界

**可回答范围**：电子产品可靠性设计、信号测试、环境实验、质量管控、生产工艺、失效分析、元器件应用、标准解读、工具使用等相关专业问题，及平台功能使用问题；**不可回答范围**：与电子产品可靠性无关的问题、敏感政治 / 违法 / 色情 / 暴力等内容、平台规则外的可可豆 / 等级作弊方法、专家一对一咨询的深度定制化问题（Phase 3 可引导对接专家）。

#### 4.1.3 语言风格

1. **专业性**：回答贴合可靠性领域专业知识，使用行业标准术语，避免口语化；
2. **建设性**：不仅给出答案，还需提供分析思路、解决方案、参考建议；
3. **简洁性**：逻辑清晰，分点阐述，避免冗余内容；
4. **礼貌性**：对无法回答的问题，礼貌回绝并说明原因，不生硬拒绝。

#### 4.1.4 行为规则

1. 匹配基于平台 RAG 专业知识库回答，非专业知识不随意作答；无关问题婉拒回答。
2. 解读用户附件时，先确认附件内容与可靠性相关，再给出针对性分析；
3. 检测到敏感词 / 不当言论时，立即提醒用户并进行过滤，无法判断时标记并知会管理员；
4. 模块联动（Phase 2）时，推荐的资源 / 话题需与用户问题**高度相关**，避免无关推荐；
5. 流式输出回答内容，提升用户体验（适配 FastAPI 异步 SSE）。

### 4.2 通知与消息系统（用户黏性核心，全模块配套）

#### 4.2.1 页面设计：消息通知页（“我的” 模块入口）

设置 **3 个分类 Tab**，标签切换式展示，支持单条 / 批量删除消息：

1. **系统通知**：平台规则更新、功能升级、账号安全提醒、管理员处罚 / 解封通知等；
2. **互动通知**：资源被下载 / 点评、话题被回复 / 评论 / 点赞、回答被采纳为最佳、被 @/ 被关注等；
3. **审核通知**：资源上传 / 申诉的审核结果、专家认证审核结果、话题违规审核结果等。

#### 4.2.2 触发规则与通知通道

管理后台可配置**通知通道开启 / 关闭**（管理员需超级管理员授权），用户可在“我的 - 设置 - 通知设置”中自定义接收范围：

| **通知类型** | **核心触发事件**                         | **默认通知通道**                     |
| ------------ | ---------------------------------------- | ------------------------------------ |
| 系统通知     | 规则更新、账号锁定 / 解封、安全异常       | 小程序内弹窗 + 消息列表               |
| 互动通知     | 资源被点评、回答被采纳、被 @              | 小程序内消息列表                     |
| 审核通知     | 资源审核通过 / 拒绝、申诉回复、专家认证结果 | 小程序内弹窗 + 消息列表               |

#### 4.2.3 通知时效与留存

1. **弹窗通知**：仅在用户在线时展示，3 秒后自动消失，可点击跳转至对应页面；
2. **消息列表**：消息永久留存，用户可手动删除，未读消息标红提示。

### 4.3 统一搜索系统（全模块配套，MVP 阶段必做）

#### 4.3.1 搜索入口与范围

平台设置 **3 个搜索入口**，各入口搜索范围**专属化**，避免跨模块无关结果：

1. **爱问搜索**：入口在爱问主页，搜索范围为**用户历史 AI 对话内容**；
2. **资源搜索**：入口在资源主页 / 分类页，搜索范围为**资源库所有资源（标题 / 简介 / 关键词）**；
3. **社区搜索**：入口在社区主页 / 分类页，搜索范围为**社区所有话题（标题 / 内容 / 关键词）**。

#### 4.3.2 搜索算法与排序（Phase 1）

1. **算法**：基于 PostgreSQL 全文检索，支持**关键字精准匹配**、**模糊匹配**，支持多关键字组合搜索（空格分隔）；
2. **排序规则**：
   - 爱问搜索：按**对话发起时间倒序**排列；
   - 资源搜索：按**相关性（关键词匹配度）** 优先，其次按**热度排序**（详见《PRD_MVP_资源模块.md》§4.2）；
   - 社区搜索：按**相关性（关键词匹配度）** 优先，其次按**讨论热度 / 发布时间**排序（详见《PRD_MVP_社区模块.md》）；**Phase 2**：升级为 Elasticsearch，支持高级搜索（如按发起用户 / 发布时间 / 分类筛选）。

#### 4.3.3 搜索体验优化

1. 输入关键字时，提供**联想词提示**（基于平台高频搜索词 / 专业术语）；
2. 无搜索结果时，给出**相关推荐**（如相近关键词、相关资源 / 话题）；
3. 支持搜索结果的**二次筛选**（如资源按分类 / 下载量筛选，话题按分类 / 热度筛选）。

### 4.4 内容生命周期管理（平台运营核心，全模块配套）

#### 4.4.1 资源生命周期

1. **上线**：用户上传→系统查重→管理员审核通过→纳入资源库，标记**“最新”**标签（7 天）；
2. **存续**：按下载量 / 点评量维持在资源库，管理员可标记**“精华”**标签；
3. **老化**：超过 1 年无下载 / 无点评的资源，标记**“闲置”**标签，搜索结果靠后；
4. **退出**：
   - 主动退出：资源贡献者可申请删除已通过的资源，管理员审核后删除；
   - 被动退出：闲置资源超过 2 年无交互、资源内容存在错误 / 违规、被多个用户举报且审核确认违规，管理员可以直接（选择或批量）删除，通知贡献者（给出删除原因）。

#### 4.4.2 社区话题生命周期

1. **上线**：用户发起→系统审核（敏感词过滤）→发布至社区，标记**“新话题”**标签（3 天）；
2. **存续**：按讨论热度维持在话题列表，热度高的话题进入热门列表，管理员可标记**“精华”**标签；
3. **老化**：超过 60 天无新回复 / 评论的话题，标记**“沉寂”**标签，搜索结果靠后；
4. **退出**：
   - 主动退出：话题发起人可删除自己的话题（无悬赏 / 已结悬赏）；
   - 被动退出：沉寂话题超过 1 年无交互、话题内容违规、被多个用户举报且审核确认违规，管理员可以直接（选择或批量）删除，通知发起人（给出删除原因）。

#### 4.4.3 用户账号生命周期

1. **注册**：完成微信 / 手机号注册，分配**新兵**等级，初始信誉分（管理后台配置（管理员需超级管理员授权））；
2. **存续**：按用户行为积累 / 扣减信誉分，升级 / 降级，正常使用平台功能；
3. **休眠**：超过 6 个月未登录的账号，标记**“休眠账号”**，限制 AI 问答 / 资源下载权限，登录后恢复；
4. **注销**：
   - 主动注销：用户在“我的 - 设置 - 账号及安全”中申请注销，清理个人信息（保留贡献的资源 / 话题）；
   - 被动注销：休眠账号超过 2 年未登录、账号严重违规被封号、刷可可豆等恶意行为被查处，管理员注销账号，通知用户（给出注销原因）。

### 4.5 文件与附件处理规范（全模块配套，MVP 阶段必做）

统一规范 **AI 对话附件**、**资源上传文件** 两个场景的文件处理规则，管理后台可按用户等级配置差异化限制（管理员需超级管理员授权），**所有文件均存储在云对象存储（阿里云 OSS / 腾讯云 COS）**：

#### 4.5.1 支持的文件格式与大小限制（默认值）

| **文件类型** | **支持格式**                                      | **单文件最大大小（默认）** | **适用场景**           |
| ------------ | ------------------------------------------------- | -------------------------- | ---------------------- |
| 文档类       | txt、doc、docx、pdf、xls、xlsx、ppt、pptx         | 20MB                       | AI 附件 / 资源上传     |
| 图片类       | PNG、JPG、JPEG、BMP、TIF、GIF                      | 5MB                        | AI 附件 / 资源上传     |

> **规划说明**：社区话题附件上传（图片/文件）不在 MVP 范围内，归入 Phase 2/3 阶段实现。Phase 2/3 规划时需在本文档4.5节补充社区话题附件的场景定义及对应规则约束。

#### 4.5.2 文件预览方式

1. **文档类**：Web 端使用 pdf.js 实现预览，仅支持前 N 页（N 由管理后台配置（管理员需超级管理员授权），即预览量）；小程序端（Phase 2）使用微信文档预览 API；
2. **图片类**：直接在线预览，支持缩放，超额下载资源 / AI 解读的图片添加用户水印；
3. **预览限制**：未登录用户仅可预览资源库图片的缩略图，无法预览文档。

#### 4.5.3 文件存储周期

1. **AI 对话附件**：随历史对话保存，历史对话被删除则附件同步删除；保存的历史对话超过存储容量限制后按先进先删原则被覆盖（即超额后新对话覆盖最早的对话）；
2. **资源上传文件**：资源审核通过后永久存储，资源被删除则文件同步删除；待审核 / 被拒绝的资源文件，保留 30 天自动清理；
3. **临时文件**：用户上传后未提交的文件，保留 24 小时自动清理。

#### 4.5.4 文件安全

1. 所有文件上传前经过**病毒扫描**，防止恶意文件上传；
2. AI 问答/资源上传时检测加密文件并提示；
3. 审核通过的资源文件被预览时添加明文水印、被下载时添加**用户数字盲水印**（用户名 + 时间），防止二次传播；
4. 资源库文件设置**反下载保护**，禁止直接右键下载，仅支持平台内正规下载渠道（消耗可可豆）。
5. 资源（文件）上传时提示：该资源不涉及版权保护或您拥有合法分享权。用户勾选 ☑️：我已确认以上内容，并承诺不侵犯他人知识产权（此功能 phase 3 实现）。
6. 对上传的资源进行标准版本标注。标准规范类资源支持版本关联和现行/已废止（仅供历史参考）状态标注（此功能 phase 3 实现）。

### 4.6 性能与体验指标要求（开发测试核心，全模块配套）

所有指标为**最低要求**，开发与测试阶段必须满足，管理员/管理后台可监控性能数据：

| **指标类型** | **具体指标**                       | **要求值**   |
| ------------ | ---------------------------------- | ------------ |
| 页面加载     | 首屏加载时间（所有主页面）         | ≤2 秒        |
|              | 子页面加载时间                     | ≤1 秒        |
|              | 图片 / 文件预览加载时间            | ≤3 秒        |
| AI 对话      | 文字输入后 AI 响应时间             | ≤1 秒        |
|              | 附件上传后 AI 解读响应时间         | ≤5 秒        |
|              | AI 流式输出速度                    | ≥5 字 / 秒   |
| 搜索功能     | 关键字搜索结果返回时间             | ≤1 秒        |
| 文件操作     | 单文件上传速度（20MB 内）          | ≤10 秒       |
|              | 单文件下载速度（20MB 内）          | ≤10 秒       |
| 系统稳定性   | 平台日可用率                       | ≥99.9%       |
|              | 并发用户数支持（Phase 1）          | ≥500 人      |
| 接口性能     | 所有接口响应时间                   | ≤500ms       |

### 4.7 外部分享与传播机制（用户增长核心，Phase 1 基础，Phase 2 完善）

#### 4.7.1 分享入口与内容

支持 **3 类内容分享**，分享入口在对应内容详情页（AI 对话轮次、资源详情、话题详情），采用**微信生态为主**的分享方式：

1. **AI 对话分享**：分享内容为**对话标题 + 核心回答片段 + 平台小程序链接**，支持分享至微信好友 / 微信群；
2. **资源分享**：分享内容为**资源标题 + 简介 + 平台小程序链接**，支持分享至微信好友 / 微信群；
3. **社区话题分享**：分享内容为**话题标题 + 核心内容 + 平台小程序链接**，支持分享至微信好友 / 微信群。

#### 4.7.2 分享落地页

非平台用户点击分享链接后，进入**平台落地页**（微信小程序中转页），展示分享内容的**部分预览**，引导用户注册 / 登录后查看完整内容，落地页包含平台核心优势与注册入口。

#### 4.7.3 分享激励（Phase 2 实现）

用户分享的内容被新用户点击并注册后，分享者可获得**可可豆奖励**（采用“邀请奖励挂账机制”：被邀请人完成首次有效操作后，双方各获 10 可可豆。此前处于挂账状态（管理后台配置奖励数量））

### 4.8 用户反馈与意见收集渠道（产品迭代核心，MVP 阶段必做）

#### 4.8.1 反馈入口

在 **“我的 - 设置 - 意见反馈”** 中设置专属反馈入口，为独立反馈页面，操作简单。

#### 4.8.2 反馈形式

支持 **两种反馈方式**，满足不同用户需求：

1. **表单反馈**：预设反馈类型（功能建议、bug 反馈、内容违规、其他问题），用户选择类型 + 填写反馈内容 + 可上传图片（截图 / 证据），提交即可；
2. **联系方式**：展示平台官方客服微信 / 邮箱，支持用户线下反馈复杂问题。

#### 4.8.3 处理流程

1. **提交**：用户提交反馈后，系统自动生成**反馈单号**，并提示“反馈已收到，我们将在 1-3 个工作日内回复”；
2. **审核**：管理员在后台查看反馈列表，按类型分类处理，标记**“待处理 / 处理中 / 已解决 / 已驳回”**；
3. **回复**：处理结果通过**系统通知**反馈至用户，包含处理意见、解决方法（如 bug 修复时间、功能建议采纳情况）；
4. **跟踪**：用户可在反馈页面通过反馈单号查看处理进度，对已解决的反馈可进行**满意度评价**（满意 / 一般 / 不满意）。

#### 4.8.4 反馈激励

（Phase 2 实现）用户的反馈被采纳（如 bug 被修复、功能建议被开发），可获得**可可豆奖励**（管理后台配置（管理员需超级管理员授权）），奖励通过系统通知发放至用户账号。

## 五、技术与开发要求

### 5.1 技术栈选型（强制要求，无替代方案）

| **技术模块** | **选型**                                     | **补充说明**                                                 |
| ------------ | -------------------------------------------- | ------------------------------------------------------------ |
| 前端         | React + Vite + TypeScript                    | 组件化开发，状态管理采用 Zustand/Redux Toolkit，适配手机端交互 |
| 后端         | Python FastAPI                               | 异步高性能，适配 AI 流式输出（SSE），所有端点异步实现        |
| 主数据库     | PostgreSQL                                   | 强制使用，依赖 JSONB、GIN 索引等特有特性，禁止 SQLite 等替代 |
| 缓存         | Redis                                        | 会话管理、接口限流（令牌桶）、热点数据、签到状态、防刷可可豆   |
| AI 接入      | 第三方 LLM API（DeepSeek/GPT-4）+ RAG 专业知识库 | 文档按章节切片入库，提升 AI 回答专业度与检索精度             |
| 文件存储     | 云对象存储（腾讯云 COS / 阿里云 OSS）        | 保障文件存储安全性与访问速度                                 |
| 文件预览     | pdf.js（Web 端）/ 微信文档预览 API（小程序端） | 资源预览加水印（浅色，不影响阅读），防止截取                 |
| 搜索         | Phase 1：PostgreSQL 全文检索；Phase 2：Elasticsearch | 统一搜索系统，支持精准 / 模糊匹配                            |
| 实时通信     | WebSocket                                    | 用于 Phase 3 专家一对一聊天，离线消息转邮件 / 短信           |
| 代码仓库     | relihub-web（前端）/ relihub-api（后端）     | 前后端代码分离，独立仓库管理                                 |
| 开发环境     | Docker + Docker Compose                       | 所有服务容器化，docker-compose up 一键启动，保障团队环境一致性 |
| 数据库迁移   | Alembic + SQLAlchemy ORM                      | 表结构变更通过迁移文件管理，容器启动时自动执行 alembic upgrade head，禁止手动修改生产 schema |

### 5.2 数据库环境一致性要求（强制要求）

1. **生产环境**：使用 PostgreSQL，配置独立数据库实例，开启数据备份（每日自动备份）；
2. **开发环境**：与生产环境完全一致，使用 PostgreSQL，通过 Docker Compose 配置，禁止使用 SQLite；
3. **测试环境**：使用独立的 PostgreSQL 测试数据库（relihub_test），所有测试用例基于该数据库执行，禁止污染开发 / 生产数据库。

### 5.3 自动化测试规范（强制要求，适用于所有模块）

1. **测试框架**：使用 pytest + pytest-asyncio + httpx.AsyncClient（ASGITransport 包装 FastAPI app），**所有测试为异步实现**，禁止使用同步 TestClient；
2. **测试数据库隔离**：测试用例基于 relihub_test 数据库执行，每个用例结束后通过 TRUNCATE ... CASCADE 清理数据，确保数据隔离；
3. **测试覆盖维度**：所有模块、所有接口必须覆盖**快乐路径**、**异常路径**、**边界条件**三个维度，测试覆盖率 ≥80%；
4. **测试结果**：测试不通过的模块 / 接口，禁止进入上线流程。

### 5.4 安全要求（核心要求，全模块配套）

1. **账号安全**：支持微信 / 手机号双重验证，密码加密存储，连续输错密码锁定账号，异常登录（异地 / 异设备）系统通知；
2. **支付安全（Phase 2）**：对接微信 / 支付宝支付，采用手机 + 微信 / 支付宝双重绑定验证，支付流程加密，记录支付日志；
3. **内容安全**：对接微信内容安全 API（文本 + 图片），配置敏感词库，AI 助手辅助过滤不当内容，建立用户举报机制；
4. **数据安全**：用户个人信息加密存储，禁止泄露，资源库文件防爬虫、防非法下载，平台数据每日自动备份；
5. **接口安全**：所有接口添加鉴权，防止非法调用，开启接口限流（令牌桶算法），防止接口轰炸；
6. **防刷机制**：防止刷可可豆、刷点赞、刷下载，通过 Redis 记录用户操作行为，对高频异常操作进行限制 / 警告。

### 5.5 测试账号配置（上线前必做）

正式发布上线前，系统**自动分配测试账号**，用于全流程模拟测试，包含：

1. **虚拟用户账号**：10 个，包含手机号 + 密码，覆盖不同用户等级（新兵、菜鸟、入门、熟手、老炮、达人、专家）；
2. **管理员账号**：1 个，包含管理员账号 + 密码，用于内容审核与日常管理；
3. **超级管理员账号**：1 个，包含超级账号 + 超级密码，用于后台配置与测试。

## 六、开发阶段与优先级划分

### 6.1 阶段划分

| **阶段** | **产品形态**                         | **核心开发内容**                                                                                             | **交付标准**                                                                                         |
| -------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------ |
| Phase 1（MVP） | Web 应用（模拟微信小程序，iPhone12Pro 尺寸） | 完成 M0-M5 核心模块基础功能，实现 AI、资源、社区基础功能，完成搜索、文件处理、性能指标达标                      | 游客 / 注册用户可正常使用 AI 问答、资源浏览 / 上传 / 下载、社区话题发起 / 参与、个人中心基础功能；管理员/管理后台可完成基础配置与审核；所有性能指标达标 |
| Phase 2  | 原生微信小程序                       | 实现 AI / 资源 / 社区深度联动，将 Web 应用迁移为原生微信小程序，适配微信生态所有功能，优化手机端交互体验，对接微信支付 / 微信分享 / 微信通知 / 充值功能。实现分享激励、用户反馈激励 | 原生微信小程序上线，功能与 Web 端完全同步，交互体验符合微信小程序官方规范 |
| Phase 3  | 应用迭代优化                         | 实现上线专家认证与一对一咨询，广告管理                                                                       | 完成全模块生态闭环，支持专家与用户的商业合作，实现用户增长与激励体系的完整落地 |

### 6.2 模块开发优先级（MVP 阶段）

1. **S0（最高优先级，核心必做）**：启动与游客模式、登录 / 注册模块、爱问（AI 助手）核心功能、管理员/管理后台（基础配置 / 内容审核）；
2. **S1（高优先级，核心必做）**：资源模块核心功能（上传 / 下载 / 收藏）、社区模块基础功能（话题发起 / 回复）、我的（个人中心）基础功能（个人信息 / 可可豆 / 等级）；
3. **S2（中优先级，配套必做）**：统一搜索系统、文件与附件处理规范、性能与体验优化、自动化测试、测试账号配置；
4. **S3（低优先级，基础完善）**：通知与消息系统基础功能、用户反馈与意见收集渠道、内容生命周期管理基础规则。
