## 一、模块目的与范围边界

### 1.1 文档目的
M3社区模块是 ReliHub 平台"话题讨论与知识互动"的核心载体。本文档用于定义话题从**发起、审核、展示、互动、悬赏、归档**的全生命周期业务规则，并规范其对应的**可可豆计费与悬赏结算模型**，确保前后端实现与上下游规则一致。

### 1.2 涵盖范围 (对应功能清单 M3-F001~F026)
- **社区主页展示**：10大话题分类卡片 + 精华话题列表 + 搜索入口。
- **话题发起流程**：标题/内容/分类/关键词/可选悬赏 + 敏感词过滤。
- **话题参与互动**：回复讨论、点赞/点踩、关注话题、举报违规。
- **话题状态流转**：待回答 → 讨论中 → 已解决 → 沉寂。
- **悬赏机制**：发起人设置悬赏，最佳回答者获得可可豆，超时自动退款。
- **分类管理**：管理后台可自定义增删分类、隐藏部分分类、修改分类名称、调整显示顺序（需超级管理员授权）。
- 注：社区话题附件上传（图片/文件）不在 MVP 范围内，归入 Phase 2/3。

### 1.3 不涵盖范围
- @AI助手参与讨论（归入 Phase 2）。
- 引用资源库文档（归入 Phase 2）。
- 超赞功能（归入 Phase 2）。
- 专家系统一对一咨询（归入 Phase 3）。
- 前端支持用户拖拽分类卡片自定义排序（归入 Phase 3）。
- 多级嵌套评论（楼中楼）结构（MVP 采用扁平化 @提及 机制，点评归入后续阶段）。
- 社区话题附件上传（图片/文件）（归入 Phase 2/3，仅 AI对话 和 资源上传 支持附件）。
- 术语说明：本模块中“内容互动单位”统称为 **帖子 (Post)**，在技术层级对应 `posts` 表。

## 二、核心业务流程与时序管控

### 2.1 话题发起与发布时序
当用户点击【发起新话题】时，按以下步骤执行：

1. **输入阶段**
   - 用户填写话题标题（必填，5~50字）
   - 用户填写话题详细内容（必填，至少10字）
   - 用户选择所属分类（必填，单选）
   - 系统主动下发预设/推荐关键字供用户勾选（最多5个关键词，可手动添加自定义关键词）
   - 用户可选择设置悬赏可可豆（可选）

2. **基础安全校验**
   - 对标题和内容执行敏感词过滤，命中时阻断提交并提示"包含敏感词，请修改后重试"
   - 注：社区话题附件上传（图片/文件）不在 MVP 范围内，归入 Phase 2/3

3. **敏感词过滤与 AI 风控分级处理**
   - **第一级 - 黑名单硬拦截**：调用全局敏感词过滤器进行校验，命中**黑名单关键词**时直接阻断提交，前端提示"包含敏感词，请修改后重试"
   - **第二级 - AI 语义风控**：调用 AI 语义分析识别潜在风险内容：
     - **高风险（明确违规）**：阻断提交，前端提示"内容审核未通过，请修改后重试"
     - **可疑（疑似违规但不确定）**：允许提交成功，但内容在前端展示为"审核中"状态，同时落库进入 M5 管理后台的待审核队列，等待管理员操作"通过"或"拒绝"

4. **悬赏设置（可选）**
   - 若用户勾选悬赏，发起人需设置悬赏可可豆数量
   - 悬赏金额最低 **10 可可豆**，上限为用户可可豆余额
   - 系统立即从发起人账户扣除悬赏金额，冻结到平台中间账户
   - 若余额不足，提示"可可豆不足，请前往获取后再试"

5. **发布成功**
   - 话题状态变更为"待回答"并上架分发
   - 基础发帖奖励：系统奖励发起人 **10 可可豆**
   - 若该分类为空分类，系统额外发放 **30 可可豆**（即全平台该分类首个话题，在基础发帖奖励外额外发放，一次性冷启动奖励）
   - 触发通知分发至关注该分类的用户

### 2.2
 深度链接协议 (Deep Linking)
- **协议方案**：`relihub://community/topic?id={topic_id}`
- **业务场景**：支持从【通知中心】、外部分享卡片直接跳转至指定社区话题详情页。
- **参数约束**：若 `topic_id` 非法或内容已被管理员删除，前端应回跳至社区首页并 Toast 提示“该话题已失效或被删除”。

### 2.3 话题互动与悬赏结算时序

1. **回复讨论**
   - 用户在话题详情页发表回复（默认最少5字，可配置）
   - 回复支持 @ 提及其他用户（被 @ 用户收到通知）
   - 回复触发敏感词黑名单拦截与 AI 语义风控（两级机制同§2.1），高风险阻断，可疑内容落库待审
   - 回复提交成功后，话题状态从"待回答"变更为"讨论中"（若尚未变更）
   - 触发通知分发至话题关注者

2. **点赞/点踩**
   - 用户对话题或回复进行点赞/点踩操作
   - 点赞与点踩互斥，支持撤回
   - 点赞/点踩影响话题热度评分
   - 点赞时触发通知分发至话题发起人/回复人

3. **采纳最佳回答**
   - 话题发起人可选择一条回复采纳为最佳回答
   - 采纳操作后，悬赏金按 **70/30 分账**发放（回答者获得悬赏金的 **floor(B × 0.7)**，平台销毁 **floor(B × 0.3)**）。
   - 话题状态变更为"已解决"
   - 采纳操作不可撤销

4. **超时未采纳**
   - 悬赏话题发布后 **30 天**内若无最佳回答被采纳：
     - 系统自动将悬赏金退回发起人账户（**平台不收取服务费，原路退回**）
     - 话题保持原状态（讨论中或沉寂）

## 三、话题分类与大厅展示规则

### 3.1 十大话题分类卡片
默认分类涵盖可靠性工程领域核心方向：
- 可靠性设计、信号测试、环境实验、元器件、失效分析、标准解读、质量管理、工艺可靠性、行业动态、综合问答

分类卡片展示规则：
- 卡片元素：图标 + 分类名称 + 话题数量
- 空分类引导：当某分类下无话题时，卡片显示**"这个分类还没有话题，您将成为第一个发言人，获得 30 可可豆 奖励！"**提示语，点击后跳转至发起话题页

### 3.2 精华话题列表规则
热度评分采用多维权重评分 `Score`：
- 首重系数 `A`：**相关性（关键字分词命中权重）**
- 侧重系数 `B`：**转化复合指数（热度指标）**
  - **完整公式**（基础公式 + 修饰符，由本模块定义）：`Heat_Index = MAX(0, [(reply_count * 5) + (like_count * 3) - (dislike_count * 5) + (distinct_views * 0.5)]) + Pinned_Boost + Featured_Boost`
  - **基础公式**：`MAX(0, (reply_count * 5) + (like_count * 3) - (dislike_count * 5) + (distinct_views * 0.5))`
    - 采用 `MAX(0, ...)` 钳位处理，确保热度值不为负数
  - **置顶加权（Pinned_Boost）**：管理员手动置顶话题 +100 分
  - **精华加权（Featured_Boost）**：管理员标记精华话题 +50 分
  - **应用场景**：本模块大厅列表排序采用完整公式；搜索系统在此基础上叠加 `Aging_Factor` 修饰符（详见 `PRD_MVP_搜索系统.md` §4.3）。
  - **点赞/点踩影响**：点踩会降低热度分值，高点踩内容在排序中下沉
- 补点系数 `C`：**上架新鲜度时间衰减**

### 3.3 话题状态标签
对齐全局内容生命周期规则（具体口径详见《PRD_内容生命周期管理.md》§3.2 话题标签规则）：

| 标签 | 触发条件 | 展示位置 |
|------|---------|---------|
| 待回答 | 新发布，等待社区响应 | 话题列表 |
| 讨论中 | 有回复，持续讨论中 | 话题列表 |
| 已解决 | 发起者采纳最佳回答 | 话题列表"解决"分类 |
| 沉寂 | 超过 60 天无任何新回复 | 搜索降权 |
| 最新 | 发布后 3 天内 | 话题列表"最新"分类 |
| 精华 | 管理员手动标记 | 话题列表"精华"分类 |
| 悬赏 | 发起时设置悬赏 | 话题列表"悬赏"分类 |

### 3.4 多维检索排序规则
检索排序采用多维权重评分，不单独依赖发布的时间：
- 首重系数 `A`：**相关性（关键字分词命中权重）**
- 侧重系数 `B`：**转化复合指数（热度指标）**
- 补点系数 `C`：**上架新鲜度时间衰减**

用户可通过二级分类筛选：最新 / 精华 / 悬赏 / 已解决

## 四、话题生命周期管理

### 4.1 状态流转规则
话题状态按以下规则流转：
```
待回答 → 讨论中 → 已解决
   ↓         ↓        ↓
  沉寂 ← ← ← ← ← ← ←
```

状态变更触发条件：
- **待回答 → 讨论中**：有新回复时
- **讨论中 → 已解决**：发起人采纳最佳回答时
- **讨论中 → 沉寂**：超过 60 天无任何新回复时
- **沉寂 → 讨论中**：沉寂话题收到新回复时（重新激活）
- **已解决 → 沉寂**：超过 60 天无任何新回复时

### 4.2 沉寂话题处置
- 超过 60 天无任何新回复（包括发起人自己的回复）→ 标记为"沉寂"
- 沉寂话题降低检索权重
- 超过 1 年无任何交互 → 进入退出机制

### 4.3 话题退出机制
1. **主动退出**：
   - 话题发起人可删除自己的话题
   - 前提：无悬赏或已结悬赏的话题
   - 删除后话题不可恢复

2. **被动退出**：
   - 沉寂话题超过 1 年无交互
   - 话题内容违规
   - 被多个用户举报且审核确认违规
   - 管理员可直接删除，通知发起人（给出删除原因）

## 五、悬赏机制规则

### 5.1 悬赏发起规则
- 发起人设置悬赏可可豆（从余额扣除，冻结到平台中间账户）
- 悬赏金额最低 **10 可可豆**，上限为用户可可豆余额
- 悬赏话题在列表中特殊标识展示
- 悬赏设置后不可修改金额

### 5.2 悬赏发放规则
- 发起人采纳最佳回答 → 悬赏金按 **70/30 分账**发放（回答者获得悬赏金的 **70%**，平台销毁 **30%**）
- 无人回答或未采纳时 → **30 天后**悬赏金退回发起人
- 分账规则：悬赏金按 **70/30 分账（与资源下载分账规则一致，回答者得 70%，平台销毁 30%）**

### 5.3 悬赏退款规则
- 悬赏话题 30 天未采纳：自动退款至发起人账户
- 退款金额 = 原悬赏金额（**平台不收取服务费，原路退回**）
- 退款触发后，话题悬赏状态变更为"已退款" (`REFUNDED`)

### 5.4 悬赏状态机对照 (对齐 DB_社区.md)

| 业务描述 | 技术枚举值 (bounty_status) | 备注 |
|---------|-------------------------|------|
| 无悬赏 | `NONE` | 话题发起时未设置悬赏 |
| 进行中 | `ACTIVE` | 已扣费并冻结，等待采纳 |
| 已结算 | `RESOLVED` | 已采纳回答，分账完成 |
| 已退款 | `REFUNDED` | 超时未采纳，全额退回 |

### 5.5 悬赏异常处理
- 悬赏发放事务失败时：触发补偿对账，重试发放或记录异常队列
- 退款事务失败时：记录异常队列，人工核对

## 六、话题互动能力

### 6.1 回复讨论规则
- 回复字数限制：默认最少 **5 字**（可配置）
- 回复支持 @ 提及其他用户
- 回复审核：敏感词黑名单拦截 + AI 语义风控（高风险阻断，可疑内容落库待审）
- 回复删除：发起人可删除自己话题下的任意回复；回复人可删除自己的回复
- 回复编辑：回复提交后 15 分钟内可编辑，超过后不可编辑

### 6.2 点赞/点踩互斥规则
- 点赞与点踩互斥，支持撤回
- 同一用户对同一话题/回复仅能点赞或点踩一次
- 点赞/点踩影响话题热度评分
- 被点赞/点踩的话题发起人/回复人收到通知

### 6.3 关注话题规则
- 用户可关注话题，接收更新通知
- 关注记录归档至 M4 个人中心
- 用户可取消关注
- 话题被删除时，自动取消所有用户的关注

### 6.4 举报规则
- 用户可举报不当话题/回复
- 举报原因选项：垃圾广告 / 色情低俗 / 暴力血腥 / 政治敏感 / 其他违规
- 举报内容进入管理员审核队列
- 多次被举报且确认违规的内容将被删除
- 举报人收到处理结果通知

### 6.5 话题置顶规则（M2级功能）
- 管理员可将优质话题置顶
- 置顶话题在列表顶部展示
- 支持取消置顶

### 6.6 分享规则（微信生态）
- 支持一键分享至微信生态
- 分享内容包含：**话题标题 + 核心内容摘要（最多50字）+ 平台小程序落地页链接**
- 分享卡片样式：标题 + 摘要片段 + 小程序图标
- 分享落地页规范（微信小程序中转页 + 部分预览 + 注册引导）归入 Phase 2
- 分享奖励分佣能力不在 MVP 范围（归入 Phase 2）

## 七、可可豆获取与消费规则

### 7.1 话题相关可可豆获取
| 获取途径 | 奖励数量 | 频次限制 | 触发条件 |
|---------|---------|---------|---------|
| 发起话题 | 10 可可豆 | 无限制 | 话题通过敏感词过滤发布 |
| 空分类首发（首个发言人） | 30 可可豆 | 仅首次/分类 | 全平台该分类下首次发起话题（在基础发帖奖励10可可豆外额外发放，一次性冷启动奖励） |
| 回答被采纳 | 悬赏金额 | 无限制 | 发起人采纳为最佳回答 |

### 7.2 悬赏可可豆消费与结算
- 悬赏金额从发起人账户扣除（下单时预扣冻结）。
- **采纳最佳回答**：系统即时按 70/30 规则将悬赏额度发放给回答者（回答者获得 **floor(B × 0.7)** 可可豆，平台销毁 **floor(B × 0.3)**），并**分别推送【悬赏达成】通知给回答者与发起人**。
- **超期退款**：30天内未手动采纳且无争议时，悬赏金自动原路返还给发起人（**平台不收取服务费**），并**推送【退款通知】**。
- 具体结算规则详见《PRD_可可豆与信誉分体系》§2.2.3。

## 八、权限边界与拦截

### 8.1 游客（M0）受限边界
- 游客可浏览前 **10 条**精华话题列表卡片并查看详情页元数据
- 划动或点击超过 10 条限制时，强制弹出价值驱动型注册引导弹窗
- 游客不可发起话题、回复、点赞、点踩、关注、举报
- 触发受限动作时弹出注册引导

### 8.2 违规用户拦截（信誉分熔断处理）
- 基于信誉分规则，低于阈值的用户将被限制发言权限
- 越权请求返回统一拦截提示 `[Error: 403] 您的账号因近期违规记录已被限制社区权限，解封时间请查看通知中心。`，并由后端错误码统一拦截

### 8.3 内容审核拦截
- 触发敏感词黑名单或 AI 预审识别为高风险（明确违规）时，直接阻断提交并给予终端提示。
- AI 预审识别为可疑（疑似违规）时，允许提交，但内容在前端展示为"审核中"状态，后台进入人工审核队列。

管理后台需提供本模块的实时读写配置能力，支持热更新下发。

| 参数名称 | 配置域 | 默认值 |
|---------|-------|-------|

| 悬赏超时自动退款 | 悬赏配置 | 30 天 |
| 沉寂话题阈值 | 生命周期配置 | 60 天 |
| 管理员标记精华权限 | 内容管理 | 需要审核权限 |
| 话题置顶管理 | 内容管理 | 管理员可操作 |
| 空分类首发发言奖励 | 用户激励配置 | 30 可可豆 |

## 十、测试验收与 NFR 指标

测试团队需覆盖以下业务红线与 NFR 核心性能基准。

### 10.1 核心业务验证用例
| 验证场景 | 预期结果 |
|---------|---------|
| 普通话题发布 | 话题上线，状态为"待回答"，获得10可可豆 |
| 空分类首发话题发布 | 话题上线，状态为"待回答"，获得10+30=40可可豆 |
| 悬赏话题发布 | 悬赏金冻结，话题上线并展示悬赏标识 |
| 普通回复 | 回复成功，话题状态变更为"讨论中" |
| 采纳最佳回答 | 悬赏金按70/30分账发放（回答者获悬赏金的70%），话题状态变更为"已解决" |
| 超时未采纳 | 30天后悬赏金退回发起人账户 |
| 话题删除（无悬赏） | 删除成功，话题不可见 |
| 话题删除（有悬赏未解决） | 删除失败，提示"当前话题存在进行中的悬赏，请采纳回答或等待悬赏到期后再删除" |
| 话题置顶 | 置顶话题在列表顶部展示 |
| 违规举报处理 | 举报内容进入审核队列，管理员处理后通知举报人 |

### 10.2 NFR 核心性能指标
以下指标以《PRD_非功能需求.md》为准，作为发布阻断红线：
- **话题列表加载**：加载 20 条话题卡片，目标值 ≤ 1 秒
- **话题详情加载**：加载话题元数据与回复列表，目标值 ≤ 500ms
- **话题搜索返回**：关键字搜索结果返回，目标值 ≤ 1 秒
- **回复提交响应**：回复提交响应 ≤ 500ms
- **悬赏结算一致性**：悬赏发放与退款事务满足 ACID 一致性要求
- **通知分发延时**：回复/点赞等动作触发通知，分发延时 ≤ 2 秒
