## 一、模块定位与范围边界

### 1.1 文档目的

M4 个人中心模块是 ReliHub 平台用户"资产可视化、行为记录、社交关系链"的管理中枢。本文档用于定义用户个人信息的展示与编辑规则、可可豆与信誉分的查询与明细、贡献与收藏等数据的统一管理规范，确保前后端实现与上下游规则一致。

### 1.2 涵盖范围 (对应功能清单 M4-F001~F028)

- **个人信息管理**：头像/昵称/单位/职务等信息的展示与编辑
- **可可豆与信誉分体系**：可可豆余额/明细/获取/消费、信誉分/明细/获取/扣减、用户等级/进度/权益
- **贡献数据**：我上传的资源、我发起的话题、我参与的讨论、获赞统计
- **社交关系**：我的收藏（资源）、我的关注（话题）
- **日常激励**：每日签到、邀请好友、邀请记录、早鸟奖励
- **消息中心**：通知中心入口（对齐《PRD_MVP_通知系统.md》）
- **系统设置**：账号安全、通知偏好、通用说明、意见反馈、账号注销

### 1.3 不涵盖范围

- 可可豆充值功能（归入 Phase 2）
- 金可可与福利豆区分展示（归入 Phase 2）
- 取现功能（归入 Phase 2）
- 扩充历史对话存储包（归入 Phase 2）
- 与专家的咨询订单管理（归入 Phase 3）

### 1.4 页面准入拦截声明

**本模块属于纯净的登录用户专有模块。** 游客用户点击"我的"底部 Tab 时，直接触发 M0 模块的注册引导弹窗（不展示个人中心页面），提前卡死权限校验入口逻辑。

## 二、个人信息展示与编辑规则

### 2.1 信息项定义

用户个人中心展示与编辑的信息项按下表执行：

| 信息项 | 必填/选填 | 说明 |
|-------|----------|------|
| 头像 | 选填 | 支持自定义上传，尺寸/格式由前端规范定义 |
| 昵称 | **必填** | 平台内唯一，不可重复 |
| 实名 | 选填（专家**必填**） | 需实名认证后方可成为专家 |
| 性别 | 选填 | - |
| 单位 | 选填 | 入门及以上等级建议填写 |
| 职务 | 选填 | 入门及以上等级建议填写 |
| 手机号 | 系统自动填充 | 注册后可用，修改需短信验证 |
| Email | 选填 | 入门及以上等级建议填写 |
| 个人简介 | 选填（专家**必填**） | - |

### 2.2 等级必填项校验规则

按用户等级区分必填/选填项，具体规则如下：

- **新兵/菜鸟**：仅昵称必填，其余信息均为选填
- **入门及以上**：需补充单位、职务、Email 等详细信息后方可享受完整等级权益
- **专家**：必须完成实名认证 + 个人简介，且信息修改需管理员二次审核后方可生效

### 2.3 编辑权限与生效规则

- **普通用户**：可随时编辑个人信息，修改后实时生效
- **专家用户**：个人信息修改需管理员二次审核，审核通过后生效

## 三、可可豆与信誉分体系

### 3.1 可可豆管理

#### 3.1.1 余额与明细展示

用户可在个人中心查看可可豆账户信息：

- **余额展示**：精确到个位，显示当前可可豆总额
- **收支明细列表**：每条记录包含来源/用途、具体事件、数量（正数为获取/负数为消费）、余额、备注（补充说明）、时间
- **明细筛选**：支持按类型（获取/消费）筛选、按时间范围筛选

#### 3.1.2 可可豆获取途径（MVP）

| 获取途径 | 奖励数量 | 频次限制 | 触发条件 | 备注 |
|---------|---------|---------|---------|------|
| 注册奖励 | 30可可豆 | 一次性 | 完成个人信息填写 | - |
| 早鸟奖励 | 20可可豆（福利豆） | 一次性 | 前200名注册用户 | 与注册奖励独立可叠加；归类为福利豆（不可取现） |
| 每日签到 | 2可可豆 | 每日1次 | 在"我的"页面点击签到 | 当日0点重置，当日未签到不累积；连续签到7天无额外可可豆奖励 |
| 邀请好友 | 10可可豆/人 | 无限制 | 被邀请人完成首次有效操作 | 邀请人获得10可可豆；被邀请人同样获得10可可豆（有效操作为固定三种：首次AI对话/首个资源上传/首个话题发起） |
| 上传资源通过审核 | 30可可豆 | 无限制 | 资源通过管理员审核 | 首次上传某分类资源额外奖励归入【早鸟/冷启动激励】；平台首发空分类资源额外奖励30可可豆（一次性） |
| 发起话题 | 10可可豆 | 无限制 | 话题通过敏感词过滤发布 | 平台首发空分类话题额外奖励30可可豆（一次性） |
| 回答被采纳 | 收取悬赏金额的70% | 无限制 | 发起人采纳为最佳回答 | 悬赏金按70/30分账发放，平台销毁30% |
| 资源被下载 | 定价×70% | 无限制 | 其他用户超额下载资源 | 基础资源权益额度内下载不产生收益；超出后按定价收取，贡献者获70%，平台销毁30% |

#### 3.1.3 可可豆消费场景（MVP）

| 消费场景 | 消费数量 | 说明 |
|---------|---------|------|
| 购买下载特惠包 | 80可可豆 | 100MB限额，永久有效直至耗尽，每次下载打8.5折 |
| 购买下载畅享包 | 350可可豆 | 500MB限额，永久有效直至耗尽，每次下载打7折 |
| 资源下载（无折扣） | 定价100% | 下载者支付100%，贡献者获70%，平台销毁30% |
| 资源下载（特惠包8.5折） | 定价85% | 贡献者获70%，平台销毁15% |
| 资源下载（畅享包7折） | 定价70% | 贡献者获70%，平台销毁0 |
| AI对话轮次扩充-入门（再聊一会） | 18可可豆/20轮次 | 永久有效直至耗尽；每日扣减轮次时优先扣减免费基础轮次，基础轮次用尽后再扣除扩充轮次 |
| AI对话轮次扩充-进阶（意犹未尽） | 40可可豆/50轮次 | 永久有效直至耗尽；每日扣减轮次时优先扣减免费基础轮次，基础轮次用尽后再扣除扩充轮次 |
| AI对话轮次扩充-高级（畅所欲言） | 70可可豆/100轮次 | 永久有效直至耗尽；每日扣减轮次时优先扣减免费基础轮次，基础轮次用尽后再扣除扩充轮次 |
| 发起话题悬赏 | 自定义≥10可可豆 | 上限为用户可可豆余额 |

#### 3.1.4 福利豆与资产豆说明

- MVP 阶段不区分金可可与福利豆，统一为可可豆账户
- Phase 2 引入金可可（资产豆，可取现）与福利豆（赠送豆，不可取现）区分

### 3.2 信誉分管理

#### 3.2.1 信誉分与等级展示

用户可在个人中心查看信誉分账户信息：

- **信誉分展示**：显示当前信誉分整数分值及对应等级徽章
- **等级体系**：新兵 → 菜鸟 → 入门 → 熟手 → 老炮 → 达人 → 专家
- **等级进度条**：展示距下一等级的信誉分差距及进度百分比
- **异常状态标识**：
    - **资产冻结**：进度条 UI 切为橙色，提示"账户资产已冻结，仅可浏览，无法执行下载/提现操作"。
    - **全限封禁**：用户无法登录，尝试登录将引导至申诉页。
- **信誉分明细列表**：每条记录包含获取/扣减原因、变动分值（正数为获取/负数为扣减）、当前总分、时间；支持按来源（资源/话题/签到/违规等）筛选
- **可见性规则**：信誉分**仅用户本人可见**；其他用户查看时仅显示发布者的等级徽章

#### 3.2.2 信誉分获取途径（MVP）

| 获取途径 | 奖励分值 | 频次限制 | 触发条件 | 备注 |
|---------|---------|---------|---------|------|
| 完成注册 | 50分 | 一次性 | 完成注册并填写个人信息 | 初始信誉分 |
| 每日签到 | 1分 | 每日1次 | 在"我的"页面点击签到 | 连续签到7天，第7天额外奖励信誉分+1（即当月第7天共获2分：1分日常+1分连续奖励）；断签后连续天数重置 |
| 邀请好友 | 10分 | 无限制 | 被邀请人完成首次有效操作 | 与可可豆奖励同步触发 |
| 上传资源通过审核 | 20 分 | 无限制 | 资源通过管理员审核 | 精华资源可额外 +20 分（管理员标记） |
| 发起话题 | 10分 | 无限制 | 话题通过敏感词过滤发布 | 精华话题可额外+10分（管理员标记） |
| 回复话题 | 5分 | 每日上限10次 | 回复内容通过敏感词过滤发布 | 防止灌水 |
| 资源被下载 | 2分/次 | 每资源每日上限10次 | 其他用户下载资源 | 同一用户对同一资源的重复下载仅计1次 |
| 资源被收藏 | 3分/次 | 每资源每日上限5次 | 其他用户收藏资源 | - |
| 资源被点赞 | 1分/次 | 每资源每日上限10次 | 其他用户点赞资源 | - |
| 话题被点赞 | 1分/次 | 每话题每日上限10次 | 其他用户点赞话题 | - |
| 回复被点赞 | 1分/次 | 每回复每日上限5次 | 其他用户点赞回复 | - |
| 回答被采纳 | 30分 | 无限制 | 发起人采纳为最佳回答 | 悬赏话题额外奖励 |

> **防刷机制**：同一用户对同一内容的重复操作（如多次点赞）仅计1次；各奖励项均有每日上限约束。

#### 3.2.3 信誉分扣减场景（MVP）

| 扣减场景 | 扣减分值 | 触发条件 | 备注 |
|---------|---------|---------|------|
| 违规内容被删除 | -20分 | 资源/话题/回复因违规被管理员删除 | 严重违规可额外扣分 |
| 恶意重复上传资源 | -10分 | AI预审检测相似度>阈值（默认80%，可配）且经管理员判定为恶意 | 善意推定：首次弹窗提醒，管理员三选一决策（7天内触发≥3次自动阻断提交） |
| 恶意举报 | -15分 | 举报被管理员判定为恶意 | 多次恶意举报可封号 |
| 长期不登录 | -5分/月 | 连续6个月未登录 | 直至降为新兵等级；登录后停止扣分 |
| 刷可可豆行为 | -50分 | 系统检测到刷可可豆行为 | 严重者直接封号 |
| 发布垃圾内容 | -10分 | 灌水、无意义内容被管理员删除 | 多次违规可封号 |
| 被举报确认违规 | -30分 | 用户举报经管理员确认违规 | 根据违规程度可额外扣分 |

> **说明**：信誉分最低为0，不会出现负数。长期不登录扣分机制旨在激励用户保持活跃，休眠账号登录后自动停止扣分。

### 3.3 用户等级体系

#### 3.3.1 等级划分与阈值

| 等级 | 信誉分范围 | 晋升下一级阈值 | 降级预警/触发阈值 |
|------|-----------|---------|---------|
| 新兵 | 0-99 | ≥ 100 | 无（降无可降） |
| 菜鸟 | 100-299 | ≥ 300 | 100 / < 80 |
| 入门 | 300-599 | ≥ 600 | 300 / < 240 |
| 熟手 | 600-999 | ≥ 1000 | 600 / < 480 |
| 老炮 | 1000-1999 | ≥ 2000 | 1000 / < 800 |
| 达人 | 2000-4999 | ≥ 5000 (申请专家) | 2000 / < 1600 |
| 专家 | ≥ 5000 | - | < 4000 (回退至达人) |

> **降级触发规则**：降级阈值 = floor(范围下限 × 0.8)，向下取整。

#### 3.3.2 等级权益

| 等级 | 核心权益 |
|------|---------|
| 新兵/菜鸟 | 基础权益，可发起悬赏（≥10可可豆） |
| 入门 | 内容权重提升，搜索排序优先 |
| 熟手 | 审核优先级提升，内容快速审核 |
| 老炮 | 高权重，搜索排序靠前 |
| 达人 | 显著提升，信誉分≥5000时提醒可申请专家认证 |
| 专家 | 平台最高等级，享受专家特权，需单独申请认证 |

#### 3.3.3 升级与降级规则

- **升级**：信誉分达到升级阈值时，自动升级（达人除外，达人需主动申请专家认证）
- **降级缓冲**：信誉分降至当前等级范围下限时，触发预警进入缓冲期，页面提示用户
- **降级触发**：信誉分继续下降至降级阈值时，触发降级，等级徽章更新
- **长期不登录**：连续6个月未登录，每月1日扣5分，直至降至新兵等级；用户重新登录后停止扣分

## 四、我的贡献与社交关系

### 4.1 我的贡献

#### 4.1.1 我上传的资源

- 展示用户上传并已通过审核的资源列表
- 信息项：资源名称、所属分类、上传时间、累计下载次数、累计可可豆收益
- 支持编辑资源基本信息（标题/描述/分类等）
- 支持删除资源（删除后资源下架，用户端不可见）

#### 4.1.2 我发起的话题

- 展示用户发起的话题列表
- 信息项：话题标题、所属分类、当前状态（待回答/讨论中/已解决/沉寂）、回复数、悬赏状态（有无悬赏/已发放/已退款）
- 支持查看话题详情
- 支持删除话题（前提：无进行中悬赏，或悬赏已结）

#### 4.1.3 我参与的讨论（M4-F014）

- 展示用户参与回复过的话题列表（按最后回复时间倒序）
- 信息项：话题标题、所属分类、我的回复摘要（截取前50字）、最后回复时间
- 支持一键跳转至话题详情页

#### 4.1.4 获赞数统计

- 汇总展示用户累计获得的点赞总数
- 明细拆分为：资源被赞数 / 话题被赞数 / 回复被赞数
- 支持按时间范围筛选

### 4.2 我的收藏与关注

#### 4.2.1 我的收藏

- 展示用户收藏的资源列表
- 信息项：资源名称、所属分类、收藏时间
- 支持一键跳转至资源详情页
- 支持取消收藏（取消后实时从列表移除）
- 当收藏的资源被删除时，自动从收藏列表中移除并通知用户
- **取消收藏边界判定**：取消动作发生后，资源的收藏总数实时 -1；此前因其他用户收藏该资源而派发给贡献者的信誉分不进行逆向扣回

#### 4.2.2 我的关注

- 展示用户关注的话题列表
- 信息项：话题标题、所属分类、关注时间、话题当前状态
- 支持一键跳转至话题详情页
- 支持取消关注（取消后实时从列表移除，话题发起人/回复人不再收到通知）
- 当话题被删除时，自动取消所有用户的关注
- **取消关注边界判定**：取消动作发生后，话题的关注总数实时 -1；此前因其他用户关注该话题而派发给发起人的信誉分不进行逆向扣回

## 五、日常激励与邀请机制

### 5.1 每日签到

- **签到入口**："我的"页面显著位置展示签到按钮
- **签到奖励**：2可可豆 + 1信誉分
- **重置规则**：每日0点自动重置，当日未签到不累积
- **连续签到展示**：显示连续签到天数（管理后台可配置是否展示）
- **签到记录展示**（M4-F019）：展示个人当月签到日历组件（历史签到高亮），兼有连续天数提示
- **防刷机制**：签到奖励的发放受底层高频防刷和同设备风控引擎管控，同一设备/账号单日多次签到仅计1次有效，防作弊规则与M5安全配置联动

### 5.2 邀请好友

#### 5.2.1 邀请码生成

- 用户生成专属邀请码（6位字母数字组合，平台内唯一）
- 同时生成邀请链接（一键复制）
- 邀请码/链接永久有效，不设过期时间

#### 5.2.2 邀请奖励规则

- **邀请人奖励**：10可可豆 + 10信誉分
- **被邀请人奖励**：10可可豆
- **触发条件**：被邀请人完成"有效操作"（固定三种：首次AI对话、首个资源上传、首个话题发起）
- **奖励发放时机**：被邀请人完成有效操作的瞬间实时发放

> **防刷机制**：邀请奖励的发放受底层高频防刷和同设备风控引擎管控。同一设备高频注册行为将被风控拦截；同一用户对同一被邀请人重复邀请仅计1次有效奖励。
> **作弊回退策略**：基于被邀请人产生的激励可可豆与信誉分一经发放，不对邀请人进行连带扣回（仅追究被邀请造假人本身的扣分）；相关防刷联动由风控网统一处理，本模块 MVP 阶段不实现邀请奖励的逆向回滚。

#### 5.2.3 邀请记录

- 展示用户邀请成功的好友列表
- 信息项：好友昵称（脱敏展示）、注册时间、完成有效操作时间、奖励发放状态（已发放/发放失败）
- 支持按时间范围筛选

### 5.3 早鸟奖励

- **奖励对象**：前200名注册用户（以成功注册时间戳排序）
- **奖励金额**：20可可豆（归类为福利豆，不可取现）
- **叠加规则**：与注册奖励30可可豆独立，可叠加发放，即新注册早鸟用户共获得50可可豆
- **防超发机制**：需采用强一致性的防超发控制策略，在极高并发情况下严格保障早鸟发放人数≤设定名额基数
- **幂等保障**：奖励发放采用幂等机制，同一注册账号最多发放1次
- **异常降级**：早鸟系统异常时，自动关闭早鸟文案展示，仅保留常规注册奖励

## 六、系统设置与安全

### 6.1 账号安全设置

- **修改密码**：验证原密码后设置新密码，密码安全策略与登录注册模块一致（强密码规则+BCrypt加密）
- **绑定/换绑手机号**：换绑须强制经过双重安全验证——第一步先校验原绑定的手机号（接收短信验证码）或原密码，通过任一项即可；第二步再验证新手机号。若原手机号不可用，须通过人工审核申诉渠道处理后方可换绑
- **第三方账号绑定**：支持绑定微信等第三方账号（具体第三方渠道由管理后台配置）

### 6.2 通知中心与设置
- **通知中心入口**：
  - **路径 A**：点击顶部导航栏（Top Bar）右侧的“铃铛”图标（**需展示全局未读红点/数字，对齐《PRD_MVP_通知系统.md》§3.2**）。
  - **路径 B**：个人中心主功能列表中“通知中心”条目（建议显示未读数摘要）。
- **通知类型配置**：用户可自定义开关各类型通知
  - 系统通知（平台公告、系统消息）
  - 互动通知（点赞、回复、@提及）
  - 悬赏通知（悬赏金发放、退款、有人采纳我的回答）
  - 激励通知（签到奖励、邀请奖励到账）
- **接收通道配置**：站内消息（默认开启）、推送通知（可选，需用户授权）
- **免打扰时段**（可选）：用户可设置免打扰时段，该时段内不发送推送通知

### 6.3 通用说明

- **用户协议**：平台服务条款与用户权利义务说明
- **隐私政策**：平台个人信息收集、使用、存储、共享规则说明
- **关于 ReliHub**：平台介绍（包括通缩机制说明——下载收费中平台留存30%可可豆直接销毁，维持可可豆长期价值）

### 6.4 意见反馈

- **反馈入口**："我的"页面设置中提供意见反馈入口
- **反馈表单**：问题描述（必填，10-500字）+ 联系方式（选填）
- **附件支持**：可上传截图附件（图片格式，最多3张，单张≤5MB）
- **处理流程**：提交后进入后台处理队列，客服/产品团队处理，结果通过站内消息通知用户

### 6.5 账号注销

- **注销入口**：设置页面底部，提供账号注销入口（需二次确认 + 短信验证）
- **注销前置校验拦截**：若当前账号存在进行中的资金流转业务（包括但不限于：进行中的悬赏话题尚未结案或退款、AI轮次扩充购买尚未消耗完毕、提现申请尚在处理中），系统必须阻断注销操作并提示用户"存在进行中的资金业务，请完结后再试"，待上述业务完结后方可继续注销流程
- **注销影响说明（需用户阅读并确认）**：
  - 保留贡献的资源/话题（转为匿名发布，昵称替换为"已注销用户"）
  - 删除历史AI对话记录
  - 永久作废账户内所有可可豆（数据库不物理删除，但账户状态标记为注销锁定，此部分可可豆永久退出财务流通层）
  - 清除评论、点赞、点踩、收藏、关注等所有互动记录
  - 解除第三方账号绑定
- **注销后**：账号不可恢复，原手机号/邮箱可重新注册新账号

## 八、测试验收与 NFR 指标

### 8.1 核心业务验证用例

| 验证场景 | 预期结果 |
|---------|---------|
| 新用户注册后查看个人中心 | 显示默认昵称，可可豆余额=注册奖励30豆，信誉分=50分，等级=新兵 |
| 前200名用户注册 | 额外获得早鸟奖励20可可豆（福利豆），总余额=50可可豆 |
| 第201名用户注册 | 仅获得注册奖励30可可豆，不获得早鸟奖励 |
| 每日签到（第1天） | 获得2可可豆+1信誉分，连续签到天数=1 |
| 每日签到（第2天未签，第3天再签） | 连续签到天数重置为1，不累积 |
| 邀请好友A并完成有效操作 | 邀请人获10可可豆+10信誉分；好友A获10可可豆 |
| 同一设备重复注册/邀请 | 高频防刷拦截，不重复发放奖励 |
| 上传资源通过审核 | 获得30可可豆+20信誉分；首次上传某分类额外+30可可豆 |
| 发起话题（非空分类） | 获得10可可豆；首次发起某分类话题额外+10可可豆 |
| 资源被下载（超额，定价100可可豆） | 贡献者获得70可可豆，平台销毁30可可豆 |
| 购买下载特惠包 | 扣除80可可豆，获得100MB下载限额 |
| 购买AI轮次扩充-入门 | 扣除18可可豆，当日增加20轮次 |
| 信誉分长期不登录（连续6个月） | 每月1日扣5信誉分，直至降至新兵等级 |
| 信誉分降至降级阈值 | 触发降级，等级徽章更新，页面提示用户 |
| 查看可可豆明细 | 正确展示收支记录，支持按类型/时间筛选 |
| 查看信誉分明细 | 正确展示分值变动记录，含增加/扣减原因 |
| 账号注销（确认二次确认） | 保留资源/话题（匿名），清除个人数据，余额清零，不可恢复 |

### 8.2 NFR 核心性能指标

| 指标项 | 目标值 (P95) | 发版阻断阈值 (P99) | 说明 |
|--------|------------|-----------------|------|
| 个人中心首页加载 | ≤1秒 | >2秒 | 含用户等级/资产/贡献数据聚合 |
| 可可豆/信誉分明细加载 | ≤500ms | >1秒 | 分页列表倒序展示，业务层限制最大查询深度（至多查近6个月记录），防止深分页性能击穿 |

| 邀请链接生成 | ≤200ms | >500ms | 含码生成与计数 |
| 等级升级判定 | ≤100ms | >200ms | 实时触发，异步更新 |
| 早鸟名额原子扣减 | ≤50ms | >100ms | 高并发下名额一致性保障 |
