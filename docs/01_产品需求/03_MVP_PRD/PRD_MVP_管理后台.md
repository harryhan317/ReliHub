## 一、模块目的与范围边界

### 1.1 文档目的

M5管理后台是ReliHub平台的"运营管控中枢"，为平台运营人员提供全业务线的可视化监控、参数动态配置、内容审核、用户管理及安全风控能力。本文档用于定义管理后台各功能模块的业务规则、数据口径及操作边界，确保运营动作与上下游产品规则一致。

**与各业务模块的联动关系**：管理后台是所有业务模块（M0/M1/M2/M3/M4）的规则配置源与数据汇聚视图，各模块 PRD 中的"管理后台配置"条款均以本文档为准。

### 1.2 涵盖范围（对应功能清单 M5-F001~F050）

- **系统看板**：核心业务指标可视化（用户/对话/资源/话题/可可豆流转）。
- **参数配置**：AI问答等级权益、等级体系、功能权限、可可豆/信誉分奖惩规则（含早鸟/邀请激励）、**AI模型/System Prompt版本化管理与回滚**(Phase 3 扩展 RAG)、敏感词库、资源/社区分类管理与显示配置、下载扩充包定价、通缩比例、个人信息等级必填项配置。
- **安全风控**：反爬虫/反刷可可豆规则、微信内容安全API配置、安全日志查看、业务层/技术层限流规则。
- **内容审核**：资源审核及申诉处理、社区内容审核（含AI可疑内容）、**资源点评审核**、意见反馈处理（举报/纠错）、审核意见反馈。
- **用户管理**：用户列表/筛选/详情查看、违规处理（警告/扣分/封号）、账号解锁、专家认证审核及**资料变更审核**。
- **管理员账号体系**：管理员登录机制、账号生命周期管理、操作二次验证。

### 1.3 不涵盖范围

- 充值记录查看（归入 Phase 2）。
- **分享点击激励**配置（归入 Phase 2；注：MVP 仅支持 **邀请成功奖励**）。
- 专家咨询抽佣比例配置（归入 Phase 3）。
- 广告管理、广告数据统计（归入 Phase 3）。
- 公告管理、Banner配置等运营工具（归入 Phase 2/3）。

### 1.4 页面准入拦截声明

本模块属于**运营人员专有后台**，需分配独立账号及角色权限（管理员/超级管理员），不支持游客或普通用户访问。所有面向用户端的功能模块（如M0启动模式、M2资源模块等）在检测到访问者身份为普通用户时，应跳转至用户端登录页，而非管理后台。

### 1.5 系统形态与界面规范

**系统形态**

本模块为**独立 PC 端 Web 管理系统**（以下简称"管理后台"），区别于面向普通用户的移动端/H5响应式产品，具体特征如下：

| 特征 | 管理后台 | 用户端 |
|-----|---------|-------|
| 终端类型 | PC 浏览器（1920×1080 及以上分辨率） | 移动端/H5（适配多尺寸屏幕） |
| 部署域名 | 独立域名或独立路由（如 `admin.reilihub.com` 或 `reilihub.com/admin`） | 主域名（如 `reilihub.com` 或 App 内嵌 Webview） |
| 技术架构 | 纯 PC 端 Web（SPA 或 MPA），不考虑移动端适配 | 移动端优先，响应式布局 |
| 登录体系 | 独立账号体系（管理员/超级管理员），与用户端账号隔离 | 用户账号体系，与管理后台账号隔离 |

> **说明**：管理后台不支持移动端访问（手机浏览器打开时展示"请使用 PC 设备访问管理后台"提示页），也不支持在用户端 App 或小程序内嵌访问。

**界面样式规范**

| 规范项 | 要求 |
|-------|------|
| 设计分辨率 | 1920×1080 为基准设计分辨率，支持 1440×900 及以上屏幕 |
| 最小宽度 | 1280px（低于此宽度时出现横向滚动条，不做自适应缩放） |
| 布局模式 | 左侧固定导航栏（宽度 200~240px）+ 右侧内容区（自适应） |
| 配色规范 | 管理后台使用专属深色/浅色主题（与用户端橙色主色调区分），超级管理员可在"个人设置"中切换 |
| 字体规范 | 中文字体：系统默认无衬线字体（如 PingFang SC / Microsoft YaHei）；英文字体：Inter / Roboto |
| 组件库 | 使用 Ant Design Pro 或同级别企业级 PC 组件库（确保数据表格/表单/图表等组件专业度） |
| 数据表格 | 支持列排序/列宽拖拽/分页（默认每页 20/50/100 条可配置）/ 全屏展示/ CSV 导出 |
| 表单校验 | 实时校验 + 提交时全量校验，校验失败时错误提示在输入框下方（不使用 Tooltip） |

**交互规范**

| 规范项 | 要求 |
|-------|------|
| 操作反馈 | 按钮点击后立即置灰（防重复提交），操作成功后 Toast 提示，失败后 Alert/抽屉错误详情 |
| 二次确认 | 关键操作（删除/封禁/重置等）需二次确认弹窗，弹窗使用模态遮罩，不可点击外部关闭 |
| 加载状态 | 异步请求必须展示 Loading 态（骨架屏或 Spinner），禁止静默等待 |
| 空状态 | 列表为空时展示空状态插画 + 引导文案（而非留白） |
| 敏感信息 | 密钥/密码等敏感字段在前端展示时脱敏（详见各章节），不支持明文复制 |

## 四、系统看板

### 4.1 核心数据概览（S0）

**平台全量数据统计**

管理后台首页（即系统看板主视图）实时展示全平台核心业务指标，数据刷新频率不低于每5分钟一次。展示指标包含：

| 指标名称 | 数据定义 | 单位 |
|---------|---------|------|
| 全平台用户总数 | 注册用户 + 游客虚拟ID的去重总数 | 人 |
| 注册用户数 | 完成注册流程的账户总数（不含游客） | 人 |
| AI对话会话总数 | 自平台上线以来的历史AI会话总条数 | 条 |
| 资源上传总量 | 自平台上线以来提交并通过审核的资源总数 | 个 |
| 话题发布总量 | 自平台上线以来发布的话题总数（不含已被管理员删除的） | 条 |

> **说明**：各指标均为累计值，非日增。若需查看日增量，需进入§4.2业务数据分析分页查看。

**DAU/MAU统计**

- 日活跃用户（DAU）：当日有任意有效操作（登录、浏览、下载、对话等任一）的去重用户数，默认展示近30天曲线折线图。
- 月活跃用户（MAU）：近30天滚动周期内有任意有效操作的去重用户总数，默认展示近6个月趋势。
- 维度切换：支持在"自然日"与"自然周"之间切换统计口径。

**游客转化漏斗**

- 展示"累计游客会话数 → 注册用户数 → 转化率"的逐日趋势。
- 转化率计算口径：转化率 = 当日新增注册用户数 ÷ 当日有会话的游客数 × 100%。
- 辅助运营人员评估各渠道拉新效率，支持按渠道维度（UTM参数）二次筛选。

### 4.2 业务数据分析（S1）

以下指标均以日为单位聚合计算，默认展示近30天，支持自定义时间范围。

**AI对话频次**

- 平均每用户每日对话会话数 = 当日会话总数 ÷ 当日活跃用户数。
- 单会话平均轮次 = 当日所有会话轮次总和 ÷ 当日会话总数。
- AI Token日消耗量（按模型版本拆分）：后台从AI服务商获取每日调用token量，估算对应费用（需在§5.5配置AI模型时填入单价）。

**资源下载量**

- 日均资源下载次数：当日资源被成功下载的总次数（含免费资源）。
- 下载资源数分布：按资源定价区间（0~10/10~50/50~100/100+可可豆）分组统计下载次数及占比。

**社区发帖量**

- 日均话题发布数：当日新发布且成功提交的话题总数（不含待审核状态的）。
- 日均回复数：当日所有话题下新增回复的总数。

**留存率统计**

- 次日留存率：第N日新增注册用户在第N+1日回访的比例。
- 7日留存率：第N日新增注册用户在第N+7日回访的比例。
- 30日留存率：第N日新增注册用户在第N+30日回访的比例。
- 支持按注册渠道维度拆分查看留存差异。

**功能使用分布**

- 统计用户在爱问（AI对话Tab）/ 资源 / 社区 / 我的 四个Tab的页面停留时长占比，辅助判断用户活跃结构。
- 展示方式：环形图 + 百分比列表，点击各Tab可下钻至该Tab下的详细行为数据。

### 4.3 可可豆与Token流转（S1）

**可可豆流转总览**

| 流转类型 | 包含项目 |
|---------|---------|
| 发放总量 | 注册奖励、签到奖励、邀请奖励、早鸟奖励、资源被下载收益、话题/回复被采纳奖励、运营活动奖励 |
| 消耗总量 | 资源下载、购买AI扩充包、购买下载扩充包、发起悬赏、支付提问 |
| 销毁总量 | 通缩模型销毁（普通下载按通缩比例；使用扩充包按实付比例-贡献者比例自动计算） |

展示近30天每日净增/净减趋势折线图，支持按"发放/消耗/销毁"三种类型切换筛选。

**Token使用统计**

展示每日AI对话Token消耗总量（按模型版本分组）、估算费用（= Token消耗量 × 已配置的单价），辅助财务核算AI成本。

### 4.5 系统通知广播 (M5-F011) (S0)

超级管理员具权通过管理后台向全平台用户下发全局系统通知。

- **广播频次限制**：全量广播消息每自然日上限 **1 次**（防运营滥用），每次群发需填写操作原因并记录审计日志。
- **免打扰豁免机制**：广播类系统公告字段 `priority` 默认为 HIGH，不受用户侧 DND（免打扰时段）约束，确保核心消息及时触达全量用户。
- **模板化下发**：支持在后台定义消息文案变量并实时预览效果。
- **平滑下发控制**：系统自动分批执行 WebSocket 触达，避免瞬间流量激增引起连接中断。

- **多维筛选**：支持按时间范围（起止日期，支持快捷选项：今日/近7天/近30天/自定义）、业务模块（AI对话/资源/社区/用户）、用户维度（注册时间区间/等级/渠道）组合筛选所有看板数据。
- **数据导出**：支持将当前筛选结果导出为 CSV 和 Excel（.xlsx）格式，包含数据源、时间范围、筛选条件、导出时间等元信息注脚。

## 五、参数配置

### 5.1 AI问答等级权益配置（S0）

**配置内容**

按用户等级（新兵→菜鸟→入门→熟手→老炮→达人→专家）分别配置以下参数：

| 参数名称 | 说明 | 可调范围 |
|---------|------|---------|
| 每日新会话上限 | 每日可发起的新会话个数（重新打开AI对话页即算新会话） | 1~999次，默认值见可可豆体系§4.1.2 |
| 单会话轮次上限 | 单个会话中用户可与AI交互的最大轮次（每轮含用户输入+AI回复） | 1~100轮，默认值见可可豆体系§4.1.2 |
| 每日问答总轮次上限 | 当日所有会话的轮次之和上限（含跨会话累计） | 1~999轮，默认值见可可豆体系§4.1.2 |
| 单会话Token上限 | 单个会话允许消耗的最大Token数（含输入+输出+上下文） | 1k~1024k，默认值见可可豆体系§4.1.2 |

**配置规则**

所有参数通过配置中心（如Apollo/Nacos）管理，不得在前端或后端代码中硬编码。超级管理员在管理后台修改并提交后，系统实时推送至各业务服务节点，业务服务≤10秒内完成热更新，用户侧刷新客户端后感知延迟≤30秒。

#### 5.4.2 资源下载贡献者收益比例与通缩销毁比例联动规则

**核心联动公式**

```
贡献者收益比例（X%） + 平台通缩销毁比例（Y%） = 100%
```

- 贡献者每次被下载时所得可可豆 = 资源定价 × X%
- 平台每次下载时销毁可可豆 = 资源定价 × Y%

**配置界面交互说明**

超级管理员进入"可可豆奖惩规则配置"页面（路径：参数配置 → 可可豆奖惩规则 → 资源下载收益比例区块），看到左侧为"贡献者收益比例"输入区，右侧联动显示"通缩销毁比例"（只读，由系统根据公式自动计算）。

**Step 1：比例调整交互**

1. 超级管理员通过以下方式之一调整贡献者比例：
   - **滑块调节**：拖动贡献者比例滑块（刻度60%~70%，步距1%）
   - **数值输入**：直接输入具体数值（输入框接受0~100整数）
2. 每次调整后，右侧"通缩销毁比例"输入框立即联动刷新（计算公式：销毁比例 = 100% - 贡献者比例），刷新延迟≤100ms（前端计算，不请求后端）。
3. 页面底部实时显示本次修改的生效预览："调整后：贡献者得X%，平台销毁Y%"。
4. 若贡献者比例输入值超出60%~70%范围，前端拦截并提示"贡献者比例须在60%~70%之间（不得超过畅享包折扣率70%）"，不允许提交。

**Step 2：提交确认交互**

1. 超级管理员点击"保存配置"按钮。
2. 系统弹出"二次验证"弹窗（详见§9.3双因素操作保护），要求输入短信验证码。
3. 验证码验证通过后，系统写入配置表（记录：配置项/修改前值/修改后值/操作人/操作时间）。
4. 配置下发至配置中心，业务服务≤10秒内完成热更新。
5. 保存成功后，页面顶部展示绿色Toast提示："配置已保存，贡献者收益比例已更新为X%，预计10秒内全量生效"。
6. 页面底部"历史贡献者比例调整记录"折叠面板自动展开（若为首次调整）或追加新记录。

**Step 3：历史记录查看**

"历史贡献者比例调整记录"折叠面板（默认收起）包含以下字段：

| 字段 | 说明 |
|-----|------|
| 调整时间 | 精确到秒 |
| 操作人 | 超级管理员账号名 |
| 修改前数值 | 贡献者比例X% / 销毁比例Y% |
| 修改后数值 | 贡献者比例X% / 销毁比例Y% |
| 关联变更单号 | 链接至审批流（如有） |

面板支持按时间范围筛选和CSV导出。

**联调测试要点**

研发在联调阶段需验证以下场景：

1. **实时联动验证**：修改贡献者比例滑块，销毁比例在≤100ms内刷新。
2. **历史存量保护**：已有下载订单不追溯重算，贡献者收益以订单创建时比例为准。
3. **新增订单验证**：新下载订单按当前配置比例执行，可通过日志核查。
4. **边界值测试**：贡献者比例设为60%/70%时，销毁比例相应为40%/30%，系统正常运行。
5. **配置一致性**：多实例部署时，各节点配置中心推送后数值一致。

**特殊场景说明**

使用下载扩充包（特惠包/畅享包）下载资源时，不适用上述固定销毁比例——实际销毁比例由系统按以下公式自动计算：

```
扩充包下载实际销毁比例 = 下载者实付比例 - 贡献者收益比例
```

- 特惠包实付85折（可可豆支付部分为85%），贡献者收益X%（当前配置），则销毁(85%-X%)。
- 畅享包实付70折，贡献者收益X%（当前配置），则销毁(70%-X%)。
- 此联动值不写入配置表，由系统在用户发起扩充包下载时实时计算扣减。
- **交叉约束**：贡献者收益比例不得超过当前最低扩充包折扣率（畅享包70%），即贡献者收益比例上限需 ≤ 70%。当畅享包折扣配置变更后，系统自动重新计算贡献者比例上限（取当前畅享包折扣值）；若变更后畅享包折扣低于当前已配置的贡献者比例上限，系统禁止保存并提示"贡献者比例不得超过畅享包折扣率[新值]%"。当贡献者比例配置 > 70%时，系统禁止保存并提示"贡献者比例不得超过畅享包折扣率70%"。
> **用户端感知**：贡献者在"我的收入"页面查看收入明细时，明细标注"本次下载贡献者收益比例：X%"，以便贡献者理解收益计算逻辑。

**重复上传资源善意推定机制**

> **核心原则**：用户可能并不清楚自己或他人曾上传过相同/相似资源，系统采用"先提醒、后审核、慎扣分"的善意推定策略。

**Step 1：AI预审检测与全量比对**

用户提交资源后，系统立即执行以下双重合规检测：
1. **全库 MD5 哈希比对**：对全平台**所有状态**（含已通过、待审核、申诉中）的资源进行瞬时哈希比对。若哈希完全一致，直接标记为“🔁 疑似重复”。
2. **AI 内容语义查重**：对全平台**已通过审核及待审核**的资源进行语义向量召回比对（采用 Cosine Similarity 语义余弦相似度算法）。查重范围不区分上传者身份。已下架/已注销/已删除的资源不纳入检测。

> **防漏洞说明**：将“待审核”资源纳入比对范围，可有效防止恶意用户通过“多点并发提交”来绕过查重系统。

**Step 2：弹窗提醒（相似度 > 阈值时触发）**

当检测到相似度超过配置阈值（默认80%，管理后台可配）时，系统向用户展示弹窗：

- 弹窗标题："⚠️ 发现相似资源"
- 弹窗内容：展示最多3条相似资源（按相似度降序），每条含：资源标题 / 上传者昵称（若为本人则标注"您本人"）/ 上传时间 / 相似度百分比
- 弹窗底部：引导文案"如果您的资源确有不同，欢迎继续提交，管理员会审核判定。"+ 两个按钮：[取消提交]（主按钮）/ [仍然提交]（次按钮）
- 文件MD5完全一致时，弹窗文案调整为"您提交的文件与平台已有资源完全一致"，并额外提示"如需查看该资源，请前往资源详情页"

用户选择[取消提交]：流程结束，不记录、不扣分、不计入重复次数。
用户选择[仍然提交]：资源进入审核队列，自动附加 **🔁 疑似重复** 标签。

**Step 3：管理员三选一审核（见§6.1审核操作）**

疑似重复资源在审核详情页增加"相似度报告"区块，管理员基于报告做出三种决策之一：

| 操作 | 按钮文案 | 效果 | 适用场景 |
|------|---------|------|----------|
| ① 通过 | "通过上架" | 正常上架，正常发放上传奖励 | 虽有相似但确为不同资源 |
| ② 拒绝（不扣分） | "拒绝（善意重复）" | 不上架；通知用户拒绝理由（必填≥10字）；**不扣信誉分** | 用户忘记上传过 / 不知平台已有 |
| ③ 拒绝（扣分） | "拒绝（恶意重复）" | 不上架；通知用户拒绝理由（必填≥10字）；**扣减信誉分**（默认-10分，可调-5~-50分） | 换标题刷奖励 / 文件MD5完全一致 / 短期内多次触发 |

管理员选择③时，系统弹出二次确认弹窗（防误操作）。

## 六、内容审核

### 6.1 资源审核（S0）

**待审核队列**

队列列表字段：资源ID（可点击跳转详情）/ 资源名称 / 提交人昵称 / 提交时间 / 分类 / 定价 / 标签（如 🔁 疑似重复）/ SLA剩余时间（倒计时）。

- 排序规则：默认按提交时间正序（先到先审）；临近SLA超时的条目自动高亮（黄色背景）并置顶；标记"🔁 疑似重复"的条目以橙色底色区分。
- **领用锁定机制**：管理员点开审核详情页时，该条目对其他管理员显示"正在处理中（锁定者：XXX）"并自动锁定 15 分钟（可配置），防止重复审核与状态冲突。锁定期内该条目不可被其他管理员操作；锁定期满后未提交审核结果，条目自动解锁并恢复可操作状态。
- 分页：每页20条，支持翻页和跳转。
- 筛选：支持按分类/提交时间范围/提交人等级/定价区间/标签（疑似重复/普通）筛选。
- **批量处理（M5-F037）**：列表左侧支持复选框（Checkbox）进行多选。点击底部出现的“批量处理条”可执行：批量通过、批量拒绝（需统一理由）、批量转派（至指定管理员）。

**审核操作**

管理员对单条资源执行以下操作之一：

| 操作 | 结果 | 后续流程 |
|-----|------|---------|
| 通过 | 资源状态变更为"已通过"并上架 | 用户收到"资源已上线"通知 |
| 拒绝 | 资源状态变更为"被拒绝" | 用户收到拒绝理由（必填，最多500字），可在修改后重新提交 |

**疑似重复资源的三选一审核操作**

对标记为 🔁 疑似重复 的资源，审核详情页增加**"相似度报告"区块**，包含：

| 报告字段 | 说明 |
|---------|------|
| 本次提交资源 | 资源标题 / 文件名 / 文件大小 / 文件MD5。**需提供【下载原文件】与【在线预览】入口**以便人工核对。 |
| 相似资源清单 | 最多5条，含：标题 / 上传者 / 上传时间 / 相似度百分比 / 快捷跳转查看 |
| 差异摘要 | AI自动生成：标题/文件名/内容差异描述（如"标题不同，内容相似度95%，文件MD5相同"） |
| 用户历史重复记录 | 该用户历史触发重复检测次数（帮助管理员判断是否累犯） |

管理员基于报告做出三种决策之一（详见§5.4.3善意推定机制）：

| 操作 | 按钮文案 | 效果 |
|------|---------|------|
| ① 通过 | "通过上架" | 正常上架，正常发放上传奖励（可可豆+信誉分） |
| ② 拒绝（善意） | "拒绝（善意重复）" | 不上架；通知用户拒绝理由（必填≥10字）；不扣信誉分 |
| ③ 拒绝（恶意） | "拒绝（恶意重复）" | 不上架；通知用户拒绝理由（必填≥10字）；扣减信誉分（见§5.4.3配置值）；需二次确认 |

> **资源状态机**：
> ```
>                    ┌─→ 已通过（上线）
>                    │
> 待审核 ─────────────┼─→ 被拒绝（善意重复）──┬─→ 申诉中
>                    │                   │
>                    ├─→ 被拒绝（恶意违规） ──┤
>                    │                   └─→ 修改后重新提交 ──→ 待审核（新记录）
> ```
> - "待审核"可通向"已通过"、"被拒绝（善意重复）"或"被拒绝（恶意违规）"三条路径。
> - "被拒绝"后用户可选择"申诉"或"修改后重新提交"（原资源删除，生成新提交记录）。
> - 非疑似重复的普通资源仍为通过/拒绝两选一（拒绝默认不扣信誉分）。
> - 无"打回修改"中间态。

**资源信息修改**

对已通过的资源，管理员可在用户管理后台手动修改以下字段：分类（需有分类管理权限）/ 标题 / 简介 / 定价 / 预览量。修改操作记录入操作日志，并触发一条系统消息通知资源贡献者"您的资源信息发生了变更，请知悉"。

**审核申诉处理**

- 用户在资源被拒绝后7个自然日内可发起申诉（每条资源限1次）。
- 申诉队列独立于待审核队列，标记为"申诉中"，由原审核员或指定审核员重新审理。
- 最终判定结果为"维持拒绝"或"审核通过"，以最终判定为准，不支持二次申诉。

**SLA管理**

- 标准审核SLA：48小时内**结单**（从提交时间戳起算）。
- 超时预警转派机制：管理员在36小时（75% SLA时间）仍未处理时，系统自动向该管理员发送站内提醒，并将该条目**转入**其他空闲管理员的处理队列；48小时仍未结单时，若该任务仍未完成，系统立即将其标记为“🚨 紧急超期”并触发超管介入。
- **SLA 穿透处理**：若任务连续发生二次转派且均为超级管理员，系统将该条目标记为“🚨 紧急超期”，并向平台所有方发送业务告警通知，同时触发外部通知（邮件/短信至预先配置的告警接收人），**转派终点规则**：任务进入"紧急超期"状态后立即停止自动转派，停留在当前超管队列中，由平台方通过外部介入处理，不再在系统内循环转派。
- **告警链路自检**：管理后台系统设置中提供"告警通知通路测试"功能，支持超级管理员手动触发测试邮件/短信，验证邮件服务器与短信网关在关键时刻不会静默故障。测试操作记录入操作日志。
- SLA倒计时：列表页每个条目右侧显示剩余时间，<6小时显示红色高亮。

### 6.2 社区内容审核（S1）

**AI语义风控与审核队列**

AI对话系统在用户提交话题/回复时，对文本内容进行语义分析：

- **黑名单关键词命中**：前端直接阻断提交（规则见§5.9，不进入审核队列）。
- **风控关键词或AI语义可疑命中**：内容允许提交成功，但进入"审核中"状态（用户可见自己的内容，但其他用户不可见），同时落库进入管理后台的"AI可疑内容"审核队列（M5-F034）。

> **前端敏感词实时拦截**：用户提交话题/回复时，前端对标题和内容执行敏感词过滤——详细规则见 M3 社区模块 PRD。

**AI可疑内容审核队列**

| 字段 | 说明 |
|-----|------|
| 内容ID | 话题ID或回复ID |
| 内容类型 | 话题 / 回复 |
| 发布人昵称 | 内容发布者 |
| 提交时间 | 内容提交时间 |
| 触发原因 | AI判断依据（如"语义疑似涉政"/"多次出现风控关键词"等）；显示对应标签：🚨 风险内容（命中敏感词库风控词）/ 🔁 疑似重复 |
| SLA剩余时间 | 48小时倒计时（AI可疑内容与举报处理为独立队列，独立计时，互不影响优先级） |

**举报处理及资源点评审核**

- 用户可在任意话题/回复/资源上点击"举报"按钮，选择举报原因（广告/人身攻击/违法违规/其他）。
- **资源点评管理**：系统实时同步资源模块下的所有“评价/点评”。评价内容自动过敏感词库，可疑评价进入“评价待审核”队列（逻辑同话题回复）。
- 被举报内容与待审评价进入"内容待处理"总队列，由社区内容审核员处理。
- 处置选项：通过（取消隐藏）/ 删除内容并警告发布者 / 忽略（恶意举报处理）。
- 举报处理SLA：48小时内处理，超过后内容自动隐藏直至处理完成。

**审核意见反馈**

管理员填写审核意见后（支持富文本，最多500字），系统通过站内消息自动通知内容发布者。意见内容可选择预设模板（如"您的内容包含违规信息，请修改后重试"）或自定义填写。

**审核工作量统计**

- 统计各审核员当日/本周的审核数量（通过/拒绝/忽略分别计数）。
- 一次通过率 = 当日首次审核通过数 ÷ 当日总审核数。
- 申诉率 = 当日被申诉的审核数 ÷ 当日总审核数。
- 以上数据用于客服质量考核，不作为强制排名依据。

### 7.3 账号安全管控（S1）

**账号锁定解锁**

- 同一账号1小时内连续5次登录失败。

**异常登录日志**

**专家认证申请队列**

队列展示：用户ID / 昵称 / 申请时间 / 资质材料预览（点击可查看附件）/ 申请理由摘要。

**认证审核操作**

| 操作 | 结果 |
|-----|------|
| 审核通过 | 用户等级变更为"专家"，获得专家专属权益（AI对话高级权益/专属标识等），系统通知用户 |
| 审核拒绝 | 填写拒绝原因（必填），系统通知用户，用户可修改材料后重新申请 |

**专家资料变更审核**

- 已认证专家修改姓名、职务、单位或个人简介等核心资料时，资料进入“待审核”状态。
- 审核期间，用户等级标识保留“专家”徽章，但对外展示的资料仍为旧版，直至管理员审核通过后更新为新版认证。
- **编辑锁定机制**：资料在"待审核"状态下，用户端的修改入口暂时置灰（不可再次提交新修改），直至当前审核流程闭环（通过/拒绝）后再开放。此机制避免同一用户产生多条冲突的待审核记录。
- 审核逻辑同认证审核。

**意见反馈工单列表**

默认排序：待处理 → 处理中 → 已解决 → 已驳回（按状态优先级）；同状态下按提交时间正序。

**SLA规则**：反馈工单需在48小时内处理完毕；超时自动提升优先级并向管理员发送提醒通知。

**状态流转（工单状态机）**

```
[待处理] → [处理中] → [已解决]
                        ↘ [已驳回]
```

- 待处理：用户刚提交，尚未有管理员接单。
- 处理中：管理员已接单，正在处理中。
- 已解决：管理员确认问题已修复/解答完毕。
- 已驳回：管理员认定为无效反馈（如重复提交/不在支持范围内）。

**处理操作**

管理员在工单详情页填写：处理结果描述（富文本，必填）/ 内部备注（可选，不对用户展示）/ 最终状态（已解决/已驳回）。

**结果通知**

工单状态变更为"已解决"或"已驳回"后，系统自动通过站内消息通知提交用户，告知处理结果摘要及查看详情的入口。

**满意度评价**（SUP-F051，✅ MVP）

用户在收到结果通知后，可对本次服务进行全量满意度评价（满意/不满意+可选简短文本，见意见反馈 PRD §3.2）。满意度结果计入客服质量考核，供运营负责人分析报表。

## 九、管理员账号体系与权限控制

### 9.1 管理员登录机制

**登录入口**

- 管理员：通过管理员专属登录页（独立域名或独立路由，如`admin.reilihub.com/login`）输入管理员账号+密码+验证码（图形验证码）登录。

> **注**：管理员账号与普通用户账号在数据库中独立存储，不可混用。管理后台登录页不接受普通用户账号密码。

**密码复杂度规则**

| 规则项 | 要求 | 违规提示 |
|-------|------|---------|
| 密码长度 | 最少8字符，最多32字符 | "密码长度须为8~32位" |
| 字符组合 | 必须同时包含：大写字母、小写字母、数字 | "密码须同时包含大写字母、小写字母和数字" |
| 特殊字符 | 必须包含以下任一：!@#$%^&*()_+-= | "密码须包含至少一个特殊字符：!@#$%^&*()_+-=" |
| 禁止项 | 禁止与账号名相同 / 禁止 4 位及以上相同字符（如 1111）或连续递增/递减字符（如 1234、4321） / 禁止常见弱密码（如 `Admin123!`、`Password1`） / **允许包含出生年份（如1985）作为组合的一部分** | "禁止使用与账号名相同的密码" / "密码允许包含年份，但禁止 4 位以上连续/重复数字" / "检测到弱密码" |
| 初始密码 | 系统随机生成12位（含大小写+数字+特殊字符），首次登录强制修改 | - |
| 定期更换提醒 | 默认90天提醒一次，可在管理后台修改周期（30/60/90/180天或关闭提醒） | 登录成功后若已达更换周期，弹窗提示"您的密码已超过{90}天未更换，请前往修改" |

> **安全加固说明**：管理后台采用比用户端更严格的“全要素强密码”规则（强制集齐 4 类字符），旨在构筑运营中枢的最高防御基线。

> **密码存储与传输**：密码经bcrypt算法单向哈希后存储（cost factor≥12），不明文保存，不传输明文（全程HTTPS TLS1.2+）。
> 
> **密码修改交互**：管理员在"修改密码"页面输入旧密码+新密码+确认密码，新密码输入时实时校验复杂度（每输入一个字符即时反馈是否满足各项规则），满足全部条件后"确认"按钮可点击。

**登录失败锁定规则**

| 规则项 | 阈值 | 处置方式 |
|-------|------|---------|
| 同一账号单小时登录失败次数 | 连续5次失败 | 触发图形验证码（下次登录需输入图形验证码） |
| 同一账号当日登录失败累计次数 | 累计10次失败 | 账号临时锁定1小时，登录页提示"账号已被临时锁定，请{时间}分钟后重试" |
| 超级管理员登录敏感操作 | 任意关键操作前 | 强制短信二次验证（不论失败次数） |

**超级管理员二次验证登录**

超级管理员登录时，除账号+密码+图形验证码外，还需通过短信验证码二次验证：

1. 超级管理员输入账号+密码+图形验证码，点击"登录"。
2. 系统验证账号+密码+图形验证码——若图形验证码错误，系统提示"验证码错误"，该账号的当日登录失败计数+1（与第1001条登录失败锁定规则联动）；若账号+密码错误，则按登录失败锁定规则累计计数。任意环节导致当日登录失败累计达10次，该账号临时锁定1小时。
3. 账号+密码+图形验证码均通过后，向超级管理员绑定的手机号发送6位短信验证码（有效期5分钟）。
3. 页面跳转至"短信验证码"输入页，显示"验证码已发送至您绑定的手机号***1234，请查收"。
4. 超级管理员输入6位验证码提交验证，验证通过后完成登录。
5. 验证失败3次后，验证码自动失效，需重新获取（点击"重新发送"），页面提示"验证码错误，请重新获取"。
6. 验证码获取需间隔60秒，防止短信轰炸。

- **会话安全与刷新机制**
    - **双 Token 机制**：系统签发 `Access Token`（有效期 2 小时）和 `Refresh Token`（有效期 7 天）。
    - **Token 绑定**：Token 签发时绑定当前客户端 IP、User-Agent Hash 及唯一设备指纹。
        - **普通管理员**：绑定 /24 子网，支持同一子网内的网络平稳切换（IP 段变化时静默刷新 Token）。
        - **超级管理员**：绑定 /24 子网网段（S0 级参数修改权限需极高安全性，结合 2FA 确保合规），IP 归属地大幅漂移时触发 Token 失效。
    - **IP 漂移与静默刷新**：
        - 普通管理员：若检测到 IP 段变更（/24 掩码变化）但 User-Agent 及设备指纹一致（常见于平稳网络切换），`Access Token` 失效，前端捕获 401 后自动调用 `Refresh Token` 接口。经验证通过后下发绑定新 IP 段的 Access Token，实现无感静默续期。
        - 超级管理员：若检测到 IP /24 子网发生变化，`Access Token` 与 `Refresh Token` 同步失效，强制退出并提示"登录网络环境变更，请重新认证"。
        - 若 IP 与设备指纹同时大幅变更，则 `Refresh Token` 同步失效，强制退出并重新登录。
- **活跃度检测**：操作间隔超过 30 分钟时，前端利用 `Refresh Token` 自动续期。
- **登出**：管理员主动点击"退出登录"后，Access/Refresh Token 均立即在服务端废弃。
- 异常登出（被顶替）：同一账号在另一设备登录时，旧会话自动失效，新会话生效，旧设备刷新页面后跳转至登录页并提示"您的账号已在其他设备登录"。

**账号创建（仅超级管理员可操作）**

2. 填写表单：管理员账号名（唯一，不可重复，用于登录）/ 管理员姓名（真实姓名）/ 手机号（用于短信验证和紧急联系）/ 角色（管理员 / 超级管理员）。
3. 系统自动发送初始密码至管理员手机号（或生成二维码供超级管理员当面告知）。
4. 新增账号操作记录入操作日志。

**账号修改（仅超级管理员可操作）**

- 变更角色：管理员→超级管理员 或 反之（降级需二次确认）。**角色变更时系统立即撤销该账号所有活跃的 Access Token 和 Refresh Token，强制其重新认证。**

**账号冻结/删除（仅超级管理员可操作）**

| 操作 | 效果 |
|-----|------|
| 冻结 | 账号无法登录；支持"批量任务转派"：在冻结前或冻结后，超级管理员均可点击该管理员账号的"批量转派"按钮，将该账号下所有待处理任务（资源审核/社区内容审核/举报处理/AI可疑内容）一次性转移至指定管理员。**系统硬约束：禁止冻结平台最后一名状态正常的超级管理员账号，前端操作时拦截并提示"禁止冻结最后一名超级管理员"** |
| 删除 | 账号永久物理删除。**系统硬约束：禁止删除平台最后一名状态正常的超级管理员账号**，操作时前端拦截并提示"禁止删除最后一名超级管理员"。**任务强制回收流转**：系统强制要求选择[将该账号名下全部待审、待办任务转移至指定人（下拉框选座）]，确认资产/任务去向后方可执行删除。操作记录永久可查。 |

> **审计要求**：冻结/删除操作均记录入操作日志，含操作人、被操作账号、操作时间、冻结/删除原因（必填）。

  - 标题："安全验证"
  - 内容："该操作需要二次验证，请输入发送至您手机的验证码"
  - 手机号显示："验证码已发送至***1234"
  - 输入框：6位数字验证码输入框（自动聚焦）
  - 按钮："确认"（灰化不可点击，直至输入6位数字）/ "取消"
  - 倒计时："重新发送({秒}s）"

Step 3：验证提交
  管理员输入6位验证码，点击"确认"。
  - 验证通过：执行封号操作，页面弹出Toast"操作成功"，弹窗关闭，用户状态变更为"永久封禁"。
  - 验证失败：输入框下方显示红色文字"验证码错误，请重新输入"，清除输入框，错误次数+1。
  - 失败达到3次：弹窗提示"验证失败次数过多，请{15}分钟后再试"，验证码失效，弹窗自动关闭。

Step 4：验证间隔与防轰炸
  - 每次点击"发送验证码"后，需间隔60秒才可再次发送。
  - 每个管理员每日（0:00~24:00）最多发送20次验证码，超限后当日不再发送，页面提示"今日验证次数已用完，请明日重试"。

Step 5：敏感操作锁定
  - 同一关键操作连续3次验证失败后，该操作入口临时锁定15分钟（不等同于账号锁定）。
  - 锁定期间，管理员无法再次触发该操作，页面提示"操作已锁定，请{15}分钟后再试"。
  - **注**：敏感操作验证码的失败计数与登录流程的失败计数**独立计算**，互不影响。

**验证码发送规则**

| 场景 | 发送条件 | 接收手机号 |
|-----|---------|-----------|
| 超级管理员登录 | 账号+密码+图形验证码全部通过后 | 超级管理员绑定手机号 |
| 关键操作二次验证 | 管理员点击"确认执行"触发验证弹窗后 | 当前登录管理员的绑定手机号 |
| 密码重置（管理员忘记密码） | 管理员通过"忘记密码"流程发起重置请求 | 管理员绑定手机号 |

> **短信验证码说明**：验证码为6位纯数字，有效期5分钟，每分钟最多发送1次，每操作日最多发送20次（防滥用）。验证码发送后实时写入日志（含手机号掩码/发送时间/操作类型）。

**操作日志强制记录**

所有后台操作（含管理员和超级管理员）均强制写入操作日志表，写入成功后返回操作结果，不存在"日志写入失败但操作成功"的场景。日志记录为只读，不可删除、不可修改、不可关闭。

**物理防篡改机制**：日志表采用哈希链（Hash Chaining）存储，每条日志记录包含当前记录的哈希值（由操作内容、时间戳、前一条日志哈希值计算生成）。任何对历史日志的篡改都会导致后续所有哈希校验失败，实现全链可追溯、可验证。

- **分段快照（Epochs）**：审计日志按月或季度形成独立哈希块，每块拥有独立的起始哈希值（由上一块末尾哈希值与本块起始时间戳生成）。历史归档块与当前块校验互不干扰，单块内部损坏不影响其他块的校验有效性。
- **白盒保护**：哈希计算与写入逻辑为系统硬编码或受保护模块（不在管理后台配置界面暴露），即使超级管理员也无法通过后台操作关闭或修改哈希链的核心校验逻辑。
