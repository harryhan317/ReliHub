## 一、文档目的与范围边界

### 1.1 文档目的

本文档用于定义 MVP 阶段 M0「启动与游客模式」的业务规则、触发条件、交互边界、配置项、验收标准和数据口径，确保产品、设计、研发、测试、运营在同一规则下执行并验收。

### 1.2 本文档覆盖范围

本文档覆盖以下能力：

- 欢迎页（展示、跳过、时长策略、结束跳转）
- 游客身份与权限体系（虚拟ID、额度限制、受限动作）
- 注册引导机制（触发条件、弹窗策略、早鸟奖励）
- 与登录注册模块的入口级衔接边界（仅边界，不展开登录注册细则）
- M0 相关后台可配置参数与生效规则
- M0 主责验收标准、埋点口径与反作弊要求

### 1.3 本文档不覆盖范围

以下内容由其他 PRD 主责，不在本文档展开：

- 登录注册页面详细交互与字段校验规则（见登录注册模块 PRD）
- 登录安全细则（连续错误锁定、协议勾选、异常登录检测）的实现细节
- 爱问/资源/社区/我的模块内部业务规则

## 二、模块边界与上下游

### 2.1 模块定位

M0 是 MVP 用户旅程的入口与分流层，承接「首次访问认知」和「游客体验转化」两大目标，核心任务是让未注册用户快速理解平台价值并完成注册转化。

### 2.2 上下游关系

- 上游输入：应用启动事件、设备访问状态（首次启动/7天内复访/超过7天未使用）
- 下游输出：游客态进入爱问首页、受限动作触发注册引导、跳转登录注册页面
- 配套系统：管理后台参数配置、可可豆与信誉分体系、埋点统计看板

### 2.3 与登录注册模块边界

- 本模块负责“何时引导去登录注册”与“从哪里进入登录注册”
- 登录注册模块负责“如何登录/注册成功”与“账号安全具体规则”
- 两模块通过来源参数和回跳参数保持流程连续

## 三、术语与规则基线

### 3.1 术语定义（本文使用）

- 游客模式：未注册用户的有限使用模式
- 快乐路径：用户按预期完成核心目标的端到端路径
- 异常路径：权限不足、额度超限、参数校验失败等非理想路径
- 受限动作：游客点击不可用功能（下载/点赞/评论/进入“我的”等）触发拦截
- 虚拟ID：系统为游客生成的临时身份标识，仅用于游客态会话与额度统计
- 游客退出：游客主动执行退出操作并确认退出，不等同于应用退到后台

### 3.2 规则来源优先级

本模块规则优先级如下：

1. `PRD_产品功能清单.md`（功能ID与阶段优先级基线）
2. `PRD_MVP_总体说明.md`（MVP边界与目标口径）
3. `PRD_总体方案_V1.1.md`（全局方案与阶段策略）
4. `PRD_可可豆与信誉分体系.md`（可可豆类别与奖励口径）
5. `PRD_术语表.md`、`PRD_非功能需求.md`（术语与非功能验收）

## 五、欢迎页规则（M0-F001~F004）

### 5.1 触发规则

欢迎页触发判定按以下优先级执行：

1. 首次安装后首次启动：展示欢迎页
2. 非首次启动且距离上次使用 ≤7天：不展示欢迎页
3. 非首次启动且距离上次使用 >7天：展示欢迎页

### 5.2 展示与交互

- 默认展示品牌 Logo 动画
- 支持双击或上划跳过
- 首次启动展示时长为 5 秒
- 长期未使用回访展示时长为 3 秒
- 用户主动跳过后，立即进入游客首屏，不再二次展示

### 5.3 名言轮换（S2）

- 欢迎页展示可靠性领域名言
- 名言内容采用预置词库随机轮换
- 词库由运营在后台维护，支持增删改
- 名言展示失败不阻塞欢迎页主流程，降级为仅展示 Logo 动画

### 5.4 页面结束跳转

- 欢迎页结束或被跳过后，进入游客模式
- 首屏固定为爱问主页初始状态

## 六、游客模式规则（M0-F005~F010）

### 6.1 游客身份分配

- 系统在游客首次进入时自动分配虚拟ID
- 虚拟ID用于游客额度统计与临时会话关联，不关联注册账号
- 游客历史会话采用临时存储，游客主动退出后清空

### 6.2 权限矩阵

| 能力域 | 游客权限 | 限制说明 | 超限/受限处理 |
|--------|----------|----------|----------------|
| AI问答 | 可用 | 每日可发起 3 个新会话 | 超限触发注册引导 |
| 资源浏览 | 只读 | 仅可查看 10 条精华资源 | 尝试查看第11条触发注册引导 |
| 社区浏览 | 只读 | 仅可查看 10 条精华话题 | 尝试查看第11条触发注册引导 |
| 互动能力 | 禁用 | 无下载/收藏/点赞/点踩/评论/回复/举报 | 点击即触发注册引导 |
| 创作能力 | 禁用 | 无法进行资源上传、发起话题 | 点击入口即触发注册引导 |
| 我的页面 | 禁用 | 无法进入"我的"页面 | 点击即触发注册引导 |

### 6.3 额度与会话规则

- AI 问答额度：每日 3 个新会话
- 轮次额度：单会话最大轮次默认 10、每日总轮次默认 15，可由后台配置
- 游客确认退出后，历史会话自动清空，不做持久化保存
- 每日额度重置时间以北京时间 00:00 为准

### 6.4 受限动作处理

游客触发受限动作时，系统按以下顺序执行：

1. 保持当前页面上下文
2. 弹出注册引导弹窗
3. 提供注册入口与返回当前页入口
4. 记录触发埋点与触发来源

## 七、注册引导与早鸟机制（M0-F011~F012）

### 7.1 触发条件

以下行为触发注册引导：

- 尝试查看第 11 条资源或话题（注：资源/话题列表采用限制拉取策略，非登录状态接口至多返回 10 条，列表底端展示固定的"登录后查看完整列表"引导组件）
- AI问答超出当日额度
- 点击创作相关入口（上传资源、发起话题）
- 点击受限功能（下载/收藏/点赞/评论/回复/进入"我的"等）

### 7.2 弹窗内容结构

- 价值点展示：注册收益、功能解锁、持续使用价值
- 操作按钮：微信一键注册（主）、手机号注册（次）、返回浏览（弱）
- 关闭弹窗：保留用户当前浏览位置，不破坏用户上下文
- 同一来源场景默认 60 秒内仅弹出一次（可由后台配置），避免重复打断

### 7.3 早鸟奖励规则

- 前 200 名注册用户额外奖励 **20 可可豆**（与注册奖励30可可豆为独立奖励，可叠加发放）
- 早鸟奖励归类为**福利豆**（与可可豆与信誉分体系口径一致：福利豆为赠送性质，不可取现）
- 若后续支持豆种配置，需在后台参数中明确默认值与可变范围

### 7.4 防重复发放与并发一致性

- 早鸟名额采用原子扣减，防止超发
- 奖励发放采用幂等机制，防止重复到账
- 并发冲突时以“名额占用成功”作为发奖前置条件
- 奖励链路失败时记录失败日志并进入补偿队列
- 具体实现（原子扣减算法、幂等保障方案、失败补偿队列）由登录注册模块定义

## 八、与登录注册模块的接口边界（M0-F013~F020）

### 8.1 入口边界

- 本模块仅定义进入登录注册页的触发与来源参数
- 登录注册成功判定、账号创建、协议校验由登录注册模块主责

### 8.2 透传参数建议

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| source_scene | string | 是 | 触发来源（resource_limit/community_limit/ai_limit/open_my） |
| return_tab | string | 否 | 注册成功后回跳Tab（ask/resource/community/my） |
| trigger_action | string | 是 | 触发动作标识（download/comment/open_my等） |
| trigger_time | string | 否 | 触发时间戳（ISO8601） |

### 8.3 回跳规则

- 注册成功后优先回到触发前业务场景
- 若原场景不可恢复，默认回到爱问首页

## 九、管理后台配置项与生效机制

### 9.1 配置项清单（M0相关）

| 配置项 | 默认值 | 配置范围 | 权限 |
|--------|--------|----------|------|
| 游客每日新会话数 | 3 | 0~1000 | 管理员需超级管理员授权 |
| 游客单会话轮次上限 | 10 | 0~100 | 管理员需超级管理员授权 |
| 游客每日总轮次上限 | 15 | 0~10000 | 管理员需超级管理员授权 |
| 资源/社区浏览条数上限 | 10 | 0~1000 | 管理员需超级管理员授权 |
| 早鸟名额 | 200 | 0~1000 | 管理员需超级管理员授权 |
| 早鸟奖励值 | 20 | 0~100 | 管理员需超级管理员授权 |
| 引导弹窗频控窗口（秒） | 60 | 0~3600 | 管理员需超级管理员授权 |
| 欢迎页时长策略 | 5s/7天跳过/3s | 规则可编辑 | 管理员需超级管理员授权 |

### 9.2 生效规则

- 文案类配置：实时生效
- 额度类配置：新计数周期生效（默认次日）
- 名额类配置：实时生效，但必须保证并发一致性控制
- 配置变更必须写入审计日志（配置人、变更前后值、生效时间）

## 十、状态机与核心流程定义

### 10.1 启动流程

应用启动 → 判定是否展示欢迎页 → 欢迎页展示/跳过 → 进入游客爱问首页

### 10.2 游客行为状态

游客浏览 → 正常可用能力执行  
游客浏览 → 触发受限动作 → 弹注册引导 → 注册/返回继续浏览

### 10.3 游客转注册流程

游客触发引导 → 进入登录注册页 → 注册成功 → 发放注册奖励与早鸟奖励（若满足条件）→ 回跳原场景

## 十一、埋点与指标口径

### 11.1 关键事件埋点

- welcome_show（欢迎页曝光）
- welcome_skip（欢迎页跳过）
- guest_limit_trigger（额度超限触发）
- guest_restricted_click（受限动作点击）
- register_guide_show（引导弹窗曝光）
- register_guide_click（引导按钮点击）
- guest_to_register_success（游客转注册成功）
- earlybird_reward_granted（早鸟奖励发放）

### 11.2 核心指标口径

- 游客转化率 = 游客转注册成功数 / 进入游客模式独立设备数
- 引导点击率 = 引导按钮点击数 / 引导弹窗曝光数
- 引导转化率 = 游客转注册成功数 / 引导按钮点击数
- 受限动作触发占比 = 受限动作触发次数 / 游客总会话次数
- 早鸟兑现率 = 早鸟奖励发放成功人数 / 符合早鸟条件注册人数

## 十二、验收标准（仅本文档主责）

### 12.1 功能验收

| 验收项 | 验收标准 |
|--------|----------|
| 欢迎页触发/跳过/时长/跳转 | 与第五章规则完全一致 |
| 游客权限矩阵 | 权限限制与触发处理与第六章一致 |
| 注册引导触发 | 第七章定义的触发场景100%触发引导 |
| 退出清空规则 | 游客确认退出后，历史会话不可恢复 |
| 早鸟奖励正确性 | 仅前200名发放，且每用户最多发放1次 |

### 12.2 性能验收（M0相关）

| 验收项 | 验收标准 |
|--------|----------|
| 欢迎页进入首屏时间 | 满足全局首屏性能目标（≤2秒） |
| 注册引导弹窗响应时间 | ≤500ms |

### 12.3 安全与反作弊验收（M0相关）

| 验收项 | 验收标准 |
|--------|----------|
| 虚拟ID生成防刷 | 同设备高频生成请求被限流或拦截 |
| 早鸟奖励防重 | 同一注册账号最多领取1次；同设备高频领取行为可被风控拦截 |
| 名额一致性 | 高并发下早鸟总发放人数不超过配置名额 |
| 审计可追溯 | 发放与拦截记录可按用户、设备、时间查询 |

### 12.4 边界说明

登录失败锁定、协议勾选、异常登录检测等登录安全验收项由登录注册模块 PRD 主责，不在本文档主责验收范围内。

## 十三、风险、反作弊与降级策略

### 13.1 核心风险

- 游客转化率低于目标
- 引导触发过频导致用户反感
- 早鸟奖励并发超发或重复发放

### 13.2 应对策略

- 优化价值文案与按钮优先级
- 引导频控策略（同场景短时不重复弹出）
- 幂等发奖 + 原子名额扣减 + 失败补偿机制
- 对可疑高频设备增加风控校验与限流

### 13.3 降级策略

- 早鸟系统异常时，自动关闭早鸟文案展示，仅保留常规注册奖励
- 引导系统异常时，降级为静态拦截提示并提供登录注册入口
