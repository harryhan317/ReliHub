## 一、模块目的与范围边界

### 1.1 文档目的
本模块（M2）是 ReliHub 平台"资源共享生态"的核心载体。本文档用于定义资源从**上传提报、AI+人工双重审核、上架展示、防泄露预览、到安全确权下载**的全生命周期业务规则，并规范其对应的**可可豆计费与通缩经济模型**，确保前后端实现与上下游规则一致。

### 1.2 涵盖范围 (对应功能清单 M2-F001~F027)
- **资源大厅展示与分类**：包括资源主页、空状态首发激励、分类浏览与全局搜索。
- **资源贡献提报流程**：表单提报、AI预审标定、人工终审闭环与申诉机制。
- **资源消费核心流程**：包括安全预览、扣费下载前置鉴权与异常回滚事务补偿。
- **通缩经济可可豆模型**：70/30 分成规则与扩充包折扣结算规则。
- **资源互动能力（S2 级）**：点赞/点踩互斥规则、评论防刷与分类生命周期管理。

### 1.3 不涵盖范围
- 下游可可豆原子级的充值支付接口对接与流水账单实体存储表设计（由 M4 可可豆资产模块主责）。
- 用 AI 对该篇资源文档进行深层次"对话解读"（明确归为 Phase 2 迭代）。
- 用户对侵权资源的法律追诉通道定义（由法务模块外源切入）。

## 二、核心业务流程与时序管控

### 2.1 资源贡献与 AI 预审时序
当贡献者点击【上传分享】时：
1. **输入阶段**：上传原文档，填充核心元数据（标题、简介、最多 5 个标签、可可豆售价 5~100,000 区间）。管理员可勾选"标记为基础资源"（`is_seed=true`），用户在权益额度内可免费下载。权益额度具体定义详见《PRD_可可豆与信誉分体系》§4.1.2 等级权益对比表。
2. **基础安全校验**：对文件名中的非法字符与 Emoji 进行规范化处理；文件上传到云端前，调用 ClamAV 引擎执行病毒与木马扫描，并记录审计日志。
3. **AI 智能预检与全量查重 (核心介入)**：
   - *双重比对逻辑*：
     - **物理比对（MD5）**：对全平台**所有状态**的存量资源进行瞬时 MD5 哈希比对。 (对齐 API_资源模块.md §2.1)
     - **语义比对（Vector）**：对全平台**已通过及待审核**的资源进行向量召回比对。
   - *重复判定与拦截*：当相似度超过配置阈值（默认80%，管理后台可配50%~99%）时，弹窗提醒用户（展示最多3条相似资源的标题/上传者/相似度），用户可选择[取消提交]或[仍然提交]。选择取消则流程结束（不记录、不扣分）；选择仍然提交则资源自动标记"🔁 疑似重复"标签进入审核队列。
   - **防秒刷阻断**：同一用户 7 天内（Redis 7天滑动窗口计数器）触发重复检测 ≥ 3 次，自动阻断提交能力。此规则旨在拦截高频并发提交相同内容绕过单步审核的漏洞。
   - *要素抽取与验证*：自动提取主旨摘要，比对用户填写分类与内容语义是否一致，输出分类建议。
   - *风险告警*：调用全局敏感词过滤器，命中时阻断提交并标记预警状态。
4. **预审报告展示 (M2-F009)**：向用户展示 AI 预审意见报告，包含重复度预警、分类一致性判断及风险标注。
5. **人工校验池 (M5工单)**：运营后台介入（状态：待审核）。**SLA：48 小时内处理完毕；超时触发自动告警并转入高优先级队列。**
6. **审批决策**：
   - 驳回：进入"被拒绝"状态，**系统即时推送通知（含驳回理由）**至贡献者（对齐《PRD_MVP_通知系统.md》§5.3）。进入"申诉中"后，贡献者可发起申诉沟通，或修改后重新提交审核。
   - 审核通过：变更为"已通过"状态并上架。**系统即时推送通知（含资源标题）**至贡献者。若该分类处于空状态且为【全平台首发奖励】，额外奖励 30 可可豆并发送【冷启动激励通知】。
   - **疑似重复资源决策**：①通过上架（正常通知）；②拒绝但不扣分（推送"拒绝（善意重复）"通知）；③拒绝并扣信誉分（推送"拒绝（恶意重复）"通知）。详见 M5 后台 §5.4.3。

### 2.2 资源消费鉴权与下发时序（防盗链约束）
*规则声明：下载鉴权必须采用闭环事务，避免抓包绕过。*
1. **触发与查账**：用户点击下载后，系统立即查询可可豆余额与历史下载记录。
   - *前置拦截*：若为新增付费下载且余额不足，则中断流程并提示"可可豆不足，可前往签到/发文获取或购买下载扩充包"。
2. **结算计算引擎**：执行全额购买扣费或使用下载扩充包额度（见第三章 3.3）。
3. **资金结算（原子事务）**：余额充足时执行扣款，并同步落库分账记录（贡献者 70%，系统销毁 30%）。
4. **安全封装下发**：
   - 生成有效期仅维持 `5 分钟` 的加密鉴权下载长 Token 链。
   - 调用底层服务将下载者的 UID 及当前时间生成不可见的数字盲水印并写入文件。
5. **异常容灾回滚机制**：
   - 若第 4 步 Token 下发失败，或终端因网络 `5xx` 未完成握手确认。
   - 启动补偿对账：退回已扣可可豆、恢复扩充包可用量，并向用户发送通知。

### 2.3 深度链接协议 (Deep Linking)
- **协议方案**：`relihub://resource/detail?id={resource_id}`
- **业务场景**：支持从【通知中心】、外部分享卡片直接跳转至指定资源详情页或下载页。
- **参数约束**：`resource_id` 必须为合法存量 ID，否则回跳至资源首页并 Toast 提示"资源不存在或已下架"。

## 三、可可豆通缩规则与商业约束

### 3.1 资产面价约束与分配法则（70/30 分账红线）
除被标记为"基础资源"且用户仍在权益额度内可免费获取的场景外，其余下载均按定价执行计费。若用户权益额度已耗尽，下载"基础资源"仍需按其定价支付可可豆（贡献者收益按 70% 发放，平台销毁 30%）。
- **定价范围**：资源定价区间为 5～100,000 可可豆。
- **结算精度与顺序**：以 1 可可豆为最小单位。所有分成与系数计算均执行 **`floor()` (向下取整)**，余下的零分由平台留存。各场景分配规则如下（具体计算逻辑详见 §2.2.2）：
  - **无折扣**：下载者按定价100%支付，贡献者获 $floor(定价 \times 0.7)$，系统销毁 $floor(定价 \times 0.3)$。
  - **特惠包（8.5折）**：下载者支付 $floor(定价 \times 0.85)$，系统销毁 $floor(定价 \times 0.15)$，贡献者获 $floor(定价 \times 0.7)$。
  - **畅享包（7折）**：下载者支付 $floor(定价 \times 0.7)$，系统销毁 0，贡献者获 $floor(定价 \times 0.7)$。
- **提现边界**：MVP 阶段不接入提现能力；金可可仅用于账面展示，取现开关在 Phase 2 默认关闭。

### 3.2 扩充包折扣规则
用户可预先购买下载扩充包以获得折扣，折扣场景仍执行同一分配规则：
- **下载特惠包（100MB限额 / 80可可豆 / 计价 8.5折）**。
- **下载畅享包（500MB限额 / 350可可豆 / 计价 7折）**。
- 具体结算分配规则及分账示例详见《PRD_可可豆与信誉分体系》§2.2.2 资源相关。
- **容量余量规则**：扩充包剩余容量不足单次文件大小时，允许超额下载 1 次（不额外扣费，视为扩充包含用量已耗尽）。

### 3.3 重下载规则："1 年内 2 次免费"
*注：以下规格限制均服从 M5 后台参数配置优先原则，当前给定的数值为出厂默认启动值。*
1. **首次下载**：按规则完成扣费并创建重下载凭证：`ExpireTime = 当前时间 + 365*24 小时`（可配置），初始 `Retries = 2`（可配置）。
2. **第 2 次重下载（且处于 ExpireTime 有效期内）**：校验凭证有效 -> 扣费 0 -> `Retries` 变更为 1。
3. **第 3 次重下载（且处于 ExpireTime 有效期内）**：校验凭证有效 -> 扣费 0 -> `Retries` 变更为 0（免费重下载次数耗尽）。
4. **第 4 次下载**（或超过有效期后的任意下载）：按新下载处理并重新扣费；成功后重建新的 `ExpireTime` 与 `Retries = 2`。

## 四、功能详解：大厅检索与热度排序

### 4.1 空态大厅的首发引流策略
当某个细分维度（如：失效分析类）由于冷启动零库存呈现"空类目"状态时；
- **常驻展示策略**：空分类卡片不隐藏，保留常驻入口。
- **首发激励提示语**：空分类常驻引导"该分类暂无资源，欢迎首发贡献，审核通过后可获得 30 可可豆奖励。"

### 4.2 热度计算规则
每个资源拥有一个实时计算或离线计点的热度值 `Heat_Index`，公式权重如下：
- **基础公式**（由本模块定义）：`Heat_Index = MAX(0, (download_count * 10) + (like_count * 5) - (dislike_count * 8) + (comment_count * 3) + (distinct_views * 1))`
- **应用场景**：本模块大厅列表排序采用基础公式；搜索结果排序将在基础公式上叠加 $Aging\_Factor$ 等修饰符（详见 `PRD_MVP_搜索系统.md` §4.2）。
- **字段风格**：统一采用 snake_case 命名以适配 PostgreSQL 规范。
- 补点系数 `C`：**上架新鲜度时间衰减。**

### 4.3 浏览量防刷机制
*注：以下规格限制均服从 M5 后台参数配置优先原则，当前给定的数值为出厂默认启动值。*
- 基于 `鉴权UserID + 设备Fingerprint` 进行去重统计。24 小时内同一用户（或游客设备）对同一资源的重复访问仅计 1 UV。
- **单日浏览量上限**：同一用户每日对同一资源的浏览次数上限为 **20 次**，超出时返回限流提示。

## 五、功能详解：上传约束、申诉机制与状态老化

### 5.1 上传规格约束
*注：以下规格限制均服从 M5 后台参数配置优先原则，当前给定的数值为出厂默认启动值。*
- 文件大小上限：默认支持单文件最大 **20 MB**。
- 支持格式白名单：默认涵盖 txt、doc/docx、pdf、excel(xls/xlsx)、PPT/PPTX 及主流图像簇 PNG/JPG/BMP/GIF/JPEG/TIF。
- 并发控制：默认同一终端最多 3 路并发上传，超出时返回限流提示。

### 5.2 状态流转与编辑矩阵
- **待审核/申诉中可编辑**：资源处于"待审核"或"申诉中"时，贡献者可执行修改、撤回、重新提交。
- **申诉沟通机制**：被拒绝资源支持发起申诉留言，管理员回复后进入再审流程。

### 5.3 资源展示标签与退出机制
对齐全局内容生命周期规则（详见《PRD_内容生命周期管理.md》§2.3、§2.4）：

**展示标签**（适用于公开资料列表）：
1. **"最新（7天）"标记**：资源上线后 7 天内展示"最新"标签
2. **"精华"标记**：由 M5 管理员人工评定并添加"精华"标签
3. **"基础资源"标记**：由管理员标注（`is_seed=true`），具备面值定价。在对应等级用户的权益额度内下载时不消耗可可豆、不产生贡献者收益；超额下载 or 非权益覆盖时按定价交易。资源详情页展示"基础资源"徽标。
4. **"闲置"标记**：资源超过 1 年**无新增下载且无新增评论**时，标记为"闲置"，搜索系统对"闲置"资源执行降权（系数 `0.3`），详见 `PRD_MVP_搜索系统.md` §4.2

**退出机制**：
5. **下架**：连续 2 年闲置，或违规举报核实成立时，管理员执行"下架"。**下架资源在用户端不可见，仅在管理后台可见并标记为"已下架"**

## 六、功能详解：安全预览与下载追溯

### 6.1 预览防截图隔离渲染（*S2级功能*，MVP可选实现）
- **渲染引擎**：采用 pdf.js 与自研渲染图层。
- **明水印规则**：服务端叠加访问者身份水印（`{User_Name}+{Current_UID}+{Timestamp}`）。
- **缓存隔离规则**：若启用 Redis 缓存，Cache Key 必须包含 `Doc_ID_Hash::::Viewer_UID_Hash`，防止跨用户水印串用。

### 6.2 可配置预览页数
- 默认预览前 N 页（默认值为 2 页）。
- 管理员在审核阶段可对单资源配置更高预览页数。

### 6.3 下载追溯标记
- 下载文件除时效 Token 外，需写入数字盲水印以支持泄露溯源与违规处置。

## 七、功能详解：互动能力（S2）

资源详情页除下载功能外，也承载基础互动能力：

### 7.1 点赞/点踩互斥规则
- 点赞与点踩互斥，支持撤回。

### 7.2 评论准入规则
- 评论最小字数默认 5 字（可配置）。
- 仅允许已有效下载用户评论；对 0 成本开放资源可放开评论准入。

### 7.3 收藏
- 用户可收藏资源，收藏记录归档至 M4 个人中心。

### 7.4 分享（微信生态）
- 支持一键分享至微信生态，内容包含标题与封面图。
- 分享奖励分佣能力不在 MVP 范围，归入 Phase 2。

## 八、权限边界与拦截

- **非登录游客（M0）受限边界**：游客可浏览前 10 条精华资源并查看详情页元数据；不可执行点赞、点踩、收藏、评论、下载、上传等操作。触发受限动作时弹出注册引导。
- **违规用户拦截（信誉分熔断处理）**：基于信誉分规则，低于阈值的用户将被关闭上传权限。越权请求返回统一拦截提示 `[Error: 403] 您的账号因近期违规记录已被限制资源权限，解封时间请查看通知中心。`，并由后端错误码统一拦截。

## 十、测试验收与 NFR 指标

测试团队需覆盖以下业务红线与 NFR 核心性能基准。

### 10.1 核心业务验证用例
| 验证场景 | 预期结果 |
|-------------|---------------------|
| **防重复扣费** | 首次下载扣费后，1 年内第 2/3 次重下载不扣费；超过有效期后再次下载按新下载扣费。 |
| **分账对账一致性**   | 定价 200、折后实付 170 的场景下，销毁与贡献者收益按统一舍入规则计算，账务对账一致。 |
| **并发预览隔离** | 用户 A、B 同时预览同一资源时，水印身份与缓存内容严格隔离，不出现串用。 |

### 10.2 NFR 核心性能指标
以下指标以《PRD_非功能需求.md》为准，作为发布阻断红线：
- **资源列表加载**：加载 20 条资源卡片，目标值 ≤ 1 秒。
- **资源详情加载**：加载资源元数据与简介，目标值 ≤ 500ms。
- **资源搜索返回**：关键字搜索结果返回，目标值 ≤ 1 秒。
- **资源预览加载/翻页**：文档预览加载 ≤ 3 秒，翻页响应 ≤ 500ms。
- **下载与上传**：下载开始响应 ≤ 1 秒；20MB 文件上传过程具备实时进度反馈并满足上位文档接口要求。
- **计费一致性**：并发下载场景下，扣费与分账事务满足 ACID 一致性要求。
