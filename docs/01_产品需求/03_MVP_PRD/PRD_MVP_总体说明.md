## 一、MVP概述

### 1.1 MVP定位与目标

#### 1.1.1 MVP核心定位

**ReliHub MVP（Minimum Viable Product）** 是 ReliHub 产品的最小可行版本，旨在用最小的开发成本验证产品核心价值，快速获取市场反馈，为后续迭代奠定基础。

**核心定位**：
- **验证产品价值**：验证"可靠性问题一站式解决方案"的产品定位是否成立
- **积累种子用户**：吸引并留存首批种子用户，建立用户基础
- **沉淀初始内容**：通过用户贡献积累资源库与社区内容
- **打磨核心体验**：优化AI问答、资源共享、社区讨论三大核心功能

#### 1.1.2 与Phase 2/3的关系

```
Phase 1 (MVP)          Phase 2                Phase 3
核心功能验证    →    模块联动与商业化    →    生态完善与扩展
    ↓                      ↓                      ↓
AI问答基础功能        AI推荐资源/话题         专家认证与咨询
资源上传下载          充值/取现功能           广告系统
社区发帖回复          微信小程序迁移          语音输入
```

#### 1.1.3 MVP成功标准

| 指标类型 | 具体指标 | 目标值 | 说明 |
|---------|---------|--------|------|
| **用户规模** | 注册用户数 | ≥1000人 | 种子用户成功导入 |
| | 游客转化率 | ≥60% | 进入游客模式的独立设备/用户数中，完成注册的比例 |
| **内容沉淀** | 资源库资源数 | ≥200个 | 覆盖8大分类 |
| | 社区话题数 | ≥100个 | 覆盖10大分类 |
| **用户活跃** | DAU/MAU | ≥20% | 日活/月活比例 |
| | 次日留存率 | ≥40% | 新用户次日回访 |
| **功能使用** | AI对话完成率 | ≥80% | 发起对话后完成问答 |
| | 资源下载量 | ≥500次/月 | 平均每资源被下载2.5次 |
| **系统性能** | 首屏加载时间 | ≤2秒 | 所有主页面 |
| | AI响应时间 | ≤1秒 | 文字输入后首字输出 |
| | 系统可用率 | ≥99.9% | 月度统计 |

> 注：以上数值为建议目标值，实际目标由运营团队根据市场情况调整。详细指标定义见 [PRD_总体方案_V1.1.md](../02_总体方案PRD/PRD_总体方案_V1.1.md)。

### 1.2 核心用户

#### 1.2.1 游客用户

**定义**：首次访问平台，尚未注册的潜在用户。

**核心诉求**：
- 快速了解平台能做什么
- 无门槛体验核心功能（AI问答/资源浏览/社区浏览）
- 明确注册后的价值收益

**权限边界**：
- AI问答：每日3个新会话（具体轮次限制详见 [PRD_可可豆与信誉分体系.md](../02_总体方案PRD/PRD_可可豆与信誉分体系.md)§4.1.2）
- 资源浏览：仅可查看10条精华资源，无下载权限
- 社区浏览：仅可查看10条精华话题，无互动权限
- 历史会话：基于虚拟ID临时存储，游客退出系统后历史对话自动清空，不做持久化保存

#### 1.2.2 注册用户

**定义**：已完成注册流程并通过基础校验的用户，具备持久化账号与基础权限。

**等级体系（全阶段定义）**：新兵 → 菜鸟 → 入门 → 熟手 → 老炮 → 达人 → 专家

> 说明：MVP阶段生效的是通用等级权益；专家等级在MVP仅作为等级结构预留，专家展示/申请/咨询动作均不在Phase 1。

**核心诉求**：
- 快速解决工作中遇到的可靠性技术问题
- 获取标准规范/器件手册/应用笔记等专业资料
- 与同行交流经验，学习最佳实践
- 通过贡献内容建立个人技术影响力

**等级权益差异化**：等级越高，AI问答次数/免费轮次、附件能力、历史会话容量、基础资源/权益额度内下载次数越高。详细权益配置见 [PRD_可可豆与信誉分体系.md](../02_总体方案PRD/PRD_可可豆与信誉分体系.md)。

#### 1.2.3 管理员与超级管理员

**超级管理员 (Super Admin)**
- 平台最高权限管理人员，负责系统配置、权限管理、紧急封禁等操作
- 可进行参数配置、管理管理员账号、导出运营数据

**管理员 (Admin)**
- 日常运营管理人员，负责内容审核、用户管理、数据监控
- 参数配置类权限需超级管理员授权

> 详细管理员权限定义见 [PRD_总体方案_V1.1.md](../02_总体方案PRD/PRD_总体方案_V1.1.md)。

### 1.3 MVP交付形态

| 维度 | 说明 |
|------|------|
| **产品形态** | Web应用（模拟微信小程序，iPhone 12 Pro尺寸） |
| **技术栈** | React + Vite + TypeScript（前端）/ FastAPI（后端）/ PostgreSQL（主库）/ Redis（缓存） |
| **部署方式** | 采用Docker容器化部署 |
| **管理后台** | 独立PC端Web管理系统 |
| **预计周期** | 3-4个月 |

## 二、功能范围（按模块概述）

> 注：本章节为MVP功能概要说明，功能全集与详细规则以 [PRD_产品功能清单.md](../02_总体方案PRD/PRD_产品功能清单.md)、[PRD_总体方案_V1.1.md](../02_总体方案PRD/PRD_总体方案_V1.1.md) 及相关专项PRD为准；MVP模块详细PRD按计划后续拆分产出。

### 2.1 M0 启动与游客模式（S0级）

| 功能 | 说明 |
|------|------|
| 欢迎页展示 | 首次安装/长期未使用时展示品牌Logo动画，支持双击/上划跳过 |
| 游客权限体系 | 系统自动分配虚拟ID，AI问答每日3个新会话，资源/社区仅浏览10条热门 |
| 注册引导机制 | 当触发超限或受限操作时弹出价值驱动型引导（如尝试查看第11条内容、超出AI问答额度、点击下载/点赞/评论/“我的”等受限功能），早鸟用户（前200名）额外奖励20可可豆 |
| 登录/注册 | 微信一键登录、手机号密码/验证码登录/注册 |

### 2.2 M1 爱问模块（S0级）

| 功能 | 说明 |
|------|------|
| AI对话核心功能 | 文字输入，流式输出（S0），多轮对话，基于LLM通用知识回答 |
| 附件上传 | 核心上传能力为S0；上传入口交互与附件解读为S1级扩展功能 |
| 历史对话管理 | 基础能力（S0）：按时间倒序展示、继续对话；扩展能力：搜索/删除（S1）、重命名（S2） |
| 对话规则 | 按等级配置轮次/Token/存储限制，详见可可豆与信誉分体系文档 |
| 专业领域限制 | 仅回答可靠性相关问题，无关问题礼貌婉拒 |

### 2.3 M2 资源模块（S1级）

| 功能 | 说明 |
|------|------|
| 资源上传 | 填写信息 → AI预审 → 人工审核，支持修改/撤回/重新提交 |
| 资源浏览与搜索 | 8大分类，二级筛选（最新/热门/精华/基础），关键字搜索 |
| 资源下载与预览 | 基础资源/权益额度内免费下载；超额下载可按定价消耗可可豆，或购买下载扩充包享受折扣；首次下载后1年内可免费重下载2次，超过后按新下载规则处理 |
| 资源互动 | 收藏、点赞/点踩、点评（均为S2级） |

> 可可豆消费机制详见 [PRD_可可豆与信誉分体系.md](../02_总体方案PRD/PRD_可可豆与信誉分体系.md)。

### 2.4 M3 社区模块（S1级）

| 功能 | 说明 |
|------|------|
| 话题发起 | 标题/内容/分类/关键词，可选设置悬赏可可豆 |
| 话题浏览与搜索 | 10大分类，二级筛选（最新/精华/悬赏/解决），话题状态标签（新话题/待回答/讨论中/已解决），关键字搜索 |
| 话题互动 | 回复、点赞/点踩、关注、举报，管理员可置顶话题 |
| 悬赏机制 | 发起人设置悬赏可可豆，采纳最佳回答者获得全部悬赏，无人回答或未采纳时30天自动退回 |

### 2.5 M4 我的模块（S1级）

| 功能 | 说明 |
|------|------|
| 个人信息展示与编辑 | 头像/昵称/单位/职务，按等级区分必填/选填项 |
| 可可豆与信誉分展示 | 展示用户核心资产（余额、明细）和信用等级（信誉分、等级进度），是用户成长体系在个人中心的直接体现，信誉分仅用户本人可见 |
| 我的贡献 | 查看上传的资源列表、发起的话题列表 |
| 日常操作 | 每日签到获得可可豆、邀请好友获得奖励 |
| 系统设置 | 账号安全、通知设置、意见反馈、账号注销 |

> S2级功能：我的收藏与关注（收藏的资源、关注的话题）、我的回复、获赞数统计。
>
> 可可豆与信誉分详细规则见 [PRD_可可豆与信誉分体系.md](../02_总体方案PRD/PRD_可可豆与信誉分体系.md)。
>
> **Phase 2/3 规划**：充值功能、金可可/福利豆区分展示、取现功能（详见第五章Phase 2/3边界说明）。

### 2.6 M5 管理后台（S0级）

| 功能 | 说明 |
|------|------|
| 系统看板 | 核心数据展示（用户数/对话数/资源数/话题数），DAU/MAU/留存率等 |
| 参数配置 | 按等级配置AI问答限制、功能权限、可可豆与信誉分规则，以及资源上下限定价区间；AI模型/System Prompt版本化管理与回滚 |
| 内容审核 | 资源审核（48小时内完成）、社区内容审核、申诉处理、举报处理 |
| 用户管理 | 用户列表、违规处理（警告/扣分/封号）、账号解锁 |
| 安全管理 | 反爬虫配置、反刷可可豆配置、内容安全API配置、限流规则 |
| 页面显示设置 | 资源/社区分类显示顺序调整、分类隐藏/显示 |

### 2.7 横向支撑系统（跨模块）

| 系统 | 功能说明 |
|------|---------|
| 统一搜索系统 | 爱问/资源/社区专属搜索，关键字精准/模糊匹配，联想词提示 |
| 通知系统 | 系统通知、互动通知（回复/点赞/采纳）、审核通知 |
| 可可豆与信誉分体系 | 获取/消费/通缩机制，等级权益差异化（核心规则为S0/S1，扩展能力随阶段演进） |
| 文件处理系统 | 上传/下载/预览/水印/存储周期管理 |
| 内容生命周期管理 | 资源/话题/用户账号状态流转、退出机制，详见专项PRD |
| 意见反馈系统 | 表单反馈、处理流程、满意度评价 |

> 优先级归属：统一搜索/文件处理（S2），通知/反馈/内容生命周期（S3），可可豆与信誉分核心规则（S0/S1）。

## 三、开发优先级

### 3.1 优先级划分原则

| 优先级 | 定义 | 包含模块 |
|--------|------|---------|
| **S0级** | 最高优先级，核心必做 | 启动与游客、登录注册、爱问核心功能、管理后台核心功能、可可豆与信誉分核心规则 |
| **S1级** | 高优先级，核心必做 | 资源模块、社区模块、我的模块核心功能 |
| **S2级** | 中优先级，配套必做 | 搜索、文件处理、性能优化 |
| **S3级** | 低优先级，基础完善 | 通知、反馈、内容生命周期 |

### 3.2 开发顺序建议

| 阶段 | 时间 | 开发内容 |
|------|------|---------|
| **第一阶段** | 1-1.5个月 | M0启动与游客、登录注册、M1爱问核心功能、M5管理后台基础功能 |
| **第二阶段** | 1-1.5个月 | M2资源模块、M3社区模块、M4我的模块核心功能 |
| **第三阶段** | 0.5-1个月 | 横向支撑系统（搜索、通知、文件处理）、性能优化、自动化测试 |

## 四、验收标准

### 4.1 功能验收标准

| 用户角色 | 验收标准 |
|---------|---------|
| **游客模式** | 游客可正常体验AI问答/资源浏览/社区浏览，超限后触发注册引导 |
| **注册用户** | 可完成AI问答/资源上传下载/社区发帖回复，按等级享受差异化权益 |
| **管理员** | 可完成系统配置/内容审核/用户管理，参数配置类权限需超级管理员授权 |
| **核心流程** | 所有核心功能主流程与异常路径测试通过 |

> 主流程路径与异常路径定义见 [PRD_术语表.md](../02_总体方案PRD/PRD_术语表.md)。

### 4.2 性能验收标准

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首屏加载时间 | ≤2秒 | 所有主页面（爱问/资源/社区/我的） |
| AI响应时间 | ≤1秒 | 文字输入后首字输出 |
| 接口响应时间 | ≤500ms | 查询类接口 |
| 系统可用率 | ≥99.9% | 月度统计 |
| 并发用户数支持 | ≥20人 | MVP阶段 |

> 详细性能指标与测试方法见 [PRD_非功能需求.md](../02_总体方案PRD/PRD_非功能需求.md)。

### 4.3 安全验收标准

| 机制 | 验收标准 |
|------|---------|
| 账号安全 | 连续错误锁定、异常登录检测生效 |
| 内容安全 | 敏感词过滤、AI内容审核生效 |
| 数据安全 | 加密存储、备份恢复机制生效 |
| 反作弊 | 反爬虫/反刷可可豆机制生效 |

### 4.4 测试验收标准

| 类型 | 验收标准 |
|------|---------|
| 单元测试 | 覆盖率 ≥80% |
| 接口测试 | 覆盖所有API端点 |
| E2E测试 | 覆盖核心业务流程 |
| 性能/安全测试 | 通过 |

## 五、与Phase 2/3的边界说明

### 5.1 MVP范围内（Phase 1）

- 基础功能实现：AI问答、资源上传下载、社区发帖回复、个人中心、管理后台
- 游客模式与注册用户权限体系
- 可可豆与信誉分体系基础功能（获取/消费/通缩/等级）
- 统一搜索系统（基于PostgreSQL全文检索）
- 文件处理系统（上传/下载/预览/水印）

### 5.2 MVP范围外（Phase 2）

| 功能 | 说明 |
|------|------|
| 模块联动 | AI推荐资源/话题、资源AI解读、社区@AI助手 |
| 商业化功能 | 充值功能、金可可/福利豆区分、取现功能 |
| 分享激励 | 分享至微信、分享落地页、分享奖励 |
| 搜索升级 | Elasticsearch替代PostgreSQL全文检索 |
| 小程序迁移 | Web应用迁移为原生微信小程序 |

### 5.3 MVP范围外（Phase 3）

| 功能 | 说明 |
|------|------|
| 专家系统 | 专家认证、一对一咨询、实时聊天 |
| 广告系统 | 广告管理、广告数据统计 |
| 高级功能 | 语音输入、分类拖拽排序、版权声明确认 |

### 5.4 边界判断原则

- MVP聚焦核心价值验证，不做模块联动
- MVP不涉及充值/支付/取现等商业化功能
- MVP不涉及专家系统、广告系统等高级功能
- MVP采用Web应用形态，Phase 2迁移为原生小程序

## 六、关键风险与应对

### 6.1 技术风险

| 风险 | 应对措施 |
|------|---------|
| AI响应时间可能超过1秒 | 优化LLM调用性能，采用流式输出，提前加载常用数据（Phase 3 RAG上线后补充知识库检索优化） |
| 并发用户数可能不足20人 | 采用Redis缓存热点数据，优化数据库查询，采用CDN加速静态资源 |

### 6.2 业务风险

| 风险 | 应对措施 |
|------|---------|
| 游客转化率可能低于60% | 优化注册引导文案，增加早鸟奖励吸引力，降低注册门槛 |
| 用户留存率可能低于40% | 优化AI问答质量，增加资源库内容，提升社区活跃度 |

### 6.3 时间风险

| 风险 | 应对措施 |
|------|---------|
| 开发周期可能超过4个月 | 严格按优先级开发，S0/S1级功能优先，S2/S3级功能可延后 |
