 ## 一、模块目的与范围边界
 
 ### 1.1 文档目的
 本模块负责统一管理用户的身份准入机制。主要目标是将游客身份持久化为真实注册用户（支持分发早鸟与注册奖励），并为老用户提供安全快捷的登录体验。同时构筑账号管理与系统安全的第一道防线。
 
 ### 1.2 涵盖范围 (对应清单 M0-F013~F020)
 *   **登录注册途径**：微信一键登录/注册（获取授权）、手机号验证码快捷登录/注册、手机号+密码登录。
 *   **账号安全恢复**：忘记密码（手机验证码身份核验与重置）。
 *   **基础安全防御**：账号错误锁定机制、强制强密码校验、用户协议前置阻断、异常登录预警（异地/异设备）、验证码频控限流治理。
 *   **辅助流转动作**：用户个人信息引导填报（注：本模块仅负责注册成功后的弹窗与入口跳转，信息完善与可可豆数据展示由 M4"我的模块"主责）。
 
 ### 1.3 不涵盖范围
 *   注册引导弹窗机制（由《PRD_MVP_启动与游客模式.md》主责）。
 *   可可豆消费与扣减接口（由可可豆与信誉分模块提供接口契约）。
 *   复杂的第三方登录绑定解绑逻辑（此为 Phase 2 演进目标）。

 ## 二、模块衔接与参数消费规则
 
 ### 2.1 游客引导进入参数消费
 承接《启动与游客模式》的导航请求，登录注册页面必须具备消费透传参数的能力：
 
 | 参数名 | 释义 | 消费与回跳应用规则 |
 |---|---|---|
 | `source_scene` | 触发来源 (如 ai_limit/resource_create_limit/topic_create_limit) | 用于埋点归因，不干预前端渲染路径。（覆盖问答、创作、互动界限） |
 | `trigger_action` | 触发受限动作 (如 download/upload_resource/create_topic，对应上传资源/发起话题等) | 登录成功后自动重放。**拦截器与次态验证**：重放请求发出前，必须重新进行 JWT 鉴权 + 业务权限判断（如是否够扣可可豆） + 资源状态校验（如高频重复点击时资源是否已被删），避免强制重放导致越权漏洞。 |
 | `return_tab` | 回跳目标 Tab (如 my) | 当 `trigger_action` 回调冲突或失效时，作为第二优先级的默认回退层进行 Tab 定向。 |
 | `trigger_time` | 请求发出的时间 | 用作过期防重放判断，超过设定时间阈值（如15分钟）的拦截跳回首页处理。 |
 
 **回跳优先级冲突处理：**
 优先触发 `trigger_action` 原地动作  > 次优跳转 `return_tab` 场景 > 若均缺失则回到 AI 问答首页。
 
 ### 2.2 可可豆体系与早鸟联动边界
 *   在判定账号为**新注册**的原子事务中，通过后端事件机制调用可可豆子系统，发放**注册奖励 30 可可豆（福利豆）**。
 *   **早鸟并发争抢奖励（20 福利豆）下发防御**：
 *   **防超发**：名额采用原子扣减规则，确保不超发。
 *   **防重复**：奖励下发必须具备严格幂等性，避免重复到账。
 *   **容错与对账**：发放失败时支持底层重试机制，超限后记录至异常补偿队列日志供人工核对（技术实现细节由研发架构阶段定夺）。

 ## 三、验证码治理与安全规范 (重点)
 
 ### 3.1 手机号边界与验证码发送管控（防刷）
 1.  **手机号归属地校验规则**：当前业务仅支持中国大陆 **+86** 手机号注册/登录。非大陆号码由前端正则拦截并提示"当前仅支持中国大陆手机号"；后端返回对应 `AUTH_` 错误码。
 2.  **防刷冷却机制**：同一个手机号请求发送验证码后，本地及服务端严格冻结 **60 秒**，期间请求拦截器拦截并提示"获取过于频繁，请稍后再试"。
 3.  **验证码并发治理（多维度防护）**：
     *   **手机号维度**：单日上限 10 次（每日北京时间 00:00 重置计数）。
     *   **设备指纹维度**：单日上限 10 次。
     *   **IP归属维度**：单 IP 1 分钟内请求超过 3 次时触发图形验证码（注：如 MVP 阶段暂不接入图形验证码服务，可降级为业务报错拦截，图形验证码顺延至 Phase 2）。
     *   **执行优先级**：手机号、设备、IP 三维并行计数，以"最先触达阈值（最严格条件）"的规则为准执行熔断/拦截。
 
 ### 3.2 短信验证码生命周期
 *   验证码长度：6位纯数字。
 *   有效时长：自发送成功起 **5 分钟** 内有效。
 *   验证后失效：一次性核销，验证成功即时作废。

 ## 四、核心业务流程与功能详解
 
 ### 4.1 M0-F013 微信一键登录规则
 *   **流程**：点击底部微信授权图标 → 请求微信开放平台鉴权 → 获取 OpenID / unionId / 昵称 / 头像。
 *   **账号绑定合规**：获取授权后，依据合规要求自动跳转"绑定手机号"页面（手机号+验证码）。
 *   **头像与昵称流转**：绑定成功时，若为全新用户，则系统默认取用微信下发的头像与昵称作为该用户 ReliHub 的初始档案资料。
 *   **绑定冲突处理**：若绑定手机号时发现该手机号已绑定其他微信账号，或当前微信 UnionID 已绑定其他手机号，系统返回互斥错误提示（如"该手机号已绑定其他微信账号"）并终止本次绑定，提示用户使用原关联账号登录。（注：MVP 阶段暂不提供解绑/换绑流程，相关诉求由人工客服处理或顺延至 Phase 2。）
 
 ### 4.2 M0-F014 手机快捷登录与 M0-F015 账密登录
 *   **单页聚合表单交互**：采用统一入口，第一步输入手机号，第二步根据用户选择分流（获取验证码或输入密码）。
 *   **自动判定注册与密码设置**：用户输入号码及验证码后，若不存在于数据库则静默注册。验证码登录成功后，若为新注册用户，系统应弹出"是否设置安全密码"的可选提示框，用户可选择"暂不设置"直接进入平台，也可跳转去设置；跳过后可在 M4 模块随时补充设置。
 *   **密码强校验规则 (M0-F015)**：在设置密码或重置密码时，执行统一安全正则校验：
 *   **规则**：8~20位，**必须包含**大小写字母、数字、特殊符号中的至少 2 项（严格对齐总体方案原始需求红线）。
 
 ### 4.3 M0-F017 账号锁定与风控动作
 *   **触发条件**：单账号使用 **密码方式** 登录连续发生 5 次失败（验证码不计入密码错误）。
 *   **惩罚策略**：账号登录通道强制锁定 **15 分钟**。前端弹出"密码错误次数过多，请15分钟后再试或通过验证码登录"。
 *   **提前解锁的例外场景处理**：
     *   **未完成验证码核验前**：15 分钟内拒绝所有密码登录尝试。
     *   **通过手机短信验证码成功登录后**：系统立即解除该账号的密码锁定状态并恢复凭证有效性（建议随后强提醒并引导用户跳转修改密码页）。
 
 ### 4.4 M0-F020 异常登录检测
 *   **触发判定条件**（注：判定需依赖管理后台配置的第三方 IP 解析库与设备指纹 SDK 进行风险标记）：
     *   **异地登录**：当前登录 IP 归属地与 `last_login_ip` 跨省变化。
     *   **新设备登录**：当前设备指纹与 `last_login_device` 不匹配。
 *   **异常处理动作**：不阻断主流程，但通过系统通知及关联手机号发送安全预警短信："您的账号刚刚在 [地区]/[设备] 登录，非本人操作请立即修改密码"。
 *   **白名单防扰**：同一新设备成功鉴权登录后 30 天内不再重复触发该新设备的预警。
 
 ### 4.5 注册成功后的信息完善衔接 (M0-F019)
 *   **流转动作**：对于静默注册（手机验证码）或微信首次绑定的全新用户，在下发 JWT Token 登录系统后，前端根据 `is_new_user` 标识弹出"完善个人档案"引导卡片（非跨页阻断，支持右上角跳过）。
 *   **边界约束**：此卡片内要收集的具体字段（必填/选填）、保存请求等内部流转逻辑，统一定义在 M4"我的模块"PRD 中，本页面仅负责时机触发。
 
 ### 4.6 忘记密码流转规则 (M0-F016)
 *   **触发入口**：在密码登录面板点击"忘记密码"唤起独立验证流程。
 *   **流转步骤**：
     1.  **账号校验**：输入绑定的手机号，下发验证码。
     2.  **身份核验**：提交手机号加短信验证码核销，进入重置面板。
     3.  **密码重置**：遵循 `M0-F015` 强安全校验规则重新设置密码并确认提交。
 *   **重置完成后处理**：重置成功后，系统强制注销该账号现有的全部失效 Token（强制下线其他已登录设备），并引导用户返回密码登录入口重新登入。
 
 ### 4.7 用户协议前置拦截规约 (M0-F018)
 *   **展示位置**：聚合表单登录/注册大按钮的紧下方，默认 **不勾选**。
 *   **交互阻断**：未勾选状态下点击获取验证码、密码登录、微信登录等请求动作，前端阻断并标红并抖动提示。
 *   **后端强校验**：若请求绕过前端到达服务端，认证网关须校验协议字段；缺失时返回 `AUTH_4005` 并阻断。
 *   **协议版本与持久化**：用户成功鉴权后，服务端记录该用户的 `agreement_timestamp` 快照。若法律合规要求重签新版协议，M5 及后端可依据该时间戳触发全站强制要求重新签署协议。

 ## 六、错误码与前端提示文案规范
 
 建立统一的中台登录接口契约，约定 `AUTH_4xxx` 对应客户端传递的参数与状态拒绝，`AUTH_5xxx` 对应服务端异常/熔断降级。前端严格根据以下枚举映射友好文案：
 
 | 场景 | 服务端业务码 (Code) | 前端推荐文案 (仅供参考) | 后续交互要求 |
 |---|---|---|---|
 | 发送频率倒计时 | `AUTH_4001` | "验证码获取过于频繁，请稍后再试" | 锁定按钮 60s |
 | 号码被日限流 | `AUTH_4002` | "当前号码/设备今日请求达上限" | 置灰发送按钮 |
 | 密码错误且未锁定 | `AUTH_4003` | "密码错误，您还有 N 次机会" | 允许继续输入 |
 | 密码连续错误锁定 | `AUTH_4004` | "账户出于安全防护已锁定15分钟" | 展示切换验证码登录入口 |
 | 协议未勾选 | `AUTH_4005`（合规阻断） | "为了您的权益，请阅读并勾选许可协议"| 协议 Checkbox 本地置红+抖动。说明：后端必须拦截未勾选状态，防止绕过前端直接请求。 |

 ## 七、数据存储审计与安全基线要求
 
 ### 7.1 传输与存储安全基线 (P0级约束)
 为保障用户凭证安全，必须满足以下要求：
 *   **传输层加密**：所有登录与鉴权相关接口强制使用 **HTTPS**。
 *   **密码安全落库**：禁止明文存储或使用低强度哈希算法存储，必须使用 **BCrypt (cost=12)** 及以上的加盐哈希算法。
 *   **日志严格脱敏**：后端网关与应用层日志中，严禁打印涉及密码、完整手机号等敏感字段的明文。
 *   **Token 生命周期管理**：采用双 Token 机制，即 Access Token（24小时）+ Refresh Token（30天）。仅在无感刷新失败时前端跳回登录页，降低会话超时带来的数据丢失风险。（注：具体 Token 策略以《PRD_非功能需求.md》为准）
 *   **多端并发登录管控**：MVP 阶段默认允许多端同时在线（暂不执行同账号异地互踢）。当发生"忘记密码重置"或"账号锁定"等高风险事件时，由后端统一使该账号全部有效 Token 失效并执行强制下线。
 
 ### 7.2 核心审计字段设计
 为了支持异常检测（M0-F020）和后续管理后台排查，用户账户（User_Auth表层）必须挂载以下审计字段：
 *   `last_login_ip`：最后一次成功的公网 IP 记录。
 *   `last_login_device`：最后一次成功的核心设备指纹摘要。
 *   `failed_login_attempts`：当前密码错误累加计数。
 *   `lock_until_time`：账户风控锁定过期时间戳。
 *   `agreement_timestamp`：核心用户协议最后一次授权同意时刻（用于强重签防线）。

 ## 八、与管理后台（M5）系统接口约束
 
 登录平台不包含后台管理界面，仅须定义向 M5 暴露的原子服务接口：
 1.  **强制下线接口契约**：接收指定 UID 和指令，使当前全部有效 Token 失效并清空端侧缓存，完成强制下线。
 2.  **后台账号解封接口**：仅向管理后台暴露接口，允许管理员通过 M5 指令式一键重置 `failed_login_attempts` 计步器并清除锁定状态。（注：此接口的主操作页面逻辑及管控全由 M5 主责定义和校验）
 3.  **配置拉取接口**：定时读取 M5 落库的安全验证动态阈值（如错误阈值、验证码单日上限等）。

 ## 九、测试矩阵与验收标准
 
 要求按量化表进行验证（主流程、异常路径、风控路径、并发路径）。
 （注：本表侧重于查验登录业务流程的闭环逻辑；关于加密强度、并发防刷压测等技术测试红线，请依循关联文档《PRD_非功能需求.md》作为唯一基准。）
 
 | 测试域 | 验收场景描述 | 验收断言标准 | 主责角色 |
 |---|---|---|---|
 | **常规登录** | （主流程路径）手机验证码/微信/密码登录 | 顺利下发 JWT Token，各分支交互正常流转。 | 测试工程师 |
 | **非法边界** | （异常路径）非+86号码、伪造重放参数或 `trigger_time` 过期 | 后端阻断请求；过期参数直接丢弃并降级回首页。 | 测开/测试 |
 | **合规阻断校验** | （安全专项）绕过前端勾选，直接调用网关发起登录请求 | `AUTH_4005` 后端拒绝下发 Token 并记录越权违规日志。 | 安全测试 |
 | **高并发一致性校验** | （并发路径）百级并发展现早鸟注册动作 | 奖励原子扣减不超发；断网重发具备幂等机制避免重复入账。 | 研发+测试 |
 | **验证码限流校验** | （异常风控）短时间内尝试海量拉取验证码 | 手机号、IP、设备三维限流触达阈值即拦截；高频由图形验证阻挡。| 研发+安全测试 |
 | **错误锁定校验** | （异常风控）连续 5 次错误密码录入 | 服务端强拒绝第 6 次（无论正确与否），15 分钟内无法通过密码登入。 | 测试工程师 |
