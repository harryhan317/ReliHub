# ReliHub 产品需求文档 (PRD)
 
 [⬅️ 返回文档门户](../README.md)

本文档说明 ReliHub 项目的文档目录结构。

> 说明：本文件中的大目录树是历史规划稿，包含尚未落地的阶段文档。  
> 当前仓库实际已落地文档以“当前已落地目录”章节为准。

## 项目概述

ReliHub 是面向电子产品可靠性领域技术人员的 AI 社区平台，采用 React + FastAPI + PostgreSQL 技术栈开发。

---

## 文档分层关系

- 输入基线：`ReliHub 产品需求输入文件 V2.2.md`
- 总体规则层：`02_总体方案PRD/`（术语、功能清单、非功能、专项规则）
- 阶段落地层：`03_MVP_PRD/`（MVP阶段模块化PRD）
- 演进原则：上层定义边界，下层细化执行；下层不得与上层冲突
