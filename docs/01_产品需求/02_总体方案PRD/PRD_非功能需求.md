## 一、文档说明

### 1.1 文档目的

本文档定义 ReliHub 平台的非功能需求,包括性能指标、安全要求、兼容性约束、可靠性标准等,为产品设计、技术实现、测试验收提供明确的量化标准。

### 1.2 适用范围

- **产品团队**: 理解产品质量标准与用户体验底线
- **技术团队**: 架构设计、技术选型、性能优化的依据
- **测试团队**: 性能测试、安全测试、兼容性测试的验收标准
- **运维团队**: 系统监控、告警阈值、容量规划的参考

### 1.3 文档结构

```
第一章: 文档说明
第二章: 性能需求 (响应时间/吞吐量/并发/资源占用)
第三章: 安全需求 (账号/数据/内容/接口/防护)
第四章: 可靠性需求 (可用性/容错/恢复/数据一致性)
第五章: 兼容性需求 (浏览器/设备/网络/第三方集成)
第六章: 可维护性需求 (日志/监控/部署/文档)
第七章: 可扩展性需求 (架构/数据/业务)
第八章: 用户体验需求 (交互/视觉/无障碍)
第九章: 合规性需求 (法律/隐私/内容审核)
第十章: 验收标准与测试方法
```

## 二、性能需求

### 2.1 响应时间要求

> 所有指标为**最低要求**,开发与测试阶段必须满足,管理后台可监控性能数据。

#### 2.1.1 页面加载性能

| 页面类型 | 性能指标 | 目标值 | 验收标准 |
|---------|---------|--------|---------|
| **首屏加载** | 首屏可见时间 (FCP) | ≤ 2秒 | 所有主页面(爱问/资源/社区/我的) |
| | 首屏完全加载 (LCP) | ≤ 3秒 | 包含图片/卡片/列表数据 |
| **子页面加载** | 页面切换时间 | ≤ 1秒 | 详情页/编辑页/设置页 |
| **列表滚动** | 滚动加载延迟 | ≤ 500ms | 资源列表/话题列表/历史对话 |
| **图片加载** | 缩略图加载 | ≤ 1秒 | 列表页卡片缩略图 |
| | 大图预览加载 | ≤ 3秒 | 资源预览/图片详情 |

**测试条件:**
- 网络环境: 4G网络 (下行4Mbps, 上行1Mbps, 延迟100ms)
- 设备基准: iPhone 12 Pro / 同等性能Android设备
- 测试工具: Lighthouse / WebPageTest

#### 2.1.2 AI对话性能

| 功能场景 | 性能指标 | 目标值 | 验收标准 |
|---------|---------|--------|---------|
| **文字问答** | 首字响应时间 (TTFB) | ≤ 1秒 | 用户发送消息后AI首字输出 |
| | 流式输出速度 | ≥ 5字/秒 | 持续输出速度 |
| | 单轮问答完成时间 | ≤ 30秒 | 纯LLM生成（Phase 3扩展RAG后包含检索耗时维持此指标） |
| **附件解读** | 附件上传响应 | ≤ 3秒 | 文件上传完成确认 |
| | 附件解读首字响应 | ≤ 5秒 | AI开始输出解读内容 |
| | 附件解读完成时间 | ≤ 60秒 | 包含文件解析+内容分析 |
| **历史对话** | 对话列表加载 | ≤ 1秒 | 加载30条历史对话 |
| | 对话详情加载 | ≤ 500ms | 加载单个对话完整内容 |

**性能优化策略:**
- 采用 Server-Sent Events (SSE) 实现流式输出
- Phase 3 RAG上线后补充 RAG 检索结果缓存策略 (Redis, TTL=1小时)
- 历史对话分页加载 (每页20条)
- 附件异步处理 (上传后立即返回,后台解析)

#### 2.1.3 资源模块性能

| 功能场景 | 性能指标 | 目标值 | 验收标准 |
|---------|---------|--------|---------|
| **资源浏览** | 资源列表加载 | ≤ 1秒 | 加载20条资源卡片 |
| | 资源详情加载 | ≤ 500ms | 加载资源元数据+简介 |
| **资源搜索** | 搜索结果返回 | ≤ 1秒 | 关键字搜索返回结果列表 |
| | 搜索联想提示 | ≤ 300ms | 输入关键字后联想词提示 |
| **资源预览** | 文档预览加载 | ≤ 3秒 | PDF/Word文档在线预览 |
| | 预览翻页响应 | ≤ 500ms | 预览页面切换 |
| **资源下载** | 下载开始响应 | ≤ 1秒 | 点击下载后开始下载 |
| | 下载速度 | ≥ 1MB/s | 20MB文件 ≤ 20秒完成 |
| **资源上传** | 上传速度 | ≥ 500KB/s | 20MB文件 ≤ 40秒完成 |
| | 上传进度反馈 | 实时更新 | 每秒更新上传进度 |

**性能优化策略:**
- 资源列表虚拟滚动 (仅渲染可见区域)
- 缩略图CDN加速 + WebP格式
- 文档预览采用 pdf.js 懒加载
- 文件上传分片上传 (每片5MB)
- 下载链接预签名 (OSS直传,避免后端转发)

| | 点赞/收藏响应 | ≤ 300ms | 点击后UI立即反馈 |
| **评论加载** | 评论列表加载 | ≤ 1秒 | 加载50条评论 |
| | 评论分页加载 | ≤ 500ms | 滚动加载下一页评论 |

**性能优化策略:**
- 话题列表分页加载 (每页20条)
- 评论树状结构扁平化存储
- 点赞/收藏乐观更新 (先更新UI,后台异步提交)
- 精华话题Redis缓存 (TTL=10分钟)

#### 2.1.5 管理后台性能

| 功能场景 | 性能指标 | 目标值 | 验收标准 |
|---------|---------|--------|---------|
| **数据看板** | 看板加载时间 | ≤ 3秒 | 加载所有核心指标图表 |
| | 图表刷新时间 | ≤ 2秒 | 切换时间范围/筛选条件 |
| **内容审核** | 审核列表加载 | ≤ 1秒 | 加载50条待审核内容 |
| | 审核操作响应 | ≤ 500ms | 通过/拒绝操作提交 |
| **用户管理** | 用户列表加载 | ≤ 2秒 | 加载100条用户记录 |
| | 用户搜索响应 | ≤ 1秒 | 按昵称/手机号搜索 |
| **参数配置** | 配置页面加载 | ≤ 1秒 | 加载所有配置项 |

- 数据看板采用异步加载 (图表独立请求)
- 审核列表分页加载 (每页50条)
- 用户列表虚拟滚动
- 配置项分组加载 (按模块拆分)

### 2.2 吞吐量与并发要求

#### 2.2.1 系统并发能力

| 阶段 | 并发用户数 | 峰值QPS | 日活用户 (DAU) | 验收标准 |
|------|-----------|---------|---------------|---------|
| **Phase 1 (MVP)** | ≥ 20人 | ≥ 100 | 100-500 | 系统稳定运行,响应时间不超标 |
| **Phase 2** | ≥ 500人 | ≥ 2000 | 2000-10000 | 水平扩展后满足要求 |
| **Phase 3** | ≥ 2000人 | ≥ 5000 | 10000-20000 | 分布式架构支撑 |

**测试方法:**
- 使用 Locust / JMeter 进行压力测试
- 模拟真实用户行为 (浏览/搜索/AI问答/上传下载)
- 监控指标: CPU使用率/内存占用/数据库连接数/响应时间

#### 2.2.2 接口性能要求

| 接口类型 | 响应时间 | 吞吐量 (TPS) | 错误率 | 验收标准 |
|---------|---------|-------------|--------|---------|
| **查询类接口** | ≤ 500ms | ≥ 500 | ≤ 0.1% | 列表查询/详情查询/搜索 |
| **写入类接口** | ≤ 1秒 | ≥ 200 | ≤ 0.1% | 发布话题/上传资源/提交反馈 |
| **AI对话接口** | 首字 ≤ 1秒 | ≥ 50 | ≤ 1% | SSE流式输出（基于 LLM） |
| **文件上传接口** | ≤ 3秒 | ≥ 50 | ≤ 0.5% | 20MB文件上传 |
| **文件下载接口** | ≤ 1秒 | ≥ 100 | ≤ 0.5% | 预签名URL生成 |

**性能优化策略:**
- 接口响应时间 P95 ≤ 目标值的1.5倍
- 接口响应时间 P99 ≤ 目标值的2倍
- 数据库查询优化 (索引/慢查询监控)
- Redis缓存热点数据 (用户信息/配置项/热门内容)

### 2.3 资源占用要求

#### 2.3.1 服务器资源

| 服务类型 | CPU使用率 | 内存使用率 | 磁盘I/O | 网络带宽 |
|---------|----------|-----------|---------|---------|
| **Web服务** | ≤ 70% | ≤ 80% | ≤ 60% | ≥ 100Mbps |
| **API服务** | ≤ 70% | ≤ 80% | ≤ 60% | ≥ 100Mbps |
| **数据库** | ≤ 60% | ≤ 70% | ≤ 70% | ≥ 100Mbps |
| **Redis缓存** | ≤ 50% | ≤ 60% | - | ≥ 100Mbps |

**监控告警:**
- CPU使用率 > 80% 持续5分钟 → 告警
- 内存使用率 > 90% → 告警
- 磁盘使用率 > 85% → 告警
- 网络带宽使用率 > 80% → 告警

#### 2.3.2 客户端资源

| 资源类型 | 限制要求 | 验收标准 |
|---------|---------|---------|
| **前端包体积** | ≤ 2MB (gzip压缩后) | 首次加载JS+CSS总大小 |
| **内存占用** | ≤ 200MB | 浏览器标签页内存占用 |
| **本地存储** | ≤ 50MB | LocalStorage + IndexedDB |
| **网络流量** | ≤ 5MB/会话 | 平均单次访问流量消耗 |

**优化策略:**
- 代码分割 (Code Splitting) + 懒加载
- 图片压缩 + WebP格式 + CDN加速
- 静态资源缓存 (Cache-Control: max-age=31536000)
- 接口数据压缩 (gzip/brotli)

### 2.4 数据库性能要求

#### 2.4.1 查询性能

| 查询类型 | 响应时间 | 验收标准 |
|---------|---------|---------|
| **主键查询** | ≤ 10ms | 单表主键查询 |
| **索引查询** | ≤ 50ms | 带索引的条件查询 |
| **复杂查询** | ≤ 200ms | 多表JOIN/聚合查询 |
| **全文检索** | ≤ 500ms | PostgreSQL全文检索 |

**优化策略:**
- 所有查询字段建立索引
- 慢查询监控 (> 1秒记录日志)
- 定期执行 VACUUM ANALYZE
- 读写分离 (主库写,从库读)

#### 2.4.2 数据库容量规划

| 数据类型 | 预估增长 | 存储策略 |
|---------|---------|---------|
| **用户数据** | 10万用户/年 | 热数据保留,冷数据归档 |
| **AI对话** | 100万条/年 | 保留最近1年,超期归档（注：此为总对话产生量，含已删除/归档；因用户等级限制历史会话保存个数，实际在线存储量会小于此值） |
| **资源文件** | 10TB/年 | OSS存储,数据库仅存元数据 |
| **社区内容** | 50万条/年 | 全量保留,定期清理违规内容 |

## 三、安全需求

### 3.1 账号安全

#### 3.1.1 身份认证

| 安全措施 | 实现要求 | 验收标准 |
|---------|---------|---------|
| **密码强度** | 8-20位,包含大小写字母/数字/特殊符号中的2种 | 注册/修改密码时强制校验 |
| **密码存储** | BCrypt加密 (cost=12) | 数据库不存储明文密码 |
| **登录失败锁定** | 连续5次错误锁定15分钟 | 管理后台可配置次数和时长（管理员需超级管理员授权） |
| **会话管理** | JWT Token (有效期24小时) | 支持刷新Token机制 |
| **异地登录检测** | IP地址/设备指纹变化 → 通知用户 | 系统通知+短信/邮件 |
| **强制登出** | 修改密码后强制登出所有设备 | 清除所有会话Token |

#### 3.1.2 多因素认证 (Phase 2)

| 认证方式 | 实现要求 | 适用场景 |
|---------|---------|---------|
| **手机验证码** | 6位数字,有效期5分钟 | 登录/注册/修改密码/敏感操作 |
| **微信授权** | 微信小程序授权登录 | 快速登录 |
| **邮箱验证** | 邮件验证码,有效期10分钟 | 找回密码/绑定邮箱 |

### 3.2 数据安全

#### 3.2.1 数据加密

| 数据类型 | 加密方式 | 验收标准 |
|---------|---------|---------|
| **用户密码** | BCrypt (cost=12) | 不可逆加密 |
| **敏感信息** | AES-256-GCM | 手机号/身份证/银行卡 |
| **传输加密** | HTTPS (TLS 1.2+) | 所有接口强制HTTPS |
| **文件加密** | OSS服务端加密 (SSE-KMS) | 上传文件自动加密 |

#### 3.2.2 数据备份

| 备份类型 | 备份频率 | 保留周期 | 验收标准 |
|---------|---------|---------|---------|
| **全量备份** | 每日凌晨3点 | 保留30天 | 自动备份到OSS |
| **增量备份** | 每小时 | 保留7天 | 数据库binlog备份 |
| **关键数据** | 实时同步 | 永久保留 | 用户数据/交易记录 |

**恢复要求:**
- RTO (恢复时间目标): ≤ 4小时
- RPO (恢复点目标): ≤ 1小时
- 每季度进行一次备份恢复演练

#### 3.2.3 数据脱敏

| 数据类型 | 脱敏规则 | 适用场景 |
|---------|---------|---------|
| **手机号** | 138****5678 | 日志/前端展示 |
| **身份证** | 110***********1234 | 日志/前端展示 |
| **邮箱** | a***@example.com | 日志/前端展示 |
| **IP地址** | 192.168.***.*** | 日志记录 |

### 3.3 内容安全

#### 3.3.1 敏感词过滤

| 过滤场景 | 实现方式 | 处理策略 |
|---------|---------|---------|
| **用户输入** | 实时检测 (前端+后端双重校验) | 发现敏感词 → 提示用户修改 |
| **AI对话** | AI助手内置敏感词检测 | 拒绝回答+记录日志+通知管理员 |
| **资源上传** | 文件内容扫描 (OCR+文本提取) | 发现敏感内容 → 拒绝上传 |
| **社区内容** | 发布前检测+定期巡检 | 发现违规 → 隐藏内容+通知用户 |

**敏感词库:**
- 政治敏感词 (国家安全/政治人物/敏感事件)
- 色情暴力词 (色情/暴力/血腥)
- 违法犯罪词 (毒品/赌博/诈骗)
- 广告垃圾词 (联系方式/推广链接)
- 管理后台可自定义敏感词库（管理员需超级管理员授权）

#### 3.3.2 内容审核机制

| 审核类型 | 审核方式 | 审核时效 | 处理流程 |
|---------|---------|---------|---------|
| **资源审核** | AI预审 + 人工审核 | 48小时内 | AI预审报告 → 管理员审核 → 通过/拒绝 |
| **话题审核** | AI实时检测 + 人工抽查 | 实时 | 敏感词过滤 → 自动发布 → 定期抽查 |
| **评论审核** | AI实时检测 + 用户举报 | 实时 | 敏感词过滤 → 自动发布 → 举报后人工审核 |
| **用户举报** | 人工审核 | 24小时内 | 用户举报 → 管理员审核 → 处理违规内容 |

**审核标准:**
- 符合《网络信息内容生态治理规定》
- 符合《互联网信息服务管理办法》
- 符合微信小程序内容安全规范

#### 3.3.3 微信内容安全API集成

| 检测类型 | API接口 | 检测内容 | 处理策略 |
|---------|---------|---------|---------|
| **文本检测** | security.msgSecCheck | 用户输入/AI回复/话题内容 | 违规 → 拒绝发布 |
| **图片检测** | security.imgSecCheck | 用户上传图片/头像 | 违规 → 拒绝上传 |
| **音频检测** | security.mediaCheckAsync | 语音输入 (Phase 3) | 违规 → 拒绝发布 |

### 3.4 接口安全

#### 3.4.1 接口鉴权

| 鉴权方式 | 实现要求 | 适用场景 |
|---------|---------|---------|
| **JWT Token** | Header: Authorization: Bearer <token> | 所有需要登录的接口 |
| **API Key** | Header: X-API-Key: <key> | 第三方集成接口 (Phase 2) |
| **签名验证** | 请求参数+时间戳+密钥 → MD5签名 | 支付回调/敏感操作 |

**Token管理:**
- Access Token有效期: 24小时
- Refresh Token有效期: 30天
- Token刷新机制: Access Token过期前15分钟自动刷新
- Token黑名单: 用户登出/修改密码后Token加入黑名单

#### 3.4.2 接口限流

| 限流类型 | 限流规则 | 限流算法 | 超限处理 |
|---------|---------|---------|---------|
| **业务层限流** | 防止用户过度消费资源 | 固定窗口 | 提示用户"操作过于频繁" |
| | 游客: 每日3个AI对话 | | |
| | 注册用户: 每日10-30个AI对话 (按等级，具体次数详见「PRD_可可豆与信誉分体系.md §4.1.2」) | | |
| **技术层限流** | 防止恶意刷接口 | 令牌桶算法 | 返回429 Too Many Requests |
| | 单IP: 每分钟100次请求 | | |
| | 单用户: 每分钟50次请求 | | |
| | 单接口: 每秒10次请求 | | |

**限流配置:**
- Redis存储限流计数器
- 管理后台可动态调整限流阈值（管理员需超级管理员授权）
- 白名单机制 (超级管理员/管理员/测试账号不限流)

#### 3.4.3 防护措施

| 攻击类型 | 防护措施 | 实现方式 |
|---------|---------|---------|
| **SQL注入** | 参数化查询 | SQLAlchemy ORM + 参数绑定 |
| **XSS攻击** | 输入过滤+输出转义 | DOMPurify + CSP策略 |
| **CSRF攻击** | CSRF Token | 表单提交携带Token |
| **DDoS攻击** | CDN防护+限流 | 阿里云/腾讯云DDoS防护 |
| **爬虫防护** | User-Agent检测+验证码 | 频繁访问 → 触发验证码 |
| **刷可可豆** | 行为分析+异常检测 | Redis记录操作行为,异常 → 封号 |

### 3.5 文件安全

#### 3.5.1 文件上传安全

| 安全措施 | 实现要求 | 验收标准 |
|---------|---------|---------|
| **文件类型校验** | 白名单机制 (仅允许指定格式) | 前端+后端双重校验 |
| **文件大小限制** | 单文件 ≤ 20MB | 超限 → 拒绝上传 |
| **文件内容检测** | 病毒扫描 (ClamAV) | 发现病毒 → 拒绝上传 |
| **文件名过滤** | 禁止特殊字符 (../ / \ : * ? " < > |) | 自动过滤或拒绝上传 |
| **加密文件检测** | 检测加密PDF/压缩包 | 提示用户"不支持加密文件" |

#### 3.5.2 文件存储安全

| 安全措施 | 实现要求 | 验收标准 |
|---------|---------|---------|
| **存储隔离** | 用户文件独立存储路径 | OSS Bucket分目录存储 |
| **访问控制** | 私有Bucket + 预签名URL | 文件不可直接访问 |
| **水印保护** | 预览/下载添加水印 | 明水印(用户名+ID)+盲水印 |
| **防盗链** | Referer校验 + Token验证 | 非平台域名无法访问 |
| **定期清理** | 临时文件24小时清理 | 定时任务自动清理 |

## 四、可靠性需求

### 4.1 系统可用性

#### 4.1.1 可用性指标

| 指标 | 目标值 | 计算方式 | 验收标准 |
|------|--------|---------|---------|
| **系统可用率** | ≥ 99.9% | (总时间-故障时间)/总时间 | 月度统计 |
| **MTBF (平均故障间隔)** | ≥ 720小时 | 总运行时间/故障次数 | 季度统计 |
| **MTTR (平均修复时间)** | ≤ 2小时 | 总故障时间/故障次数 | 季度统计 |

**可用性保障:**
- 99.9%可用率 = 每月允许故障时间 ≤ 43.2分钟
- 核心服务双机热备 (主备切换时间 ≤ 30秒)
- 数据库主从复制 (延迟 ≤ 1秒)
- 负载均衡 (Nginx + 多实例部署)

### 4.2 容错能力

#### 4.2.1 服务降级策略

| 降级场景 | 降级策略 | 用户体验 |
|---------|---------|---------|
| **AI服务异常** | 返回预设回复 + 提示稍后重试 | "AI助手暂时繁忙,请稍后重试" |
| **搜索服务异常** | 降级为数据库LIKE查询 | 搜索速度变慢,但功能可用 |
| **OSS服务异常** | 禁用文件上传/下载 | 提示"文件服务维护中" |
| **Redis缓存异常** | 直接查询数据库 | 响应时间变慢,但功能可用 |
| **第三方API异常** | 跳过非核心功能 | 微信登录失败 → 提示使用手机号登录 |

#### 4.2.2 熔断机制

| 熔断对象 | 熔断条件 | 熔断时长 | 恢复策略 |
|---------|---------|---------|---------|
| **AI接口** | 连续10次失败 或 错误率>50% | 60秒 | 半开状态测试1次,成功→恢复 |
| **第三方API** | 连续5次超时 或 错误率>30% | 30秒 | 半开状态测试1次,成功→恢复 |
| **数据库** | 连接池耗尽 或 响应时间>5秒 | 10秒 | 自动重连 |

### 4.3 数据一致性

#### 4.3.1 事务一致性

| 业务场景 | 一致性要求 | 实现方式 |
|---------|-----------|---------|
| **可可豆交易** | 强一致性 | 数据库事务 (ACID) |
| **资源下载** | 强一致性 | 扣除可可豆+记录下载 → 原子操作 |
| **话题悬赏** | 强一致性 | 扣除悬赏+发布话题 → 原子操作 |
| **点赞/收藏** | 最终一致性 | 异步更新计数器 |

#### 4.3.2 分布式一致性 (Phase 2)

| 场景 | 一致性策略 | 实现方式 |
|------|-----------|---------|
| **缓存一致性** | Cache-Aside模式 | 更新数据库 → 删除缓存 |
| **主从同步** | 异步复制 | 主库写 → 从库读 (延迟≤1秒) |
| **分布式锁** | Redis分布式锁 | 防止重复扣款/重复下载 |

### 4.4 故障恢复

#### 4.4.1 故障检测

| 检测方式 | 检测频率 | 告警阈值 | 告警通道 |
|---------|---------|---------|---------|
| **健康检查** | 每30秒 | 连续3次失败 | 短信+邮件+钉钉 |
| **性能监控** | 每1分钟 | CPU>80% / 内存>90% | 短信+邮件 |
| **错误日志** | 实时 | ERROR级别日志 | 钉钉群通知 |
| **业务监控** | 每5分钟 | 核心指标异常 (如AI对话成功率<90%) | 短信+邮件 |

#### 4.4.2 故障恢复流程

```
故障发生
    ↓
自动检测 (健康检查/监控告警)
    ↓
自动恢复 (服务重启/主备切换)
    ↓
恢复失败?
    ↓ 是
人工介入 (运维团队处理)
    ↓
故障修复
    ↓
服务恢复
    ↓
故障复盘 (分析原因+改进措施)
```

**恢复时间目标:**
- 自动恢复: ≤ 5分钟
- 人工恢复: ≤ 2小时
- 数据恢复: ≤ 4小时

## 十、验收标准与测试方法

### 10.1 性能测试

#### 10.1.1 测试方法

| 测试类型 | 测试工具 | 测试场景 | 验收标准 |
|---------|---------|---------|---------|

| **稳定性测试** | Locust / JMeter | 持续运行24小时 | 无内存泄漏 / 无性能衰减 |
| **前端性能测试** | Lighthouse | 测试首屏加载/FCP/LCP | 所有指标达标 |

| 数据类型 | 数据量 | 数据来源 |
|---------|--------|---------|
| **用户数据** | 10万条 | 脚本生成 |

| **资源数据** | 1万条 | 脚本生成 + 真实数据 |
| **社区数据** | 10万条 | 脚本生成 |

### 10.2 安全测试

#### 10.2.1 测试方法

| **渗透测试** | 人工测试 | 模拟黑客攻击 | 无法突破防护 |
| **代码审计** | SonarQube | 代码安全问题 | 无高危问题 |
| **依赖检查** | Snyk / npm audit | 第三方依赖漏洞 | 无高危漏洞 |
