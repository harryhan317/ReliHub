## 一、产品定位与目标

### 1.1 产品定位

**ReliHub** 是面向电子产品可靠性领域技术人员的专业AI社区平台,定位为"可靠性问题的一站式解决方案"。

**核心定位:**
- **领域专注**: 聚焦电子产品可靠性全链路(设计/测试/质量/工艺/失效分析)
- **AI驱动**: 基于LLM通用知识的专业AI助手(ReliBot)提供7×24小时技术支持（Phase 3扩展RAG专业知识库）
- **社区共建**: 构建"我为人人、人人为我"的资源共享与知识交流生态
- **价值闭环**: 以可可豆🫘为核心的通缩型经济模型保障贡献者长期收益

**产品形态演进:**

| 阶段 | 产品形态 | 核心目标 | 预计周期 |
|------|---------|---------|---------|
| **Phase 1 (MVP)** | Web应用(模拟微信小程序) | 验证产品价值,积累种子用户与内容 | 3-4个月 |
| **Phase 2** | 原生微信小程序 | 深度集成微信生态,实现模块联动与商业化 | 2-3个月 |
| **Phase 3** | 功能完善与生态扩展 | 专家认证/一对一咨询/广告系统 | 持续迭代 |

**技术形态:**
- Phase 1: React + FastAPI + PostgreSQL,以iPhone 12 Pro尺寸为设计基准
- 管理后台: 独立PC端Web管理系统
- 部署方式: Docker容器化,一键启动

### 1.2 核心目标

**四大核心目标:**

1. **汇聚专业人才**: 聚集电子产品可靠性领域全链路技术人员与专家,建立行业人脉网络
2. **沉淀专业知识**: 通过AI助手+资源库+社区讨论,系统化沉淀可靠性领域知识与经验
3. **提升解决效率**: 用户遇到可靠性问题时,30分钟内获得初步解决方案或专业建议
4. **促进技术传播**: 加速可靠性标准、最佳实践、工具方法在行业内的普及应用

**MVP阶段关键成功指标 (Phase 1):**

> 注: 以下数值为建议目标值，实际目标由运营团队根据市场情况调整

| 指标类型 | 具体指标 | 目标值 | 验收标准 |
|---------|---------|--------|---------|
| **用户规模** | 注册用户数 | ≥1000人 | 种子用户成功导入 |
| | 游客转化率 | ≥60% | 游客体验后注册比例（分母：进入游客模式的独立设备数） |
| **内容沉淀** | 资源库资源数 | ≥200个 | 覆盖8大分类 |
| | 社区话题数 | ≥100个 | 覆盖10大分类 |
| **用户活跃** | DAU/MAU | ≥20% | 日活/月活比例 |
| | 次日留存率 | ≥40% | 新用户次日回访 |
| **功能使用** | AI对话完成率 | ≥80% | 发起对话后完成问答 |
| | 资源下载量 | ≥500次/月 | 平均每资源被下载2.5次 |
| **系统性能** | 首屏加载时间 | ≤2秒 | 所有主页面 |
| | AI响应时间 | ≤1秒 | 文字输入后首字输出 |
| | 系统可用率 | ≥99.9% | 月度统计 |

### 1.3 用户角色

#### 1.3.1 游客用户

**画像:**
- 首次访问平台,尚未注册的潜在用户
- 对可靠性领域有兴趣,希望快速体验平台价值

**核心诉求:**
- 快速了解平台能做什么
- 无门槛体验核心功能(AI问答/资源浏览/社区浏览)
- 明确注册后的价值收益

**权限边界:**
- AI问答: 每日3个新会话(具体轮次限制详见「PRD_可可豆与信誉分体系.md §4.1.2」)
- 资源浏览: 仅可查看10条精华资源,无下载权限
- 社区浏览: 仅可查看10条精华话题,无互动权限
- 个人中心: 无法进入

#### 1.3.2 核心用户(注册用户)

**画像:**
- 电路设计/质量/测试/工艺/失效分析工程师
- 年龄: 25-45岁,本科及以上学历
- 工作场景: 遇到可靠性问题需要快速查询标准/咨询专家/下载资料

**核心诉求:**
- 快速解决工作中遇到的可靠性技术问题
- 获取标准规范/器件手册/应用笔记等专业资料
- 与同行交流经验,学习最佳实践
- 通过贡献内容建立个人技术影响力

**典型场景:**
- 设计评审前查询降额设计规范
- 测试发现问题时咨询AI助手分析原因
- 下载失效分析案例作为参考
- 在社区发起话题讨论技术难点

**等级体系:**
> 注: 以下数值为默认值，由管理后台可配置（管理员需超级管理员授权）

- 新兵(0-99分) → 菜鸟(100-299分) → 入门(300-599分) → 熟手(600-999分) → 老炮(1000-1999分) → 达人(≥2000分) → 专家(≥5000分)
- 等级越高,AI问答次数/基础资源/权益额度内下载次数越多

#### 1.3.3 专家用户 (Phase 3)

**画像:**
- 可靠性领域资深专家/行业大咖
- 15年以上从业经验,具备深厚理论基础与实战经验
- 希望分享经验扩大影响力,寻找咨询项目机会

**核心诉求:**
- 建立个人技术品牌,扩大行业影响力
- 通过一对一咨询获取商业收益
- 与行业同行深度交流,拓展人脉资源

**权限特性:**
- 提供付费一对一咨询服务
- 专属专家认证标识

#### 1.3.4 管理员与超级管理员

**超级管理员 (Super Admin)**

画像:
- 平台最高权限管理人员
- 负责系统配置、权限管理、紧急封禁等最高权限操作

核心诉求:
- 灵活配置系统参数，适应运营策略调整
- 管理管理员账号权限，实现分级授权
- 紧急情况下快速响应，处理重大违规事件
- 实时监控平台整体数据，保障系统稳定

权限范围:
- 参数配置: 按用户等级配置功能权限与规则（最高权限）
- 权限管理: 管理管理员账号，分配权限
- 系统看板: 查看核心运营数据
- 内容审核: 审核资源/话题，处理申诉
- 用户管理: 查看用户列表，处理违规行为
- 安全管理: 配置反爬虫/反刷可可豆🫘规则、紧急封禁
- 模型配置: 配置AI大模型、密钥、URL
- 数据导出: 导出运营数据

**管理员 (Admin)**

画像:
- 日常运营管理人员
- 负责内容审核、用户管理、数据监控

核心诉求:
- 高效完成资源/话题审核，保障内容质量
- 实时监控平台数据，及时发现异常
- 处理意见反馈与申诉，维护平台秩序
- 执行运营策略，调整功能参数

权限范围:
- 系统看板: 查看核心运营数据
- 内容审核: 审核资源/话题，处理申诉
- 用户管理: 查看用户列表，处理违规行为
- 参数配置: 按用户等级配置功能权限与规则（需超级管理员授权）
- 安全管理: 查看安全日志，配置反爬虫/反刷规则（需超级管理员授权）

```

    ↓
入门 (300-599分)
    ↓
熟手 (600-999分)
    ↓

    ↓
```

## 五、设计总则

### 5.1 外观风格

**设计基准:**
- 设备尺寸: iPhone 12 Pro (390×844 pt, 2532×1170 px)
- 安全区域: 顶部44pt, 底部34pt (刘海屏适配)
- 底部导航栏: 50pt高度

**视觉调性:**
- 科技感与高级感并重
- 配色体现可靠性与专业性
- 全平台风格统一(页面/弹窗/按键/图标/卡片/字体)

**产品形态:**
- Phase 1: Web应用,用户端页面带手机边框,模拟微信小程序显示效果
- 管理后台: 独立PC端Web管理系统,无手机边框

**详细设计规范参见:**
- `docs/06_UI_UX_Design/设计规范_色彩字体图标.md`
- `docs/06_UI_UX_Design/设计规范_组件库.md`
- `docs/06_UI_UX_Design/设计规范_移动端布局.md`

### 5.2 核心设计原则

**1. 逻辑清晰,操作顺畅**
- 功能模块划分明确
- 交互路径不超过5级子页面
- 无冗余操作,核心功能3次点击内可达

**2. 权限分级,体验分层**
- 区分游客/注册用户/不同等级用户/专家/管理员权限
- 匹配对应功能与体验
- 渐进式引导,避免信息过载

**3. 模块联动,生态闭环 (Phase 2)**
- 实现AI/资源/社区三大核心模块的深度联动
- 避免功能孤立,形成完整知识闭环

**4. 价值驱动,用户增长**
- 通过可可豆🫘(平台**代币**)/等级体系激励用户贡献
- 通过注册福利/广告发放可可豆🫘降低参与门槛
- 游客模式快速体验核心价值

**5. 安全合规,稳定可靠**
- 保障数据安全/内容安全
- 防止爬虫/刷可可豆🫘等行为
- 符合微信内容安全规范

**6. 异常处理,用户友好**
- AI回答超时/资源访问超时等异常情况
- 系统应提示错误原因或提醒用户重试
- 提升用户体验,避免用户困惑

### 6.2 MVP开发优先级

**S0级(最高优先级,核心必做):**
- 启动与游客模式
- 登录/注册模块
- 爱问(AI助手)核心功能
- 管理后台(基础配置与内容审核)

**S1级(高优先级,核心必做):**
- 资源模块核心功能(上传/下载/收藏)
- 社区模块基础功能(话题发起/回复)
- 我的(个人中心)基础功能(个人信息/可可豆🫘/信誉分与等级)

**S2级(中优先级,配套必做):**
- 统一搜索系统
- 文件与附件处理规范
- 性能与体验优化
- 自动化测试
- 测试账号配置

**S3级(低优先级,基础完善):**
- 通知与消息系统基础功能
- 意见反馈与收集渠道
- 内容生命周期管理基础规则

### 6.3 MVP验收标准

**功能维度:**
- ✓ 游客模式可正常体验AI问答/资源浏览/社区浏览
- ✓ 注册用户可完成AI问答/资源上传下载/社区发帖回复
- ✓ 管理员可完成系统配置/内容审核/用户管理
- ✓ 所有核心功能快乐路径与异常路径测试通过

**性能维度:**
- ✓ 首屏加载时间 ≤ 2秒
- ✓ AI响应时间 ≤ 1秒(文字输入后首字输出)
- ✓ 接口响应时间 ≤ 500ms
- ✓ 系统可用率 ≥ 99.9%
- ✓ 并发用户数支持 ≥ 20人

**安全维度:**
- ✓ 账号安全机制生效(连续错误锁定/异常登录检测)
- ✓ 内容安全机制生效(敏感词过滤/AI内容审核)
- ✓ 数据安全机制生效(加密存储/备份恢复)
- ✓ 反爬虫/反刷可可豆🫘机制生效

**测试维度:**
- ✓ 单元测试覆盖率 ≥ 80%
- ✓ 接口测试覆盖所有API端点
- ✓ E2E测试覆盖核心业务流程
- ✓ 性能测试/安全测试通过
