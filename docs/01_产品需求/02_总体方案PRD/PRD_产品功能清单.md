## 一、文档说明

### 1.1 文档目的

本文档是 ReliHub 项目的**功能全景清单**，系统化梳理平台所有功能特性，明确标注 MVP 范围与后续迭代计划。本清单是后续所有详细 PRD 文档的拆分依据。

### 1.2 功能分类说明

**按开发阶段分类:**
- **Phase 1 (MVP)**: 核心必做功能，验证产品价值
- **Phase 2**: 模块联动与商业化功能
- **Phase 3**: 生态完善与高级功能

**按优先级分类:**
- **S0**: 最高优先级，核心必做
- **S1**: 高优先级，核心必做
- **S2**: 中优先级，配套必做
- **S3**: 低优先级，基础完善

### 1.3 标注说明

| 标记 | 含义 |
|------|------|
| ✅ | MVP 阶段必做 |
| 🔶 | Phase 2 实现 |
| 🔷 | Phase 3 实现 |
| ⚡ | 核心功能 |
