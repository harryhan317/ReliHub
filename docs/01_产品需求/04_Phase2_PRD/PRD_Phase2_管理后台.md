## 一、模块目的与范围边界

### 1.1 文档目的

本文档定义 ReliHub Phase 2 阶段管理后台的扩展功能，与 [PRD_MVP_管理后台.md](../03_MVP_PRD/PRD_MVP_管理后台.md) 配合使用。Phase 2 在 MVP 功能基础上新增充值记录查看、分享激励配置、反馈激励配置三个功能模块。

### 1.2 涵盖范围（对应功能清单 M5-F051~F053）

- **充值记录查看**：用户充值可可豆的订单记录查询与统计
- **分享激励配置**：配置用户通过分享内容获取可可豆奖励的规则
- **反馈激励配置**：配置用户提交有效反馈获取可可豆奖励的规则

### 1.3 与 MVP 管理后台的关系

本文档是 [PRD_MVP_管理后台.md](../03_MVP_PRD/PRD_MVP_管理后台.md) 的补充，不重复定义 MVP 已有的功能规则。Phase 2 新增功能在本文档中详细定义，已有的 MVP 功能请参阅主文档。

## 三、充值记录查看（M5-F051）

### 3.1 功能概述

超级管理员可在管理后查看全平台用户的充值记录，了解用户可可豆充值消费情况。

### 3.2 充值记录列表

**列表字段**：

| 字段 | 说明 |
|-----|------|
| 订单ID | 系统生成的唯一订单号 |
| 用户昵称 | 充值用户（可点击跳转用户详情） |
| 充值金额 | 人民币金额（元） |
| 充值数量 | 获得的可可豆数量 |
| 充值时间 | 精确到秒的时间戳 |
| 支付方式 | 微信支付/支付宝等 |
| 订单状态 | 支付成功/支付失败/已退款/处理中 |

**筛选条件**：

| 字段 | 筛选器类型 |
|-----|-----------|
| 订单时间 | 起止日期选择器 |
| 用户昵称 | 模糊搜索 |
| 订单状态 | 下拉多选 |
| 支付方式 | 下拉单选 |

**统计汇总**：

- 当日/当周/当月充值总额
- 累计充值用户数
- 平均客单价

### 3.3 订单详情

点击订单ID进入详情页，包含完整订单信息及支付流水号。

## 四、分享激励配置（M5-F052）

### 4.1 功能概述

超级管理员可配置用户通过分享内容获取可可豆奖励的规则。

### 4.2 分享激励规则配置

| 配置项 | 说明 | 默认值 |
|-------|------|--------|
| 每日分享奖励上限 | 每个用户每日可获得分享奖励的最大可可豆数 | 10可可豆/日 |
| 单次分享奖励 | 用户分享内容并被有效点击后获得的可可豆 | 1可可豆/次 |
| 有效点击判定 | 通过分享链接进入且完成注册视为有效 | - |
| 奖励发放时机 | 即时发放 / 24小时后发放（防刷） | 即时发放 |
| 奖励开关 | 开启/关闭分享激励功能 | 开启 |

### 4.3 分享激励统计

| 字段 | 说明 |
|-----|------|
| 今日分享次数 | 当日成功分享次数 |
| 今日奖励发放 | 当日发放的分享奖励总量 |
| 有效转化率 | 分享点击→注册的转化比例 |
| TOP分享用户 | 分享次数最多的用户排行 |

### 4.4 风控规则

- 同一用户每日分享奖励上限封顶
- 同一IP段每日分享注册人数上限（防止工作室刷单）
- 奖励异常波动预警（单日奖励发放超过均值3倍触发告警）

## 五、反馈激励配置（M5-F053）

### 5.1 功能概述

超级管理员可配置用户提交有效反馈获取可可豆奖励的规则。

### 5.2 反馈激励规则配置

| 配置项 | 说明 | 默认值 |
|-------|------|--------|
| 有效反馈奖励 | 用户提交的反馈被标记为"有效"后获得的可可豆 | 2可可豆/条 |
| 每日反馈奖励上限 | 每个用户每日可获得的反馈奖励最大可可豆数 | 10可可豆/日 |
| 有效反馈判定 | 管理员判定反馈是否为有效反馈（非无效反馈/重复反馈） | - |
| 奖励发放时机 | 即时发放 / 审核通过后发放 | 审核通过后发放 |
| 奖励开关 | 开启/关闭反馈激励功能 | 开启 |

### 5.3 反馈激励统计

| 字段 | 说明 |
|-----|------|
| 今日提交反馈数 | 当日用户提交的反馈总数 |
| 今日有效反馈数 | 当日被判定为有效的反馈数 |
| 今日奖励发放 | 当日发放的反馈奖励总量 |
| 有效反馈率 | 有效反馈占总反馈的比例 |

### 5.4 无效反馈定义

以下情况反馈不计入有效反馈，不发放奖励：
- 与历史反馈内容重复（文字相似度>80%）
- 与已有FAQ/帮助文档重复
- 与已提交且正在处理中的反馈重复
- 反馈内容无实质诉求（如"很好""不错"等）
