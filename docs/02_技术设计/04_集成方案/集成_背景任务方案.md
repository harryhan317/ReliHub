# 集成方案 - 背景任务方案 (集成_背景任务方案)

## 1. 架构组件
- **任务引擎**: Celery 5.x.
- **Broker/Result Backend**: Redis.
- **调度器**: Celery Beat (执行 Cron 任务)。

## 2. 定时任务清单 (Periodic Tasks)

### 2.1 社区悬赏自动退款 (对齐 PRD §5.5.3)
- **名称**: `community.check_bounty_expiry`
- **执行频率**: 每小时一次 (`0 * * * *`)。
- **逻辑**: 
  - 查询 `topics.bounty_status = ACTIVE` 且 `created_at < now - 30 days` 的记录。
  - 调用 `SettlementService.refund_bounty(topic_id)`。
  - 更新状态为 `REFUNDED`，并推送“悬赏已自动退款”通知给发起人。

### 2.2 文件生命周期管理 (对齐 PRD §8.1)
- **名称**: `file.lifecycle_cleanup`
- **执行频率**: 每日凌晨 2:00 (`0 2 * * *`)。
- **逻辑**: 
  - 清理过期超过 24h 的上传碎片。
  - 查询 `lifecycle_status = SOFT_DELETED` 且超过 30 天的文件，执行物理强删流程。

### 2.3 热度分数重算与状态演进
- **名称**: `heat.recalculate_scores`
- **执行频率**: 间隔 15 分钟。
- **逻辑**: 
  - 根据 `view_count`, `download_count` 等计算 `heat_score` 并写入 PG。
  - **[!] 状态演进说明**：资源“最新 (7天)”标签采用 **Query-time 动态计算** (`WHERE created_at > now - 7 days`)，不设落库状态位，故无需定时任务切换。

### 2.4 账号被动注销与脱敏 (对齐 PRD §4.3)
- **名称**: `user.passive_deactivation_task`
- **执行频率**: 每日凌晨 3:00 (`0 3 * * *`)。
- **逻辑**: 
  - 查询命中被动注销条件（休眠 2 年且无余额）的用户。
  - 执行 PII 物理擦除、`phone_blind_index` 清理、标记 `is_deleted = true`。
  - 同步触发 `file.lifecycle_cleanup` 处理关联附件。

## 3. 监控与异常
- 使用 Sentry 接入 Celery Error Handler。
- 任务执行失败时重试次数：3次 (指数退避)。
