# 集成方案 - 通知系统实现方案 (集成_通知系统实现方案)

## 1. 业务需求 (对齐 PRD §3.1.2)
ReliHub 需支持系统消息、互操作通知（点赞/回复）、以及业务预警的实时推送。

## 2. 技术选型
- **传输协议**: SSE (Server-Sent Events)。适配单向推流需求，较 WebSocket 更轻量且原生支持 HTTP。
- **消息总线**: Redis Pub/Sub。

## 3. SSE 与 Pub/Sub 交互流程
1. **连接建立**: 前端请求 `GET /api/v1/notifications/stream?token=xxx`。
2. **鉴权与订阅**:
   - 后端解析 Token，获取 `user_id`。
   - 打开一个 Redis Subscriber，订阅频道 `notify:user:{user_id}`。
3. **消息下发**: 
   - 当业务系统产生通知时，调用 `PUBLISH notify:user:{user_id} <JSON_DATA>`。
   - SSE 处理进程监听到消息，立即按 `data: <JSON>\n\n` 格式推送至前端。
4. **幂等与补偿**:
   - 若用户离线，消息仅持久化至 `notifications` 表。
   - 客户端重新连接后，先请求 `GET /api/v1/notifications` 拉取未读历史。

## 4. 性能优化
- 采用异步 IO (FastAPI/AnyIO) 处理长连接，避免阻塞 worker 进程。
- 单应用节点支持 2000+ 并发 SSE 连接。
