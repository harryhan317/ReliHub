# 架构设计 - 安全设计规范 (架构设计_安全设计)

## 1. 身份与访问控制 (IAM)
- **认证**: JWT RS256。
  - Access Token: 2h 有效期。
  - Refresh Token: 7d 有效期，存入 HttpOnly Cookie。
- **鉴权 (RBAC)**:
  - `GUEST`: 仅检索预览，AI 限额严格。
  - `MEMBER`: 核心用户，可上传/下载。
  - `EXPERT`: 经认证专家，具备专有标识与话题采纳权。
  - `ADMIN`: 全量管理权限，**IP 子网绑定于 API Gateway 层**（详见 API_管理后台.md §3）。

## 2. 数据安全
- **传输安全**: 全量 HTTPS (强制 TLS 1.2+)。
- **存储安全**:
  - 用户密码: 严禁明文，使用 `Argon2` 或 `Bcrypt` 加密。
  - **PII 盲索引 (Blind Index)**: 针对手机号/邮箱等加密字段，使用 `HMAC-SHA256(data, secret_key)` 生成不可逆哈希索引。`secret_key` 由 KMS 托管，支持 1y 一次的版本轮转。查询时应用当前活跃版本 Salt。
  - 文件防盗链: OSS 桶设为私有，下载通过 5min 短效签名为准。
  - 数字水印: 下载时注入后端盲水印，防止资源泄露溯源。

## 3. 应用层防御
- **查重检测**: 基于向量余弦相似度阻塞重复上传。
- **注入防护**: SQLAlchemy 参数化查询，原生防御 SQL 注入。
- **跨站防护**: 开启 CSRF Token 校验 (针对 Form 提交) 与 CORS 白名单配置。
- **输入过滤**: Pydantic 严格校验类型，敏感内容接入微信内容安全 API 预检。

## 4. 逻辑安全
- **管理员保护**: 
  - `/admin/*` 路径及管理后台 API 强绑定 `users.admin_subnet` (CIDR)。
  - **拦截点位于 API Gateway 层**：网关提取 JWT 中的 `role` 与 `UID`，异步查询缓存中的子网配置并匹配 `X-Forwarded-For` 真实 IP。
  - 非匹配请求返回 `40302 Admin_IP_Restricted`。
- **反爬虫**: 基于 Redis 的令牌桶算法限速，针对高频搜索请求触发验证码。
