## 🎯 核心设计原则

### 5 大核心原则

1. **简洁至上 (Simplicity)** ⭐⭐⭐⭐⭐
   - 少即是多，去除一切不必要的元素
   - 界面元素精简，只保留必要内容

2. **内容为王 (Content is King)** ⭐⭐⭐⭐⭐
   - 让内容成为焦点，界面为内容服务
   - 充足留白突出内容

3. **直观易用 (Intuitive)** ⭐⭐⭐⭐⭐
   - 无需说明书，用户凭直觉就能使用
   - 符合常识的交互，即时清晰的反馈

4. **一致性 (Consistency)** ⭐⭐⭐⭐
   - 整个产品保持统一的设计语言
   - 全局统一色彩/字体/间距/圆角/阴影

5. **人性化关怀 (Human-Centered)** ⭐⭐⭐⭐⭐
   - 尊重用户，关怀用户，让技术有温度
   - 尊重用户时间，用温暖的语言对话

## 🎨 关键设计规范

### 色彩系统

```css
/* 主色 - 苹果深空灰 */
--color-primary: #1D1D1F;        /* 深空灰 */
--color-primary-light: #6E6E73;  /* 中灰色 */
--color-primary-lighter: #86868B;/* 浅灰色 */

/* 强调色 - 苹果蓝 */
--color-accent: #0071E3;         /* 苹果蓝 */
--color-accent-hover: #0077ED;   /* 悬停状态 */

/* 功能色 */
--color-success: #34C759;        /* 苹果绿 */
--color-warning: #FF9500;        /* 苹果黄 */
--color-error: #FF3B30;          /* 苹果红 */
```

**使用原则**: 60-30-10 法则
- 60% 主色 (白/灰)
- 30% 辅助色 (内容色)
- 10% 强调色 (蓝色)

---

### 字体系统

```css
/* 优先使用系统字体 */
--font-family: -apple-system, BlinkMacSystemFont, "SF Pro Text", 
               "Helvetica Neue", "PingFang SC", "Microsoft YaHei";

/* 字号规范 */
--text-4xl: 34px;  /* 超大标题 */
--text-3xl: 28px;  /* 大标题 */
--text-2xl: 22px;  /* 模块标题 */
--text-xl: 20px;   /* 正文标题 */
--text-lg: 17px;   /* 正文大字 */
--text-base: 17px; /* 正文默认 */
--text-sm: 15px;   /* 次要内容 */
--text-xs: 13px;   /* 提示文字 */
```

---

### 间距系统 (8px 网格)

```css
--space-2: 8px;    /* 紧凑间距 */
--space-3: 12px;   /* 标准紧凑 */
--space-4: 16px;   /* 标准间距 */
--space-5: 20px;   /* 舒适间距 */
--space-6: 24px;   /* 大间距 */
--space-8: 32px;   /* 超大间距 */
--space-12: 48px;  /* 模块间距 */
--space-16: 64px;  /* 页面间距 */
```

**原则**: 宁大勿小，保证充足留白

---

### 圆角规范

```css
--radius-sm: 8px;    /* 按钮/输入框 */
--radius-md: 12px;   /* 卡片/模态框 */
--radius-lg: 16px;   /* 大卡片 */
--radius-xl: 20px;   /* 装饰元素 */
--radius-full: 9999px; /* 头像/徽章 */
```

---

### 交互动效

#### 持续时间

```css
--duration-instant: 100ms;   /* 按钮点击 */
--duration-fast: 150ms;      /* 颜色变化 */
--duration-normal: 200ms;    /* 页面切换 */
--duration-slow: 300ms;      /* 复杂动画 */
```

#### 缓动函数

```css
/* 标准缓动 */
--ease-standard: cubic-bezier(0.4, 0.0, 0.2, 1);

/* 快速进入 */
--ease-in-quick: cubic-bezier(0.4, 0.0, 1, 1);

/* 慢速离开 */
--ease-out-slow: cubic-bezier(0.0, 0.0, 0.2, 1);

/* 弹性效果 */
--ease-bounce: cubic-bezier(0.68, -0.55, 0.265, 1.55);
```

#### 标准动画

**按钮点击**:
```css
.button:active {
  transform: scale(0.96);
  transition: transform 100ms ease-standard;
}
```

**模态框弹出**:
```css
.modal-enter {
  opacity: 0;
  transform: scale(0.95) translateY(10px);
  transition: all 200ms ease-out-slow;
}
```

## 🛠️ 技术栈更新

### 新增依赖

```json
{
  "dependencies": {
    "framer-motion": "^10.x",    // 苹果式流畅动画
    "lucide-react": "^0.x"       // 简洁线条图标
  }
}
```

### 使用示例

**Framer Motion 动画**:
```tsx
import { motion } from 'framer-motion';

<motion.button
  whileTap={{ scale: 0.96 }}
  transition={{ duration: 0.1, ease: "easeInOut" }}
>
  点击我
</motion.button>
```

**Lucide React 图标**:
```tsx
import { Home, Search, User } from 'lucide-react';

<Home size={24} strokeWidth={2} />
```

## ✅ 设计检查清单

### 视觉检查

- [ ] 界面简洁，无多余装饰
- [ ] 留白充足，内容突出
- [ ] 色彩克制，不超过 3 种主色
- [ ] 字体层次清晰
- [ ] 间距统一 (8px 网格)
- [ ] 圆角一致
- [ ] 阴影自然柔和

---

### 交互检查

- [ ] 操作直观，无需思考
- [ ] 反馈即时清晰
- [ ] 动画流畅自然
- [ ] 符合用户预期
- [ ] 错误提示友好
- [ ] 加载状态清晰

---

### 内容检查

- [ ] 文字简洁明了
- [ ] 标题层次清晰
- [ ] 图片高质量
- [ ] 图标语义明确
- [ ] 无冗余信息

---

### 响应式检查

- [ ] 手机体验优秀
- [ ] 平板布局合理
- [ ] PC 信息完整
- [ ] 触控区域 ≥ 44px (iOS 标准)
- [ ] 字体大小合适

---

### 无障碍检查

- [ ] 色彩对比度达标 (≥ 4.5:1)
- [ ] 键盘导航可用
- [ ] 焦点状态清晰
- [ ] 屏幕阅读器友好
- [ ] 动画可关闭

## 🎓 实施步骤

### 1. 学习阶段 (1-2 天)

**设计师**:
- 学习苹果设计美学规范文档
- 研究 Apple HIG 官方指南
- 分析优秀苹果风格应用案例

**前端工程师**:
- 学习苹果设计美学规范文档
- 学习 Framer Motion 使用
- 学习 Tailwind CSS 实现设计系统

**产品经理**:
- 理解苹果设计美学原则
- 在 PRD 中明确体验要求
- 评审设计稿是否符合规范

---

### 2. 设计阶段 (3-5 天)

**设计师**:
1. 创建设计系统组件库 (Figma)
2. 定义色彩/字体/间距/圆角/阴影
3. 设计核心页面高保真稿
4. 制作交互原型
5. 设计评审 + 修改

**交付物**:
- Figma 设计稿链接
- 设计系统文档
- 交互原型

---

### 3. 开发阶段 (持续)

**前端工程师**:
1. 使用 Tailwind CSS 实现设计系统
2. 开发基础组件库 (Button/Input/Card/Modal)
3. 实现交互动效 (Framer Motion)
4. 页面开发
5. 响应式适配
6. 无障碍优化

**交付物**:
- 可复用组件库
- 完整页面
- 单元测试

---

### 4. 验收阶段 (持续)

**产品经理**:
- 评审设计还原度
- 检查用户体验
- 确认符合苹果美学

**测试工程师**:
- UI 测试 (视觉回归)
- 交互测试 (动效流畅度)
- 响应式测试
- 无障碍测试
