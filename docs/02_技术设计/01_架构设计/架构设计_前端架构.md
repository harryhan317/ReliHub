# 架构设计 - 前端架构 (架构设计_前端架构)

## 1. 技术栈选型
- **框架**: [React 18 (Hooks & Context API)](https://react.dev/)
- **构建工具**: [Vite 5.x](https://vitejs.dev/)
- **样式**: [Tailwind CSS 3.x](https://tailwindcss.com/)
- **路由**: [React Router 6.x](https://reactrouter.com/)
- **状态管理**: [Zustand](https://docs.pmnd.rs/zustand/getting-started/introduction) (轻量级状态管理)
- **网络请求**: [Axios](https://axios-http.com/)
- **组件库**: 原生 Tailwind 构建组件，部分复杂组件使用 [Headless UI](https://headlessui.com/)

## 2. 目录结构规范 (`src/reliHub-web`)
```
src/
├── assets/         # 静态资源 (图片, 字体)
├── components/     # 公共基础组件 (Button, Input, Modal)
├── context/        # 全局 Context (Auth, Theme)
├── hooks/          # 自定义 Hooks (useAuth, useLocalStorage)
├── layouts/        # 页面布局 (MainLayout, AdminLayout)
├── pages/          # 业务页面 (Home, AI, Community, Profile)
├── services/       # API 接口封装
├── store/          # Zustand 状态店
├── types/          # TypeScript 类型定义
└── utils/          # 工具函数 (Formatter, Validator)
```

## 3. 核心设计原则
- **响应式优先**: 采用 Mobile-First 策略，所有页面必须完全适配手机端与 PC 端。
- **组件化**: 严禁在页面中编写超过 300 行的业务逻辑，复杂逻辑必须提取为自定义 Hooks 或子组件。
- **错误边界**: 关键模块封装 `ErrorBoundary`，防止单一组件崩溃导致白屏。
- **SEO 优化**: 核心页面使用语义化 HTML 结构，确保 Meta 标签动态更新。

## 4. 关键业务流程
### 4.1 鉴权流
- 登录成功后将 JWT 存入 `localStorage` 与 `useAuthStore`。
- 配置 Axios Interceptor，自动在 Header 中携带 `Authorization: Bearer <Token>`。
- 401 状态码触发自动清除 Session 并跳转至登录页。

### 4.2 AI 实时通信
- 使用原生 `EventSource` 实现 SSE (Server-Sent Events) 流式显示。
- 使用 `useImmer` 管理复杂的对话状态更新，确保平滑滚动与打字机效果。

## 4. 异步任务与实时体验 (Premium UI/UX)
- **状态同步**: 针对文件转码、AI 解析等长耗时任务，采用 **"SSE 主动推送 + 状态轮询兜底"** 策略。
- **UI 呈现**: 
  - 任务发起后立即进入 `LOADING` 态并展示微缩进度条。
  - 通过 SSE 接收 `CONVERSION_COMPLETED` 事件并无缝切换预览图层。
  - 确保翻页响应 ≤ 500ms（基于前端预加载）。
