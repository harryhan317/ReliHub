## 🎯 设计理念

### 核心原则

ReliHub 采用**苹果设计美学 (Apple Design Philosophy)** 作为产品交互界面的设计指导方针，追求**简洁、优雅、直观、人性化**的用户体验。

> **"设计不仅仅是外观和感觉，设计是关于产品如何运作的。"** — Steve Jobs

---

## 📐 设计原则

### 1. 简洁至上 (Simplicity) ⭐⭐⭐⭐⭐

**核心理念**: 少即是多，去除一切不必要的元素

**实施规则**:
- ✅ **视觉简洁**: 界面元素精简，只保留必要内容
- ✅ **交互简洁**: 操作流程最短化，减少用户思考
- ✅ **信息简洁**: 信息层次清晰，重点突出
- ❌ **避免**: 过度装饰、复杂动画、冗余信息

**示例**:
```
✅ 正确：单一主要操作按钮，清晰明确
❌ 错误：多个同等重要的按钮，让用户选择困难
```

---

### 2. 内容为王 (Content is King) ⭐⭐⭐⭐⭐

**核心理念**: 让内容成为焦点，界面为内容服务

**实施规则**:
- ✅ **留白艺术**: 充足的留白突出内容本身
- ✅ **字体优先**: 使用高质量字体，通过字号/字重建立层次
- ✅ **图片质量**: 只使用高质量图片，宁缺毋滥
- ❌ **避免**: 花哨背景、过度边框、干扰内容的装饰

**苹果式布局**:
```
┌─────────────────────────────┐
│  [简洁导航]                  │
├─────────────────────────────┤
│                             │
│         内容区域             │
│      (充足留白环绕)          │
│                             │
└─────────────────────────────┘
```

---

### 3. 直观易用 (Intuitive) ⭐⭐⭐⭐⭐

**核心理念**: 无需说明书，用户凭直觉就能使用

**实施规则**:
- ✅ **符合常识**: 使用用户熟悉的图标和交互模式
- ✅ **即时反馈**: 每个操作都有清晰的视觉反馈
- ✅ **自然映射**: 控件行为符合用户预期
- ❌ **避免**: 隐藏功能、反直觉操作、需要学习成本

**反馈示例**:
- 按钮点击：轻微缩放 + 颜色变化 (100ms)
- 加载状态：优雅的进度指示器
- 操作成功：简洁的 Toast 提示

---

### 4. 一致性 (Consistency) ⭐⭐⭐⭐

**核心理念**: 整个产品保持统一的设计语言

**实施规则**:
- ✅ **统一组件**: 相同功能使用相同组件
- ✅ **统一交互**: 相同操作有相同反馈
- ✅ **统一视觉**: 色彩/字体/间距全局一致
- ❌ **避免**: 页面间风格差异、组件行为不一致

---

### 5. 人性化关怀 (Human-Centered) ⭐⭐⭐⭐⭐

**核心理念**: 尊重用户，关怀用户，让技术有温度

**实施规则**:
- ✅ **尊重时间**: 快速响应，不浪费用户时间
- ✅ **尊重选择**: 提供默认值，但允许自定义
- ✅ **尊重情感**: 用温暖的语言与用户对话
- ❌ **避免**: 冷漠的错误提示、强制操作、打断用户

## 🎨 视觉设计规范

### 色彩系统

#### 主色调

```css
/* ReliHub 主色系统 - 遵循苹果设计美学与前端UI规范统一版本 */
/* 更新于 2026年4月5日 - 颜色冲突修复，采用方案 A */

/* 主要交互色 - 苹果蓝 */
--color-primary: #007AFF;        /* 苹果蓝 - 主操作、链接、按钮 */
--color-primary-hover: #0051D5;  /* 悬停状态深蓝 */
--color-primary-active: #003DA6; /* 激活状态更深蓝 */

/* 中立文字色系统 */
--color-text-primary: #000000;   /* 黑色 - 主文本 */
--color-text-secondary: #666666; /* 深灰 - 次要文本 */
--color-text-tertiary: #999999;  /* 浅灰 - 提示/禁用文字 */

/* 强调色 (与主色 hover 一致) */
--color-accent: #0051D5;         /* 悬停/强调状态 */
--color-accent-active: #003DA6;  /* 激活状态 */
```

#### 功能色

```css
/* 成功 - 苹果绿 */
--color-success: #34C759;

/* 警告 - 苹果黄 */
--color-warning: #FF9500;

/* 错误 - 苹果红 */
--color-error: #FF3B30;

/* 信息 - 蓝绿 */
--color-info: #00C7BE;
```

#### 背景色

```css
/* 纯白背景 */
--bg-primary: #FFFFFF;

/* 浅灰背景 (分组/卡片) */
--bg-secondary: #F5F5F7;

/* 深灰背景 (遮罩) */
--bg-overlay: rgba(0, 0, 0, 0.4);
```

#### 使用原则

1. **60-30-10 法则**:
   - 60% 主色 (白/灰)
   - 30% 辅助色 (内容色)
   - 10% 强调色 (蓝色)

2. **克制的色彩使用**:
   - ✅ 全页面不超过 3 种主色
   - ✅ 功能色仅用于状态提示
   - ❌ 避免大面积高饱和度色彩

### 字体系统

#### 字体家族

```css
/* 优先使用系统字体，确保最佳渲染效果 */
--font-family: -apple-system, BlinkMacSystemFont, "SF Pro Text", 
               "Helvetica Neue", Helvetica, Arial, "PingFang SC", 
               "Hiragino Sans GB", "Microsoft YaHei", sans-serif;

/* 等宽字体 (代码显示) */
--font-mono: "SF Mono", "Monaco", "Inconsolata", 
             "Fira Mono", "Droid Sans Mono", monospace;
```

#### 字号规范 (基于 iOS 标准)

```css
/* 超大标题 - 页面标题 */
--text-4xl: 34px;  /* 2.125rem */
--text-3xl: 28px;  /* 1.75rem */

/* 大标题 - 模块标题 */
--text-2xl: 22px;  /* 1.375rem */
--text-xl: 20px;   /* 1.25rem */

/* 正文标题 */
--text-lg: 17px;   /* 1.0625rem */

/* 正文字号 */
--text-base: 17px; /* 1.0625rem - iOS 默认 */
--text-sm: 15px;   /* 0.9375rem - 次要内容 */
--text-xs: 13px;   /* 0.8125rem - 提示文字 */
```

#### 字重规范

```css
/* 粗体 - 重要标题 */
--font-bold: 700;

/* 半粗体 - 次级标题 */
--font-semibold: 600;

/* 中等 - 强调文字 */
--font-medium: 500;

/* 常规 - 正文内容 */
--font-regular: 400;
```

#### 行高规范

```css
/* 紧凑行高 - 标题 */
--leading-tight: 1.2;

/* 标准行高 - 正文 */
--leading-normal: 1.5;

/* 宽松行高 - 长文本 */
--leading-relaxed: 1.625;
```

#### 字体使用原则

1. **字号层次**: 不超过 5 个层级
2. **字重对比**: 使用字重而非颜色建立层次
3. **行长控制**: 45-75 字符/行 (中文 30-50 字)
4. **行距舒适**: 正文行高 ≥ 1.5 倍字号

### 间距系统

#### 8px 网格系统

```css
/* 基础间距单位：8px */
--space-1: 4px;    /* 0.25rem - 超紧凑 */
--space-2: 8px;    /* 0.5rem - 紧凑 */
--space-3: 12px;   /* 0.75rem - 标准紧凑 */
--space-4: 16px;   /* 1rem - 标准间距 */
--space-5: 20px;   /* 1.25rem - 舒适间距 */
--space-6: 24px;   /* 1.5rem - 大间距 */
--space-8: 32px;   /* 2rem - 超大间距 */
--space-10: 40px;  /* 2.5rem - 分隔间距 */
--space-12: 48px;  /* 3rem - 模块间距 */
--space-16: 64px;  /* 4rem - 页面间距 */
--space-20: 80px;  /* 5rem - 超大分隔 */
--space-24: 96px;  /* 6rem - 特殊场景 */
```

#### 间距使用原则

1. **统一单位**: 所有间距为 8 的倍数 (4px 用于超紧凑场景)
2. **呼吸感**: 宁大勿小，保证充足留白
3. **层次分明**: 使用不同间距建立视觉层次
4. **响应式**: 移动端适当减小间距

**示例**:
```
卡片内部：padding = 16px (space-4)
卡片之间：margin = 16px (space-4)
模块之间：margin = 48px (space-12)
```

### 阴影规范

```css
/* 轻微阴影 - 卡片悬浮 */
--shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.05);

/* 标准阴影 - 悬浮卡片 */
--shadow-md: 0 4px 6px -1px rgba(0, 0, 0, 0.1),
             0 2px 4px -1px rgba(0, 0, 0, 0.06);

             0 4px 6px -2px rgba(0, 0, 0, 0.05);

/* 超大阴影 - 重要浮层 */
```

#### 阴影使用原则

1. **克制使用**: 阴影用于表达层次，非装饰
2. **自然柔和**: 避免生硬的黑影
3. **性能优先**: 避免过度使用复杂阴影

## 🎬 交互动效规范

### 动效原则

1. **目的明确**: 每个动画都有明确的功能目的
2. **快速流畅**: 持续时间短，缓动自然
3. **不打断**: 不阻塞用户操作，可中断
4. **有意义**: 提供空间关系、状态变化等信息

---

### 持续时间

```css
/* 即时反馈 - 按钮点击 */
--duration-instant: 100ms;

/* 快速过渡 - 颜色/图标变化 */
--duration-fast: 150ms;

/* 标准过渡 - 页面切换/模态框 */
--duration-normal: 200ms;

/* 慢速过渡 - 复杂动画 */
--duration-slow: 300ms;

/* 超长动画 - 引导动画 */
--duration-very-slow: 500ms;
```

---

### 缓动函数

```css
/* 标准缓动 - 大多数场景 */
--ease-standard: cubic-bezier(0.4, 0.0, 0.2, 1);

/* 快速进入 - 进入动画 */
--ease-in-quick: cubic-bezier(0.4, 0.0, 1, 1);

/* 慢速离开 - 离开动画 */
--ease-out-slow: cubic-bezier(0.0, 0.0, 0.2, 1);

/* 弹性效果 - 成功反馈 */
--ease-bounce: cubic-bezier(0.68, -0.55, 0.265, 1.55);
```

---

### 标准动画

#### 1. 按钮点击反馈

```css
.button-click {
  transform: scale(0.96);
  transition: transform 100ms ease-standard;
}
```

**说明**: 轻微缩放，提供触觉般的反馈

---

#### 2. 模态框弹出

```css
.modal-enter {
  opacity: 0;
  transform: scale(0.95) translateY(10px);
  transition: all 200ms ease-out-slow;
}

.modal-enter-active {
  opacity: 1;
  transform: scale(1) translateY(0);
}
```

**说明**: 从下方轻微淡入，自然流畅

---

#### 3. 列表项进入

```css
.list-item-enter {
  opacity: 0;
  transform: translateY(20px);
  transition: all 300ms ease-out-slow;
}

.list-item-enter-active {
  opacity: 1;
  transform: translateY(0);
}
```

**说明**: 依次进入，营造节奏感

---

#### 4. 加载动画

```css
@keyframes spinner {
  0% { transform: rotate(0deg); }
  100% { transform: rotate(360deg); }
}

.spinner {
  width: 20px;
  height: 20px;
  border: 2px solid #F5F5F7;
  border-top-color: #0071E3;
  border-radius: 50%;
  animation: spinner 0.8s linear infinite;

```

**说明**: 简洁的圆形加载，不干扰用户

#### 5. 成功反馈

```css

  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.1); }
}

.success-checkmark {
  animation: success-bounce 300ms ease-bounce;
}
```

**说明**: 轻微弹跳，传达愉悦感

## 📱 响应式设计规范

### 断点定义

```css
/* 超小屏 - 小屏手机 */
--breakpoint-xs: 375px;

/* 小屏 - 标准手机 */
--breakpoint-sm: 640px;

/* 中屏 - 平板竖屏 */
--breakpoint-md: 768px;

/* 大屏 - 平板横屏/小屏 PC */
--breakpoint-lg: 1024px;

/* 超大屏 - 标准 PC */
--breakpoint-xl: 1280px;

/* 超大屏 - 大屏 PC */
--breakpoint-2xl: 1536px;
```

---

### 适配原则

1. **Mobile First**: 从小屏开始设计，逐步增强
2. **内容优先**: 小屏只显示核心内容
3. **触控友好**: 按钮 ≥ 44x44px (iOS 标准)
4. **渐进增强**: 大屏增加装饰和辅助信息

---

### 布局策略

#### 手机 (< 768px)

- 单栏布局
- 全宽卡片
- 底部导航
- 隐藏次要信息

#### 平板 (768px - 1024px)

- 双栏布局
- 卡片网格 (2 列)
- 侧边导航
- 显示次要信息

#### PC (> 1024px)

- 多栏布局
- 卡片网格 (3-4 列)
- 顶部导航
- 完整信息展示

## ♿ 无障碍设计规范

### 色彩对比度

- **大标题**: 对比度 ≥ 3:1 (WCAG AA)

- 所有交互元素可键盘访问

- 所有图片有 alt 文本

- 按钮有清晰文字
- 状态变化有 ARIA 通知
