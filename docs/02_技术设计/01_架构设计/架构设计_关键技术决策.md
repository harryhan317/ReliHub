# 架构设计 - 关键技术决策 (TDD)
 
 ## 1. 核心决策矩阵
 
 | 决策编号 | 决策主题 | 决策方案 | 决策依据 |
 |---------|---------|----------|---------|
 | TD-001 | AI 对话 Prompt 版本化 | 采用 **Session Snapshot (会话快照)** 模式。创建会话时记录当前 `active_prompt_version`，后续对话始终沿用该版本。 | 确保用户历史对话逻辑的一致性，防止全局 Prompt 更新导致历史输出崩坏。 |
 | TD-002 | 早鸟奖励并发控制 | 采用 **Redis INCR (原子计数)** 预扣模式。 | PRD 要求前 200 名，高并发注册场景下 DB 计数由于事务隔离级别可能导致超发。 |
 | TD-003 | 等级权益校验位置 | 采用 **Custom JWT Context + Middleware** 拦截。 | 在 API 网关/中间件层通过 JWT 中的 7 等级标识直接匹配内存中的限额矩阵（Quotas）。 |
 | TD-004 | 管理后台登录限制 | 采用 **IP Subnet Binding (/24)**。 | 针对 MVP 管理安全性，限制只能在内网或特定子网段登录，降低撞库风险。 |
 | TD-005 | 操作审计链路 | 采用 **Hash Chained Logging (哈希链审计日志)**。 | 将管理员的配置修改记录按顺序生成 SHA-256 链，确保审计日志不可被物理篡改。 |
 | TD-006 | 文件预览方案 | 采用 **Asynchronous Worker Transcoding (异步转码流水线)**。 | TIF/Word/PPT 采用 LibreOffice 独立任务队列转码为 PDF 输出，SLA 要求开始转码到返回 5s 内。 |
