# 技术设计 - 总体架构设计
 
 ## 1. 系统概述
 ReliHub 采用前后端分离架构，前端为移动端适配的 Web 应用，后端为基于 FastAPI 的高性能异步架构。
 
 ## 2. 技术栈 (Tech Stack)
 - **前端**: React 18 + Tailwind CSS + Lucide Icons
 - **后端**: Python 3.10+ + FastAPI + SQLAlchemy 2.0
 - **数据库**: PostgreSQL 14 (全文检索 + 关系型数据)
 - **缓存**: Redis (Session, Rate Limiting, High-freq Config)
 - **模型**: DeepSeek-V3 / GPT-4o (根据 PRD_总体方案 V1.1 对齐)
 - **基础设施**: Docker + Nginx
 - **后台任务**: Celery (Redis Broker)，负责异步转码、数字盲水印注入、30天悬赏退回、向量查重队列
 
 ## 3. MVP 性能目标 (对齐 PRD_非功能需求)
 - **并发用户数**: ≥ 20 人 (Phase 1 MVP)
 - **峰值 QPS**: ≥ 100
 - **响应时间 (P95)**: 
   - 查询类接口 ≤ 500ms
   - 写入类接口 ≤ 1s
   - AI 首字响应 ≤ 1s
 ```mermaid
 graph TD
     A[User Client - Mobile Web] --> B[Nginx Reverse Proxy]
     B --> C[FastAPI Application Server]
     C --> D[(PostgreSQL)]
     C --> E[(Redis)]
     C --> F[LLM API Services]
     C --> G[OSS Storage]
     C --> H[(Celery Worker)]
     H --> E[(Redis Broker)]
 ```
 
 ## 4. 关键技术方案
 - **AI 流式传输**: 采用 SSE (Server-Sent Events) 实现 1.0s 内的首字响应。
 - **文件处理**: 采用预签名 URL (Pre-signed URL) 方案，文件直接上传至 OSS，减轻应用服务器吞吐压力。
 - **后台任务**: Celery (Redis Broker) 异步任务队列，实现文件转码、数字盲水印注入、社区悬赏 30 天退回、向 量查重等后台处理。
 - **水印技术**: 基于 Canvas 的前端可见水印 + 后端盲水印注入。
 - **安全机制**: JWT 鉴权 + 管理员 IP /24 段动态锁定。
 
 ## 5. Phase 3 扩展 RAG 预埋针对性设计
 - **向量数据库**: 预留 PGVector 插件支持，用于存储文档 Embedding。
 - **Embedding 服务**: 抽象 `IEmbeddingProvider` 接口，支持 Phase 3 切换国产/国际主流模型。
 - **检索流水线**: 架构支持“检索 + 增强 + 生成”流程，MVP 阶段仅执行“System Prompt + 生成”。
