## 1. 错误码设计与响应规范

为保证前端交互能够精确捕获业务异常并进行分支处理，平台各接口响应中的 `code` 字段需严格遵循本表。

### 1.1 错误码结构与预留规则
采用 `[模块前缀]_[4位数字]` 的字符串格式，例如 `ECON_4001`。
- **预留规则**：编号在模块内按顺序填入，保留跳号（如结尾0或5）供未来细分场景扩展。
- **编号复用说明**：如 `SYS_4290` 采用 `429` 属于刻意为之，是因为它与网关 `HTTP 429` 同属限流拦截场景，但它是**业务层的独立逻辑计数**（防刷赞等），必须严格同网关层 Nginx 限流作维度隔离。
- **000000**：仅表示绝对成功（与 HTTP 200 且无业务驳回同等）。

### 1.2 HTTP 状态码映射原则
业务错误码必须与标准的 HTTP Status Code 配合使用，严禁全部返回 200 并仅靠业务 code 报错。
- **HTTP 400 (Bad Request)**：客户端参数校验失败或常规业务受阻（如 `ECON/RES/COM` 大部分错误）。
- **HTTP 401 (Unauthorized)**：未登录或 Token 失效（`AUTH` 相关）。
- **HTTP 403 (Forbidden)**：权限/等级不足、被封禁或风控拦截导致无权操作（`USER` 封禁、`SYS_4290` 业务高频）。
- **HTTP 404 (Not Found)**：操作的实体资源不存在或已被删除。
- **HTTP 429 (Too Many Requests)**：网关层全局级暴力限流（Nginx级别的纯拒绝）。
- **HTTP 500 (Internal Server Error)**：未捕获的代码级别逻辑崩溃异常（对应 `SYS_5000`）。
- **HTTP 502 / 504 (Bad Gateway / Gateway Timeout)**：外部 RPC 断连或第三方底层组件失效（对应 `SYS_5002`）。
- **HTTP 503 (Service Unavailable)**：系统主动暂停服务进行升级维护（对应 `SYS_5001`）。

### 1.3 标准 JSON 响应示例
为了彻底贯彻多语言规范，所有涉及变量参数的报错，严禁把时间或数值强行拼装进 `msg`，必须通过下放标准 snake_case 风格的 `data` 承接。

**示例 A：余额不足 (ECON_4001)**
```json
{
  "code": "ECON_4001",
  "msg": "可可豆余额不足以覆盖本次交易",
  "data": { "balance_welfare": 5, "balance_asset": 0, "required_total": 20 }
}
```

**示例 B：封禁倒计时 (USER_4010)**
```json
{
  "code": "USER_4010",
  "msg": "您的账号已被安全锁定",
  "data": { "ban_until": "2026-04-03T00:00:00Z", "reason": "违规引流" }
}
```

**示例 C：系统升级维护 (SYS_5001)**
```json
{
  "code": "SYS_5001",
  "msg": "系统升级维护中",
  "data": { "maintenance_until": "2026-04-01T06:00:00Z" }
}
```

## 2. 模块级业务错误码字典

> **多语言规范**：`msg` 字段限定为**简体中文**的 fallback 提示文案，前端应尽量基于 `code` 做 i18n 资源文件翻译映射，严禁解析 `msg` 来控制页面逻辑分支。
> **主语约束**：对于面向用户的提示语，统一采用第二人称“您”。

### 2.1 鉴权与账号模块 (AUTH & USER) (映射 HTTP 401/403)
| 错误码 | 描述说明 | 前端建议交互 / 应对策略 |
|--------|---------|-----------------------|
| `AUTH_4000` | Token 缺失或无效格式 | 强制跳转登录页 |
| `AUTH_4001` | Access Token 已过期 | 尝试 Refresh Token 续期；如续期仍失败则直接降级到登录页 |
| `AUTH_4002` | 账号异地登录踢出 | 强制登出并弹窗提示“您的账号已在其他设备登录，如非本人操作请立即修改密码” |
| `AUTH_4003` | 本地化验证码等校验核注失败 | 提示“验证码错误，请重新输入” |
| `AUTH_4004` | 验证码获取频率短期内超限 | 提示“验证码发送过于频繁，请稍后重试” |
| `AUTH_4005` | Refresh Token 刷新重试失败/过期 | 清除本地鉴权缓存，无条件跳转回登录页重新认证 |
| `AUTH_4006` | 管理员 IP 白名单校验失败 | 提示"当前网络环境受限，请联系运维确认"，拦截入侵 |
| `USER_4010` | 账号已被封禁/被安全锁定 | 跳转拦截页，依托 `data.ban_until` 解析时间以展示系统解封倒计 |
| `USER_4011` | 账号长周期不活跃进入休眠期 | 提示“您的账号已进入休眠状态，请重新激活” |
| `USER_4012` | 用户瞬时信誉分低于操作红线阈值 | 从 `data: {current_reputation: 45, required_reputation: 50, deficit: 5}` 提取差额，提示“您的信誉分不够，无法执行该项操作” |
| `USER_4013` | 用户当前层级不足以越权操作 | 提示“当前操作需要更高级别的账号权限，请查看等级晋升指南” |
| `USER_4014` | 专家资格认证申请遭到审核工单驳回 | 提示“您的专家申请未通过”，并从 `data.reject_reason` 展示未通过的详细原因 |

### 2.2 经济模型与悬赏流转模块 (ECON & BOUNTY) (映射 HTTP 400/403)
*触发 `ECON_4001` 等资金拦截断档时，底层引擎应同步在 `attempted_transaction` 写入尝试交易死信记录。*

| 错误码 | 描述说明 | 前端建议交互 / 应对策略 |
|--------|---------|-----------------------|
| `ECON_4001` | 可可豆双资总余额不足以覆盖当期交易 | 唤起动态弹窗：“您的可可豆余额不足以覆盖本次交易（系统可用 0，共计需要 Y）”，继而引导获取或充值 |
| `ECON_4002` | 跨池并发状态锁定产生隔离行锁冲突 | 提示“系统正在处理您的同类扣费订单，请勿重复操作” |
| `ECON_4004` | 订单涉金数额直接突破平台定义硬性上限 | 提示“单次悬赏金额不可超过平台规定的单笔最高额度” |
| `ECON_4005` | 账户资产行已因为严重风控处于冻结状态 | 警惕提示“您的账户资产已被冻结，无法执行资金操作，请联系管理员” |
| `BOUNTY_4001`| 用户当前双豆总金额不足锁定该单悬赏| 提示“您的可可豆余额不足以锁定该悬赏金，请降低金额或获取更多可可豆” |
| `BOUNTY_4002`| 悬赏时效逾期已被底座自动核销失效| 提示“该悬赏已逾期（默认30天），系统已自动关闭并退款至发布者账户” |
| `BOUNTY_4003`| 预备采纳发放时检测发榜者财产状态冻结| 提示“执行悬赏采纳时，检测到悬赏发布者账户已冻结，无法完成悬赏金发放” |

### 2.3 资源处理与云文件模块 (RES) (映射 HTTP 400/404)
| 错误码 | 描述说明 | 前端建议交互 / 应对策略 |
|--------|---------|-----------------------|
| `RES_4001` | 文件体积超过服务器分片组接限制 | 提示“单文件体积超标，请压缩至规约限制内后重试” |
| `RES_4002` | 文件的指纹或扩展提取未能与白名单安全库匹配| 提示“系统暂不支持该文件格式，请上传标准格式文件” |
| `RES_4004` | **[已废弃]** 用于精确完全一致秒传命中 | *(此码转换为内部态，命中绝对一致查重时 API 直返 200 并接管资料同号复用，禁止阻断报错)* |
| `RES_4005` | AI特征向量判定新物料语义重复相像度(>80%) | 唤出确认框：“检测到平台包含高度相似的语义内容，是否强制提交进入人工审核？” |
| `RES_4006` | 文件经特征沙盒截获含有机考违禁元素 | 阻截落库请求，抛射警示“文件未通过安全校验，存在病毒或违规内容风险” |
| `RES_4007` | 用户的月度免费总量消耗殆尽再兼缺豆结账阻绝 | 阻断下载跳转，提示“免费额度已耗尽，请兑换或购买流量包” |
| `RES_4008` | 所提取资源物理注销或 Lifecycle 为 DELETED | 传导标准 HTTP 404 表述“对不起，该资源已被删除或原始链接失效” |
| `RES_4009` | OSS 在线解析非标准化文件/加密 PDF 打回失败 | 提示“该文件格式不支持在线预览，请下载后查看” |

### 2.5 社区互动与发帖模块 (COM) (映射 HTTP 400/403)
| 错误码 | 描述说明 | 前端建议交互 / 应对策略 |
|--------|---------|-----------------------|
| `COM_4001` | 回复或是发帖文理间遭遇系统黑名单屏蔽网 | 提示“您的内容包含系统违禁词汇，请检查并修改后再提交” |
| `COM_4002` | 发帖水军防灌水阈值触界 (频率限速) | 提示“您发文过于频繁，请稍后再试” |
| `COM_4003` | 向强制关闭锁定或者终结的帖子回发增量 | 提示“该话题已关闭，请选择其他话题参与讨论” |
| `COM_4004` | 悬赏标的物尝试投向本人进行自我采纳 | 提示“系统规则不允许采纳本人发布的回复作为最佳答案” |
| `COM_4005` | 连续等幂冲按捕捉到的重复点赞干预 | 提示“您已点赞过该内容，请勿重复操作” |
| `COM_4007` | 针对已经被打上最佳标识的话题重发悬赏令 | 提示“该话题已有最佳答案，不可重复采纳” |

### 2.6 系统底层网关 (SYS) (映射 HTTP 429/500/502/503)
| 错误码 | 描述说明 | 前端建议交互 / 应对策略 |
|--------|---------|-----------------------|
| `SYS_4290` | (HTTP 429) 应用层规则独立计数的阻击防刷限流 | 提示"操作过于频繁，请稍后重试" |
| `SYS_5000` | (HTTP 500) 主线逻辑代码越界引发的逻辑性抛错 | 展现兜底图形提示"服务器处理异常，请稍后重试" |
| `SYS_5001` | (HTTP 503) 服务主动下线或实施定期维护升级 | 从 `data.maintenance_until` 提取时长提示"系统升级维护中，预计于 X 点恢复开放" |
| `SYS_5002` | (HTTP 502/504) 对外组件（支付/OSS）建立RPC时的通讯超时 | 提示"第三方服务通信超时，请稍后重试" |

### 2.7 搜索模块 (SEARCH) (映射 HTTP 400/401/403/404/429/500)
| 错误码 | 描述说明 | 前端建议交互 / 应对策略 |
|--------|---------|-----------------------|
| `SEARCH_4001` | 搜索词为空或长度超限（1-50 字符） | 从 `data.actual_len` 取实际长度，提示"请输入 1-50 个字符的搜索词" |
| `SEARCH_4002` | `type` 参数枚举值非法 | 从 `data.allowed` 展示合法范围，提示"搜索类型参数错误" |
| `SEARCH_4003` | 日期格式错误或 `end_date` 早于 `start_date` | 提示"请输入格式正确的日期范围（YYYY-MM-DD）" |
| `SEARCH_4004` | `sort_by` 参数枚举值非法 | 提示"排序参数错误"，恢复默认排序值 |
| `SEARCH_4011` | 未登录或 Token 失效，访问需鉴权的搜索接口 | 跳转登录页，保存当前搜索词以便登录后还原 |
| `SEARCH_4031` | AI 历史搜索越权访问（非本人） | 提示"仅能搜索您自己的 AI 对话记录" |
| `SEARCH_4041` | 搜索历史记录 ID 不存在或已删除 | 从 `data.history_id` 取 ID，提示"该记录不存在或已被删除"，静默刷新列表 |
| `SEARCH_5001` | 搜索服务内部异常（如索引服务不可用） | 提示"搜索服务暂时不可用，请稍后重试"，提供刷新按钮 |

## 3. OpenAPI 与 FastAPI 开发实现建议（Implementation Guide）
1. **统一异常层门面封包**：
   严禁后端开发者在 Router 业务块层面上直接 `return {"code": 500, "msg": "xx"}`。系统外壳必须实现全局异常拦截拦截器接盘基类错误（如 `BusinessException(ErrorCodeEnum.ECON_4001, data={"deficit": 10})`）。
2. **OpenAPI (Swagger) 精确展现**：
   在 FastAPI 配置中，请广泛利用内置注解结构如 `@router.get(..., responses={401: {"model": ErrorResponse}, 403: {"model": ErrorResponse}})` ，让前端开发员一瞥查阅标准化承兑模型。
3. **占位下放隔离 (Payload Vs Notification Msg)**：
   绝不允许把动态差额或时间信息人工嵌进返回的 `msg`。一切具备变动的实勘数值，必须全部存放于 `data: {}` 区域给终端拼合组装（如 `current_reputation`，`ban_until` 等明细对等属性变量）。
