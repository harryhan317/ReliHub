## 1. 概述

## 2. 数据库 Fixture 使用规范

### 2.1 强制要求

> ⚠️ **重要**: 所有测试必须使用 PostgreSQL 数据库，严禁使用 SQLite。

| 要求项 | 说明 |
|--------|------|
| **数据库类型** | PostgreSQL 15+ |
| **禁止使用** | SQLite、MySQL 等非 PostgreSQL 数据库 |
| **Fixture 来源** | 必须使用 `conftest.py` 提供的共享 fixture |

### 2.2 正确的 Fixture 使用方式

#### ✅ 正确示例

```python
class TestUserService:
    """Test UserService CRUD operations"""
    
    @pytest.fixture
    def user_service(self, db_session):
        """Create UserService instance with shared db_session"""
        return UserService(db_session)
    
    def test_create_user(self, db_session, user_service):
        """Test creating a new user"""
        user = user_service.create_user(
            phone="13800138000",
            nickname="test_user"
        )
        
        assert user.id is not None
        assert user.phone == "13800138000"
```

#### ❌ 错误示例

```python
class TestUserService:
    """Test UserService CRUD operations"""
    
    @pytest.fixture
    def db_session(self):  # 错误：重复定义 db_session
        engine = create_engine("sqlite:///:memory:")  # 错误：使用 SQLite
        Base.metadata.create_all(engine)
        SessionLocal = sessionmaker(bind=engine)
        session = SessionLocal()
        yield session
        session.close()
```

### 2.3 Fixture 导入方式

| 导入方式 | 说明 | 状态 |
|----------|------|------|
| 自动发现 | pytest 自动发现 `conftest.py` 中的 fixture | ✅ 推荐 |
| 显式导入 | 不需要显式导入，pytest 自动注入 | ✅ 正确 |
| 重复定义 | 在测试类中重新定义同名 fixture | ❌ 禁止 |

## 3. Fixture 作用域设置

### 3.1 作用域选择指南

| 作用域 | 使用场景 | 示例 |
|--------|----------|------|
| `function` | 每个测试函数独立数据 (默认) | `db_session` |
| `class` | 类中所有测试共享数据 | 测试类共享配置 |
| `module` | 模块中所有测试共享数据 | 模块级初始化 |
| `session` | 整个测试会话共享数据 | 数据库连接池 |

### 3.2 当前项目 Fixture 定义

```python
# conftest.py

@pytest.fixture(scope="function")
def db_session():
    """
    Create a fresh PostgreSQL database session for each test.
    Each test gets a clean database state via transaction rollback.
    """
    engine = create_engine(TEST_DATABASE_URL, echo=False, future=True)
    connection = engine.connect()
    transaction = connection.begin()
    Base.metadata.create_all(connection)
    
    SessionLocal = sessionmaker(bind=connection, class_=Session, expire_on_commit=False)
    session = SessionLocal()
    
    try:
        yield session
    finally:
        session.close()
        transaction.rollback()
        connection.close()
        engine.dispose()
```

## 4. 数据初始化逻辑

### 4.1 测试数据创建规范

| 规范项 | 说明 |
|--------|------|
| **独立性** | 每个测试用例应独立创建所需数据 |
| **可重复性** | 测试数据应可重复创建，不依赖外部状态 |
| **清理机制** | 使用事务回滚自动清理数据 |

### 4.2 测试数据创建示例

```python
class TestResourceService:
    """Test ResourceService"""
    
    @pytest.fixture
    def resource_service(self, db_session):
        return ResourceService(db_session)
    
    @pytest.fixture
    def test_user(self, db_session):
        """Create test user for each test"""
        user = User(
            id=str(uuid.uuid4()),
            phone="13800138000",
            nickname="test_user",
        )
        db_session.add(user)
        db_session.commit()
        return user
    
    @pytest.fixture
    def test_category(self, db_session):
        """Create test category for each test"""
        category = ResourceCategory(
            id=1,
            name="测试分类"
        )
        db_session.add(category)
        db_session.commit()
        return category
    
    def test_create_resource(self, db_session, resource_service, test_user, test_category):
        """Test creating a resource"""
        request = ResourceCreateRequest(
            title="Test Resource",
            description="Test description",
            category_id=test_category.id,
        )
        
        resource = resource_service.create_resource(
            uploader_id=test_user.id,
            request=request
        )
        
        assert resource.id is not None
```

## 5. 测试用例与 Fixture 关联性

### 5.1 Fixture 依赖注入

测试用例通过函数参数声明 fixture 依赖，pytest 自动注入：

```python
def test_example(db_session, test_user, resource_service):
    # db_session: 数据库会话 (来自 conftest.py)
    # test_user: 测试用户 (来自类级别 fixture)
    # resource_service: 服务实例 (来自类级别 fixture)
    pass
```

### 5.2 Fixture 组合使用

```python
class TestTopicService:
    """Test TopicService"""
    
    @pytest.fixture
    def topic_service(self, db_session):
        return TopicService(db_session)
    
    @pytest.fixture
    def test_user(self, db_session):
        user = User(id=str(uuid.uuid4()), phone="13800138000", nickname="test_user")
        db_session.add(user)
        db_session.commit()
        return user
    
    @pytest.fixture
    def test_category(self, db_session):
        category = TopicCategory(id=1, name="技术讨论")
        db_session.add(category)
        db_session.commit()
        return category
    
    # 测试用例自动获取所有需要的 fixture
    def test_create_topic(self, topic_service, test_user, test_category):
        request = TopicCreateRequest(
            title="Test Topic",
            content="Test content",
            category_id=test_category.id,
        )
        
        topic = topic_service.create_topic(
            author_id=test_user.id,
            request=request
        )
        
        assert topic.id is not None
```

## 6. 代码审查检查清单

### 6.1 必查项目

| 检查项 | 检查内容 | 通过标准 |
|--------|----------|----------|
| **数据库类型** | 是否使用 PostgreSQL | 必须使用 PostgreSQL |
| **Fixture 来源** | 是否使用共享 fixture | 必须来自 conftest.py |
| **重复定义** | 是否重复定义 db_session | 不允许重复定义 |
| **作用域设置** | fixture 作用域是否合理 | 符合使用场景 |
| **数据隔离** | 测试数据是否独立 | 每个测试独立创建数据 |

### 6.2 审查流程

```
1. 检查测试文件头部是否有 SQLite 相关导入
   ↓
2. 搜索 create_engine("sqlite") 或 :memory:
   ↓
3. 检查是否有类级别 db_session fixture 定义
   ↓
4. 验证测试方法是否正确使用 fixture 参数
   ↓
5. 确认测试数据创建和清理逻辑
```

### 6.3 常见问题及解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 外键约束错误 | 未创建关联数据 | 先创建父记录再创建子记录 |
| 数据污染 | 测试间数据未清理 | 确保使用事务回滚 |
| Fixture 未找到 | fixture 未正确定义 | 检查 conftest.py 位置 |
| 连接池耗尽 | fixture 未正确关闭 | 确保 finally 块中清理资源 |

## 7. 自动化检查脚本

### 7.1 SQLite 使用检查

```bash
#!/bin/bash
# check_sqlite.sh

echo "Checking for SQLite usage in test files..."

SQLITE_FILES=$(grep -r "create_engine.*sqlite" backend/tests --include="*.py" -l 2>/dev/null || true)
MEMORY_DB=$(grep -r ":memory:" backend/tests --include="*.py" -l 2>/dev/null || true)

if [ ! -z "$SQLITE_FILES" ]; then
    echo "❌ Found SQLite engine creation in:"
    echo "$SQLITE_FILES"
    exit 1
fi

if [ ! -z "$MEMORY_DB" ]; then
    echo "⚠️ Found :memory: references in:"
    echo "$MEMORY_DB"
fi

echo "✅ SQLite check passed"
```

### 7.2 Fixture 使用检查

```python
# check_fixtures.py
import os
import re

def check_fixture_usage(test_dir):
    issues = []
    
    for root, dirs, files in os.walk(test_dir):
        for file in files:
            if file.startswith('test_') and file.endswith('.py'):
                filepath = os.path.join(root, file)
                
                with open(filepath, 'r') as f:
                    content = f.read()
                
                # Check for SQLite engine
                if 'create_engine("sqlite' in content or "create_engine('sqlite" in content:
                    issues.append(f"{filepath}: Uses SQLite engine")
                
                # Check for class-level db_session fixture
                if re.search(r'class\s+\w+.*:\s*\n.*@pytest\.fixture.*\n.*def\s+db_session', content, re.DOTALL):
                    issues.append(f"{filepath}: Redefines db_session fixture")
    
    return issues

issues = check_fixture_usage('backend/tests')
if issues:
    print("Issues found:")
    for issue in issues:
        print(f"  - {issue}")
else:
    print("✅ All fixtures validated")
```
