# 测试用例 - 意见反馈 (TC_意见反馈)

## 1. 模块概述
意见反馈模块连接了用户与平台运营。测试重点在于 **FB 业务单号的生成逻辑**、**管理员回复后的状态流转** 以及 **用户注销后的匿名化哈希关联**。

---

## 2. P0 冒烟测试 (Smokes)

### TC-FB-001 意见反馈提交与单号生成 (P0)
- **优先级**: P0
- **前置条件**: 用户已登录。
- **测试步骤**:
  1. 调用 `/api/v1/feedback` 提交一条功能异常反馈。
  2. 验证响应主体。
- **预期结果**:
  - 返回 HTTP 201。
  - 获取到 `business_id` (格式示例：`FB20260402000001`)。
  - 数据库 `feedbacks.status` 初始值为 `PENDING`。
- **接口依赖**: `/api/v1/feedback`

---

### TC-FB-002 管理员回复与工单结单 (P0)
- **优先级**: P0
- **前置条件**: 某反馈工单处于 `PENDING` 状态。
- **测试步骤**:
  1. 管理员登录后台，定位该工单。
  2. 调用 `/api/v1/admin/feedbacks/{id}/reply` 填写回复内容。
- **预期结果**:
  - 工单 `status` 更新为 `RESOLVED`。
  - `handler_id` 记录该管理员 UUID。
  - 用户收到“反馈已处理”的通知消息。
- **接口依赖**: `/api/v1/admin/feedbacks/{id}/reply`

---

## 3. P1 核心功能测试

### TC-FB-003 用户注销后的匿名化哈希留存验证 (关键)
- **优先级**: P1
- **前置条件**: 用户 A 已提交过 3 条历史反馈。
- **测试步骤**:
  1. 用户 A 申请注销。
  2. 管理员在后台查看这 3 条历史反馈。
- **预期结果**:
  - 反馈记录中的 `user_id` 已更新为 `0000...0000` (匿名 UUID)。
  - `anonymized_user_hash` 字段成功写入值（HMAC-SHA256 生成）。
  - **核心验证**：这 3 条反馈记录的 `anonymized_user_hash` 必须**完全一致**（确保管理员能识别是同一匿名人所提意见）。
  - `contact_info` (手机/邮箱) 字段已**物理置 NULL**。
- **数据来源**: `DB_反馈.md` §2.1

## 4. P3 异常与边界

### TC-FB-901 图片附件上传上限拦截
- **优先级**: P3
- **前置条件**: 准备 4 张图片附件。
- **测试步骤**:
  1. 尝试提交包含 4 张图片的反馈。
- **预期结果**:
  - 拦截并提示“反馈附件最多支持 3 张图片”。
  - 校验图片单张体积 > 5MB 时的拦截。
