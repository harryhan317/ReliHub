# 测试用例 - 管理后台 (TC_管理后台)

## 1. 模块概述
管理后台是 ReliHub 的指挥中心。测试重点在于 **管理员 IP 子网锁定安全校验**、**资源的基础标识维护** 以及 **全量广播通知的频率限制**。

---

## 2. P0 冒烟测试 (Smokes)

### TC-M5-001 管理员登录与 IP 子网白名单过滤
- **优先级**: P0
- **前置条件**: 已预设管理员张三的信息，且其白名单子网为 `10.20.30.0/24`。
- **测试步骤**:
  1. 模拟管理员使用真实 IP `10.20.30.5` 进行登录。
  2. 模拟管理员使用未经授权 IP `192.168.1.1` 进行登录。
- **预期结果**:
  - `10.20.30.5`: 成功获取鉴权 Token。
  - `192.168.1.1`: 接口返回阻断态标准错误码 `AUTH_4006`，并在响应正文中抛出限定文案："当前网络环境受限，请联系运维确认"。
- **安全基准**: `架构设计_安全设计.md` §3.1

---

### TC-M5-002 资源标注与标签演进 (is_seed)
- **优先级**: P0
- **前置条件**: 管理员已登录。
- **测试步骤**:
  1. 定位某个普通资源，修改其属性 `is_seed = true`。
  2. 定位另一个资源，手动执行“下架”操作。
- **预期结果**:
  - `resources.is_seed` 更新为 `true`。
  - 下架资源的 `status` 更新为 `REJECTED` (或已下架态)，且在用户公开列表**不可见**。
- **流转引用**: `PRD_内容生命周期管理.md` §2.4

---

## 3. P1 核心功能测试

### TC-M5-003 全量广播通知频率限制验证
- **优先级**: P1
- **前置条件**: 今天已发送过一次名为“系统维护公告”的全量通知。
- **测试步骤**:
  1. 尝试发送第二次全量通知。
- **预期结果**:
  - 拦截发送并提示“全量广播类通知每日仅限 1 次，请谨慎操作”。
  - 验证通知队列中未产生重复数据。
- **规范来源**: `PRD_MVP_通知系统.md` §6.1

---

### TC-M5-004 用户封禁与解封联动验证
- **优先级**: P1
- **前置条件**: 定位活跃用户 A。
- **测试步骤**:
  1. 执行“全限封禁”操作，时长设置为 24 小时。
  2. 模拟用户 A 再次尝试登录。
  3. 执行“手动解封”。
- **预期结果**:
  - 封禁中登录：接口返回 `USER_4010` (账户封禁)，数据携带 `ban_until` 日期。
  - 解封后登录：成功登录，状态恢复为 `NORMAL`。

---

## 4. P3 异常与边界

### TC-M5-901 系统提示词 (Prompt) 注入防御测试
- **优先级**: P3
- **前置条件**: 管理员已登录后台 Prompt 配置页。
- **测试步骤**:
  1. 尝试提交包含 SQL 注入或恶意 Python 代码的 Prompt。
- **预期结果**:
  - 服务端执行严格的转义处理。
  - 发端前进行敏感字符初检。
