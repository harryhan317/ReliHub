# 测试用例 - 搜索模块 (TC_搜索模块)

## 1. 模块概述
搜索模块是 ReliHub 的核心发现机制，支持对资源、话题及全站内容的检索。测试重点在于**检索精准度**、**多维度过滤**以及**性能响应**。

---

## 2. P0 冒烟测试 (Smokes)

### TC-SEARCH-001 全站关键字基础搜索
- **优先级**: P0
- **前置条件**: 数据库中已存在包含关键字 "Python" 的资源和话题。
- **测试步骤**:
  1. 在搜索栏输入 "Python"。
  2. 点击搜索按钮或回车。
  3. 观察返回的结果列表。
- **预期结果**:
  - 返回 HTTP 200。
  - 结果列表包含标题或描述中含有 "Python" 的资源。
  - 结果按 "热度" (heat_score) 默认降序排列。
- **接口依赖**: `/api/v1/resources` (含 search 参数)

---

## 3. P1 核心功能测试

### TC-SEARCH-002 资源按分类过滤
- **优先级**: P1
- **前置条件**: 存在不同分类（如：编程语言、设计稿）的资源。
- **测试步骤**:
  1. 选择分类 "编程语言"。
  2. 执行搜索。
- **预期结果**:
  - 仅返回分类 ID 匹配的资源。
  - 验证空结果集时的显示逻辑。

### TC-SEARCH-003 多重排序验证
- **优先级**: P1
- **前置条件**: 存在多份资源。
- **测试步骤**:
  1. 分别按 "时间"、"下载量"、"价格" 进行排序。
- **预期结果**:
  - 排序结果与数据库实际数据一致。
  - 验证价格降序/升序切换。

---

## 4. P2 性能与安全

### TC-SEARCH-004 搜索敏感词拦截
- **优先级**: P2
- **前置条件**: 系统已配置敏感词库。
- **测试步骤**:
  1. 输入包含违规内容（如：政治敏感、色情）的关键字进行搜索。
- **预期结果**:
  - 系统拦截搜索请求。
  - 返回错误码 `AUTH_4006` (或对应的敏感词拦截错误)。
  - 不展示违规内容相关的任何结果。

### TC-SEARCH-005 大规模结果集分页
- **优先级**: P2
- **前置条件**: 数据库中有超过 100 条匹配结果。
- **测试步骤**:
  1. 切换页码（第 1 页 -> 第 5 页）。
  2. 修改 `page_size` 为 50。
- **预期结果**:
  - 数据不重复、不遗漏。
  - 分页参数有效作用于 SQL 语句。
