# ReliHub 测试文档总览 (Testing Overview)

本文档定义了 ReliHub MVP 阶段的测试策略、准入/准出标准以及环境配置要求，为研发与测试团队提供统一的质量验收基准。

---

## 1. 测试策略

ReliHub 采用多级测试模型，确保从 API 原子性到业务长链路的高可靠性。

| 测试类型 | 目标 | 执行方式 |
|---------|------|---------|
| **冒烟测试 (P0)** | 核心业务路径（注册、下载、发奖、结算） | 自动化 (CI/CD) |
| **API 测试 (P1)** | 接口规约、错误码（MOD_XXXX）覆盖 | 自动化 (Postman/Pytest) |
| **金融审计 (P0)** | 取整计算 (floor)、SYSTEM_BURN 账户对账 | 灰盒 (DB 校验) |
| **业务状态机 (P1)** | 审核流转、等级变更、注销匿名化 | 黑盒 + 数据埋点 |
| **安全性测试 (P1)** | 管理员 IP 锁定、Blind Index 擦除、违禁词拦截 | 灰盒 |

## 2. 测试环境准备

在开始执行测试用例前，必须部署并核实以下环境组件：

### 2.1 依赖组件
- **数据库**: PostgreSQL 15+ (必须安装 `pgvector` 扩展)。
- **缓存**: Redis 7.2+ (必须配置 Redis 哨兵或集群模式)。
- **背景任务**: Celery 5.3+ (需启动 `celery_worker` 与 `celery_beat`)。

### 2.2 测试数据库要求

> ⚠️ **重要**: 所有测试必须使用 PostgreSQL 数据库，严禁使用 SQLite 或其他非标准数据库。

| 要求项 | 说明 |
|--------|------|
| **数据库类型** | PostgreSQL 15+ (与生产环境保持一致) |
| **禁止使用** | SQLite、MySQL、其他非PostgreSQL数据库 |
| **测试数据库** | `relihub_test` (独立于开发/生产数据库) |
| **连接配置** | `TEST_DATABASE_URL=postgresql://postgres:postgres@localhost:5432/relihub_test` |
| **隔离策略** | 每个测试用例使用事务回滚，确保数据隔离 |

**为什么必须使用 PostgreSQL？**
1. **外键约束**: PostgreSQL 严格强制外键约束，能提前发现数据完整性问题
2. **数据类型兼容**: JSON、BOOLEAN、ARRAY 等数据类型与 SQLite 存在差异
3. **性能测试准确性**: PostgreSQL 的查询优化器与 SQLite 差异较大
4. **生产环境一致性**: 确保测试结果与生产环境行为一致

### 2.3 测试工具
- **API 客户端**: Postman v10.x 或 Insomnia。
- **自动化框架**: Pytest + Requests (针对后端 API)。
- **AI Mock**: 在开发环境下建议开启 `MOCK_AI_RESPONSE=True` 以绕过 LLM Token 成本。

## 4. 准入与准出标准

### 4.1 准入标准 (Entry Criteria)
- 对应的模块接口文档 (`API_*.md`) 已封板。
- 数据库结构 (`DB_*.md`) 已在测试库完成迁移。
- `seed_test_data.sql` 已成功导入测试环境。

### 4.2 准出标准 (Exit Criteria)
- 所有 **P0 (冒烟)** 测试用例 100% 通过。
- 所有 **P1 (核心)** 测试累计通过率 ≥ 95%。
- 已定义的 **错误码 (MOD_XXXX)** 在异常场景下响应准确率 100%。
- 金融分账产生的残余碎银 100% 准确归入 `SYSTEM_BURN` 账户。
