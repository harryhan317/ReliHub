# ReliHub 项目文档门户 (Documentation Portal)
 
 欢迎来到 ReliHub 项目文档库。本项目采用“需求引导开发”模式，文档分为产品需求 (PRD) 与技术设计 (TDD) 两大核心支柱。
 
 ---
 
 ## 📂 目录导航
 
 ### 1. [01_产品需求 (PRD)](./01_产品需求/)
 - **核心内容**：总体方案、功能清单、非功能需求、MVP 专项 PRD。
 - **阅读对象**：产品、设计、测试、开发及利益相关方。
 - **状态**：✅ 100% 对齐审计版。
 
 ### 2. [02_技术设计 (TDD)](./02_技术设计/)
 - **核心内容**：系统架构、数据库建模、API 定义、集成方案。
 - **阅读对象**：开发人员、架构师、前端/后端负责人。
 - **状态**：🏗️ 建设中 (底层支架已搭建)。
 
 ### 3. 03_测试文档 (Planned)
 - 测试计划、用例、自动化脚本。
 
 ### 4. 04_运维文档 (Planned)
 - Docker 部署、环境配置、监控说明。
 
 ---
 
 ## 📁 文档维护准则
 - **单向依赖**：技术设计 (02) 必须遵循产品需求 (01) 定义的业务逻辑。
 - **版本对齐**：任何重大需求变更必须先更新 `01_产品需求` 中的对应的 PRD，再同步调整对应的技术文档。
 - **命名规范**：遵循 `docs/01_产品需求/README.md` 中定义的命名约定。
