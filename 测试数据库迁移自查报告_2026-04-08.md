## 7. 整改方案

### 7.1 已完成整改

#### 整改项1: 外键约束问题修复

**问题描述**: `test_ai_integration.py` 中的 `test_create_message` 测试使用不存在的 `session_id`，导致PostgreSQL外键约束错误。

**整改措施**:
```python
# 整改前
def test_create_message(self, db_session):
    service = AIMessageService(db_session)
    message = service.create_message(
        session_id="session-123",  # 不存在的session
        ...
    )

# 整改后
def test_create_message(self, db_session):
    session_service = AISessionService(db_session)
    message_service = AIMessageService(db_session)
    
    session = session_service.create_session("user-123", MockRequest())
    message = message_service.create_message(
        session_id=session.id,  # 使用有效的session_id
        ...
    )
```

**整改结果**: 测试通过，外键约束正常工作

#### 整改项2: 移除重复fixture定义

**问题描述**: 多个测试类重复定义 `db_session` fixture，造成代码冗余。

**整改措施**: 移除类级别的 `db_session` fixture，统一使用 `conftest.py` 提供的共享fixture。

**整改结果**: 代码简化，维护性提升
